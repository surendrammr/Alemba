We have received increased reports from Customers of the browser tab crashing when Analysts are performing actions in Alemba Service Manager Core Interface.

Following our initial investigations, we believe this is being caused by a recent change to the underlying Chromium Browser Engine that is used by Google Chrome and Microsoft Edge.

A change was introduced in version v104 and is present in v105 to restrict the amount of memory the browser tabs use on the Analysts Client Device.

Details of the change can be found in the Chromium Bug Report.

We are awaiting the M106 Stable Release of Chromium Browsers where this issue should now be resolved. The current release date for this is the 27th September 2022.

We will continue to monitor this situation with this Browser release.

How does this affect Alemba Customers?

Alemba’s Ongoing Support

Alemba will continue to support customers and will provide the help and assistance. We recommend that affected Analysts can mitigate the effect of this Browser Issue by:

  • Close ASM Tabs when not in use.
  • Avoid performing very large searches.
  • Avoid Bulk operations such as Bulk Call Forward, Bulk Call Close or Bulk Actioning/Viewing or Opening Calls.
  • Try using Firefox to access ASM Core.
  • Use ASM Nano Interface for Analysts.
  • Regularly Save any updates made to items.
  • Regularly logging out of ASM and closing the ASM Browser Tab.
  • Restarting their Browser at regular intervals.
  • Remove/Disable any Browser Extensions.

You are also able to test the latest Google Chrome Beta or Microsoft Edge Beta.
