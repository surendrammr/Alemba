Microsoft have announced that effective of 1st October 2022 that they will start to permanently disable Basic Authentication (also known as Legacy Authentication) in all tenants with Microsoft Office 365/Exchange Online.

Basic Authentication is the use of a username and password to connect, for example, to your ASM Incoming or Outgoing Email Server(s).

The following Protocols are affected: Exchange Web Services (EWS), POP, IMAP, Exchange ActiveSync (EAS), MAPI, RPC, Offline Address Book (OAB) and Remote PowerShell.

SMTP Auth will also be disabled if it is not currently being used.

How does this affect Alemba Customers?

All customers using Alemba Service Manager (ASM) and vFire Core who use Microsoft Office 365/Exchange Online to receive and process emails. This may include the sending of system notifications and the ability to log new calls, update calls, requests and tasks. It may also affect the use of the Email Event Connector.

If you are hosting your own email services on a Microsoft Exchange server in your local domain, you should not be affected by this upcoming cloud service change by Microsoft. However, it is recommended that you confirm this directly with your Microsoft account manager.

How will you know you may have a problem?

Incoming Emails will stop being processed from your configured Incoming Email Server(s) and you should see the following error in the Event Logs: HTTP 401 error: bad username or password.

What options do Alemba provide?

Alemba Service Manager v10.3 (and above) already supports Modern Authentication to connect to Microsoft Office 365 using OAuth 2.0 through Exchange Web Services (EWS). Alemba recommends reconfiguring your Incoming Email Servers to use OAuth2.0/EWS as soon as possible to avoid any possible disruption.
https://alemba.com/releases/asmx-10-3#oauth

Details of how to configure ASM for OAuth2.0/EWS can be found in this document.

Alemba Service Manage EOS v10.5.5 (and above) customers can also use the MS Graph API for Incoming and Outgoing Email.

Details of how to configure ASM to use the MS Graph API can be found in this document.

We are on an older version of vFire Core or Alemba Service Manager, what can we do?

Upgrade to the latest version of Alemba Service Manager (currently ASM EOS v10.5.8) to use OAuth2.0/EWS or MS Graph API connections.

There are many great new features and performance improvements and Alemba recommends all customers to update the software to take advantage of these improvements. For more details on this release, please visit our blog.

If this option is of interest, then please raise this with your Technical Account Manager or by raising a “Get Latest Version” request through the Alemba Support Portal.

Update:

Alemba are pleased to be able to announce that following our investigations we will be able to provide Customers with a patch to enable email connectivity via the Microsoft Graph API.

This patch is available to Customer Systems on vFire Core v9.3 and above. There is an expected lead time of 2 weeks to deliver the patch to customers.

To request this patch, please use the “Add Me” button to the Known Issue available via the Alemba Support Portal Homepage.

Details of how to configure vFire/ASM to use the MS Graph API can be found in this document.

MS Graph API

We are investigating the possibility of whether it is technically possible to support the use of the MS Graph API to older versions of Alemba Service Manager and vFire Core.

Please register your interest by logging a Support Call through the Alemba Support Portal and we will update you as to our progress investigating this option.

Please Note: There is no guarantee that this option will be technically possible, so customers are advised to also proceed with an alternate option as their primary solution to this issue. Alemba will provide an update at the end of June 2022.

Alemba’s Ongoing Support

Alemba will continue to support customers and will provide the help and assistance required to transition to a Modern Authentication method during and after that date.

You can see updated details of all our planned software releases and schedule at: https://alemba.com/product-roadmap.
