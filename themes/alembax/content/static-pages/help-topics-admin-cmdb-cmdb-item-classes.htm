CMDB Item Classes

CMDB item classes allow you to profile your organization’s CMDB items or categorize them according to their function in an organization, for example, Sales, or Marketing. You can create, rename or delete classes. Classes are not linked to CMDB item types.

You may wish to classify your organization’s CMDB items by defining and configuring your own CMDB Item Types. ASM Core supports the creation of a multi-level hierarchy of CMDB entity types but not of CMDB Item Classes.

Creating a CMDB Item Class

Before you start

You must have CMDB Setup enabled within your General Access security role in order to configure any CMDB administration settings. Depending on if or how your system is partitioned, ensure that you are working in the correct partition.

  1. Select ≡ > Admin > System Administration.

    The System Administration window appears.

    In the Explorer pane, expand CMDB.

  2. Select CMDB Item Classes to open the window.
  1. Select . A new row is added to the browse table.
  2. In the Name column, type the name of the class you want to create.
  3. Select to save the changes. Provide the Change Reasons if prompted to do so.

Viewing CMDB Item Classes

  1. Select ≡ > Admin > System Administration.

    The System Administration window appears.

    In the Explorer pane, expand CMDB.

  2. Select CMDB Item Classes to open the window.
  3. Existing CMDB Item Classes are displayed in a browse table.

Renaming a CMDB Item Class

Before you start

You must have CMDB Setup enabled within your General Access security role in order to configure any CMDB administration settings. Depending on if or how your system is partitioned, ensure that you are working in the correct partition.

  1. Select ≡ > Admin > System Administration.

    The System Administration window appears.

    In the Explorer pane, expand CMDB.

  2. Select CMDB Item Classes to open the window.
  1. Select the item class you want to edit.
  2. Type the name you want to assign to the class.
  3. Select to save the changes. Provide the Change Reasons if prompted to do so.

Deleting a CMDB Item Class

Before you start

You must have CMDB Setup enabled within your General Access security role in order to configure any CMDB administration settings. Depending on if or how your system is partitioned, ensure that you are working in the correct partition.

You cannot reverse this procedure. If you delete a class by mistake, you have to re-create the entry.

  1. Select ≡ > Admin > System Administration.

    The System Administration window appears.

    In the Explorer pane, expand CMDB.

  2. Select CMDB Item Classes to open the window.
  1. From the browse table, select the class entry you want to delete.
  2. Select . If you attempt to delete the system default, you will receive a warning message.
