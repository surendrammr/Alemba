Creating a Notification Workflow

Before you start

The following settings must be in place before you can build a rule:

  • IPK Workflow Rules Builder must be enabled in IPK Settings .
  • IPK Workflow must be selected in the IPK Management Role: Options Tab of your IPK Management Security Role.

In ASM, you can create Notification Workflows, that is, workflows that send an email, pager, or internal message to specified analysts or groups. You can also select the phase at which the notification is to be sent: on call log/update, call resolution and/or call closure.

  1. Select Menu and then Admin. From the submenu, select IPK Workflow Rules. The IPK Workflow Rules window appears. The explorer pane on the left of this window shows three groups: IPK Workflow Rules, Forum Management and Auto Close Configuration.
  2. From the IPK Workflow Rules group, select IPK Notification Workflow.
  3. The IPK Notification Workflow window appears. It lists all the Notification Workflows that have been defined in ASM. To add a new workflow, click .
  4. Set up the Notification workflow.
