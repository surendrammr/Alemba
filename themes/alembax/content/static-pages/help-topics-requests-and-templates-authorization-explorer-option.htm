Authorizing Requests

The Authorization Explorer option enables you to authorize the current request so it can start being implemented. Authorizing a request releases all tasks and approvals in the request.

You can pre-authorize requests by authorizing the workflow template. By default (unless configured differently by the system administrator), tasks and approvals are not authorized when first created.

Before you start

Authorization must be selected in the Workflow Management Settings window to activate the explorer option.

If the request is pre-authorized (at the level of the workflow template), the option is disabled on the Request Details window. If a request has been authorized either in the template or manually, and additional tasks are added to the workflow (at the level of the request), these tasks are added as un-authorized and suspended. The Authorize button will be enabled if there are any unauthorized tasks.

  3. Complete the details.

    4. Status Select the status which best reflects the reason for your authorization or rejection of the request
    Password Type in the password you use to log on to ASM Core. This level of security ensures that an analyst with the appropriate permissions is authorizing the request. (Depending on the settings by the system administrator, you may not have to enter your password.)
    Comments Add any comments to the authorization (or rejection) for the request
    Send Email

    Email must be selected under Workflow Management on the Messaging window within System Administration.

    Select to send an email to the analysts or groups who will receive tasks following authorization of the request. This is useful to notify those recipients who are not logged on to ASM Core that the request has been authorized.

    If you select this option, an email will be sent to the task assignees regardless of whether the Email option was selected on the task itself.

    This feature only applies to the tasks that have no dependencies: emails will only be sent to the recipients of tasks which are not dependent on the completion of other tasks.
    Send Pager

    Pager must be selected under Workflow Management on the Messaging window within System Administration.

    Select to send a pager message to the analysts or groups who will receive tasks following authorization of the request. This is useful to notify those recipients who are not logged on to ASM Core that the request has been authorized.

    This feature applies only to recipients of tasks that are not dependent.
    Email Requestor Select to notify the person who logged the request (for example, a user through the Self Service Portal) that their request has been authorized (or rejected).
  4. Click Authorize to release the tasks and approvals within the request for action. Otherwise, click Reject to reject the request, and confirm the rejection.

Once the request is authorized or rejected, a small orange square appears next to the Authorization option on the Explorer pane to indicate that a decision has been made on the request.

Pre-Authorizing a Workflow Template

Authorization must be selected in the Workflow Management Settings window.

You can pre-authorize requests based on a workflow template from the Workflow Template Details window. This means that requests based on the template will become active (and the dependent tasks automatically released) as soon as the request is saved. If the template on which the request is based is not pre-authorized, the request manager has to authorize each request before it becomes active.

Select the Authorization explorer option from the Workflow Template Details window, and follow the instructions as for a request.

If a request has been authorized either by pre-authorizing the template or manually, and additional tasks are added to the request workflow (at the level of the request), these tasks are added as un-authorized and suspended.

To release these tasks, the request manager needs to go to the Authorization window and authorize the new tasks in the same way the request was authorized.
