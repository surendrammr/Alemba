IPK Management Security Roles

An IPK Management security role covers permissions for calls and related entities. It also includes permissions for Call screen sets, IPK groups (including default groups), IPK statuses, IPK streams, and forms.

Before you start

You must have Security Roles setup selected in the Admin tab of your own General Access security role before you can assign or remove permissions for any security roles.

  1. Select ≡ > Admin > System Administration.

    The System Administration window appears.

    In the Explorer pane, expand Security Roles, then select IPK Management. All existing IPK Management security roles are displayed.

    The pre-defined role All typically has all role options selected. However, some of the options are not selected by default and you should explore the permissions set before assigning the role.

  2. The permissions for IPK Management security roles are available on several tabs. Select the appropriate one for your needs:
Options to set basic security rights
Groups to link groups to the security role and set the default group
Screen Sets to choose the call screen sets to which the role has access.

This tab appears in the place of IPK Status and Streams, when your system does not have Enable IPK Status and Streams selected on the IPK Settings window
IPK Statuses to choose the IPK statuses to which the role has access

This tab is when your system has IPK Status and Streams selected on the IPK Settings window
IPK Streams

to choose the IPK streams to which the role has access

This tab is when your system has IPK Status and Streams selected on the IPK Settings window
Forms to assign access levels to various forms, if they have been created
Quick Launch to assign access to quick launch applications the role can access when actioning requests and tasks.

This tab is only enabled when the Quick Launch functionality is enabled globally in system settings
