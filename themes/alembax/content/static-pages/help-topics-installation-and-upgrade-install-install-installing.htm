Installing Alemba Service Manager™

You install Alemba Service Manager™ on the server only.

Consult Alemba® Support before installing any third party software on the Alemba Service Manager™ server. Alemba Ltd is not responsible for any problems encountered with ASM because of third party software installed on the ASM server. We recommend that you install any third party software on a test server first, and review the Alemba® Knowledge Bank for any known issues.

For sites with higher security, we recommend that you set up SSL on the web server.

Before you start

Before you install Alemba Service Manager™:

  • Ensure your system meets the minimum technical requirements and is configured correctly. For more information, see Prerequisites checklist.
  • Obtain an ASM license key from your Account Manager.

  • If required, speak to your Account Manager about installing a Rapid Start database. Rapid Start is designed to get you started with key ITSM processes. It contains a set of preconfigured data and configurations.

    For more information about Rapid Start, see the Rapid Start Reference Guide.

  • If required, create a blank database in your database engine for a brand new ASM System.
  • Installing ASM will stop IIS and any dependent services. Ensure that you install ASM at a time when these services are not needed.

Downloading and installing the Alemba Service Manager™ software

  1. Using Self Service Portal, select Browse The Service Request Catalog, then submit an Order for the Alemba Service Manager 10.x Setup file (where 10.x is the appropriate version number).

    You will receive an email with a link to download the software.

  2. Download and extract the ZIP file.

  3. Right-click the Alemba Service Manager 10.x Setup.msi, then select Run as Administrator on the shortcut menu.

    The User Account Control window appears, prompting you to allow the Alemba Service Manager™ application to begin the setup process.

  4. Select Yes to open the InstallShield Welcome page.
  5. Select Next to continue.

  6. Read and accept the License Agreement terms and conditions, then select Next.

    The Customer Information window automatically displays your current login details.

  7. If required, modify this information, then select Next.

  8. On the Setup Type window, we recommend you select Complete to install ASM to the default directory: C:\Program Files\Alemba\Service Manager.

    Alternatively, to install the product on a different directory, select Custom, then select Next. On the Custom Setup page, select Change and browse to a different directory. Select OK to save your changes.

  9. Select Next to continue.
  10. In the Ready to Install the Program window, select Install to complete installation.

  11. On the InstallShield Wizard Completed page, select Finish to exit the InstallShield Wizard.

If you are installing a Rapid Start database, refer to the Rapid Start Installation Guide.

If you are not installing a Rapid Start database, use the Server Console to create and configure your new system. For more information, see the ASM Server Console Guide.
