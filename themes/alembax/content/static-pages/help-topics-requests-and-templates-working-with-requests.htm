Working with Requests

Requests are created when change is required, for example when you must implement a workaround to resolve a call logged at the Service Desk. When a request is created, the appropriate template, including predefined attributes and a dependency diagram linking all tasks and approvals, must be selected.

The main steps in managing a request are as follows:

  1. Create the request, using the appropriate workflow template.
  2. Create the tasks and approvals that need to be completed for the request and forward those that require manual completion to the appropriate analysts.
  3. If tasks and approvals have been suspended, authorize the tasks and approvals to release them for action.

    4. Tasks and approvals are automatically created as suspended when you are creating a request. You will need to authorize each task and approval to release them. Your system may be configured to automatically authorize all tasks on taking action on (saving, deferring, or forwarding) the request, or through the Authorization window.

  4. Action the request. You can save a request at any stage and add more details later if required.
  5. Complete the request after it is implemented.
