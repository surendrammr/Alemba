Freezing CMDB Items

ASM Core gives you the ability to ‘freeze’ items within the CMDB. Freezing an item sets a lock on the item. This means no modifications can be made to the item unless the item is part of a change request.

Freezing a CMDB item

Before you start

You must have CMDB Item Freeze selected in the CMDB Items tab of your Configuration Management security role.

At least one freeze status must have been defined, with the Frozen option selected.

You can link a non-frozen CMDB item to a frozen item. However, if two linked CMDB items are frozen, you will not be able to modify the link between them. If only one item is frozen, the link can be modified from the CMDB item that is not frozen.

  1. Display the <Entity> Details window, if it is not already on screen by searching for the CMDB Item and then selecting it from the browse table.
  2. Select . The Freeze/Unfreeze dialog box appears .
  3. Select a Freeze Status for the CMDB item.
  4. To freeze CMDB items linked to the current item:
  • In the Propagate Freeze/Unfreeze browse table, select the Link Type you wish to propagate the freeze to.
  • In the Levels column, click on the current level to see a drop down arrow.
  • Click on the down arrow to see a list of available levels .
  • Select a Level (1, 2, 3, or Unlimited) to propagate the freeze to the linked item, as well as items linked to that item. If you select 1, the items listed under the Linked Items tab (matching the particular Link Type, Role, and CMDB Type) are also frozen. If you select 2, the freeze is also applied to the items listed under the Linked Items tab for the linked items (and so on).

    • If a frozen item is encountered when a freeze is being propagated, the propagation stops at the frozen item and the freeze will not apply to items beyond the encountered frozen item irrespective of the level that has been selected.

  • Select OK to apply the freeze.
  • The details are redisplayed, with a banner across the top declaring that the item is frozen. The fields within the Details window are grayed out and disabled and the button is replaced with a button, allowing you to unfreeze the item.

    • Unfreezing a CMDB item

    Unfreezing a frozen CMDB item releases the lock on the item. This means you can edit the item.

    Before you start

    You must have CMDB Item Unfreeze selected in the CMDB Items tab of your Configuration Management security role.

    At least one freeze status must have been defined, with the Frozen option deselected.

    1. Display the <Entity> Details window, if it is not already on screen by searching for the CMDB Item and then selecting it from the browse table. The banner at the top of the Details window will state that the item is frozen.
    2. Select . The Freeze/Unfreeze dialog box appears.
    3. From the Freeze Status list, select an option. This is the unfreeze status that will be allocated to the CMDB item.

      4. Only statuses with Frozen deselected in the Freeze Status window are displayed.

    4. In the Propagate Freeze/Unfreeze browse table, select the Link Type you wish to unfreeze from the browse table.
    5. Select Propagate and select the number of levels you want to unfreeze to in the Levels column. All CMDB items matching the selected link details will be unfrozen.
    6. Select OK to unfreeze the item.

    Automating Freezing and Unfreezing

    You can make one or many CMDB items frozen as part of a workflow using the Manage CMDB task. Use a Transaction Type of Update Item to set the Freeze Status attribute for the Linked CIs on the request to a value that has the Frozen flag selected.

    If you use the Request Scheduling feature, you can use the above method to automatically make items frozen at a planned point in the future. The same techniques can be used to automate the unfreezing of items.
