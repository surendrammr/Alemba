Creating Forms

Creating a Service Desk Form

Before you start

To create a service desk form using ASM Designer, you must have Designer selected in your General Access Security Role.

To maintain an audit trail of changes made to screens, forms and message templates, Designer must be selected in the Auditing window.

  1. Display the ASM Designer window if it is not already visible on screen.

    Select ≡ > Admin > Designer.

    This window consists of three tabs that display different views:
    Find a Screen Can be used to search for and work with specific screens, forms or message templates, as well as create new in-house screens
    View screens Displays details of currently selected screens and is used to modify screens
    Skins Enables you to create and modify skins for your Self Service Portal(s)
  2. Select a screen type from the list in the left pane to populate the browse table and activate the buttons at the bottom of the window.
  3. Select Create. The Screen Details window appears.
  4. Complete the details:

    5. Create Select Form from the drop-down list
    Name Type a name for the new form.
    Screen Set Select a screen set to which the new form must be associated. You can select from any screen set, since there can be multiple forms per screen set. For entity types for which it is not possible to create multiple screen sets (such as Organization or Person) this list only displays the default screen set for the entity.
     

    If you have selected Knowledge as the entity, this list displays the visible Knowledge Base types. When you select CMDB Item as the entity, this list displays the CMDB Item sub-types. If you select Contract as the entity, the drop down list displays the contract types.
    Copy From Select an existing form on which to base your new one. You can select any active form for the selected entity.

    If you are creating a new Service Desk Form for the Person entity, one of the entries in the Copy From field will be “Person Default Form”.

    If you have deleted all forms in a screen set, this field may not display any values. However, you can restore any deleted forms.

  5. Select OK to create the new form.

    6. If the system detects that there is a deleted screen, form, or message template for the name and screen set you selected, a warning appears. Select Yes to permanently remove the deleted screen from the database and create the new screen. Select No to close the window without creating the new screen and restore a deleted screen.

  6. Select Save at the bottom of the window to save changes to the current screen or Save As to save the content as an HTML file. The file can then be edited with any HTML editor. However, you should be aware that any instability as a result of modifications made outside the ASM Designer is not covered by your Alemba® support and maintenance agreement. When you export a screen, form or message template, images embedded on to the template are not exported. When you load an exported screen, form or message template into ASM Core, you will need to re-add any images. Save All saves changes to all open screens. If auditing is turned on, the Change Reasons window will display, prompting you to add a change reason.

  7. An entry for the new form is created in the browse table.
