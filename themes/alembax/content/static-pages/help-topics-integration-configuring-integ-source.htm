Configuring an Integration Source

Sources enables us to define and configure instances of external sources with which ASM can connect in order to import or update any object, manage alert or exchange information. In order to achieve this, we need to define the source.

To import resources from a source for Federated CMDB integration, or handle events for Event Management, you must configure the source.

Before you start

You must have Integration Setup selected in the Admin tab of your General Access Security Role.

 

  1. Select ≡ > Admin > Integration.

    The Integration Platform Settings window appears.

  2. Select the Sources option from the explorer pane. Add a new source, or select one of the existing sources.
  3. The Integration Source Details window allows you to create an instance of a source for a particular connector. It displays the details for the selected source when integrated with the selected connector. Some details are common while others are particular to the connector.
  4. Complete the details.
    5. Name Type a name to identify the source
    Status Select a status for the source in the drop-down list. This can be Active or Inactive, depending on whether the source is currently in use:

    Select Active to activate the source and perform any integration tasks on the source, such as a scheduled scan.

    Select Inactive to disable any tasks for the source.
    Connector This field is automatically populated based on the selected connector, as defined in the Connector DLL, and is read-only.
    Assembly.TypeName This field is automatically populated based on the selected connector, as defined in the Connector DLL, and is read-only.
  5. Specify the Connection Parameters and any other parameters for the specific connector. These fields are dependent on the source and connector, that is, only the fields which apply to the source appear, and they may be named differently for different sources. For more information, select the type of integration connector whose source details you are defining:
    • Federated CMDB (Directory Server)
    • Federated CMDB (Network Discovery Tool)
    • Event Management
    • Inbound and Outbound Actions
  6. Connector-specific fields are described in the Connector Installation documentation, as well as in the technical notes for each connector.
  7. Select to save the changes.
  8. Select from the toolbar to verify that ASM Core can connect to this source. A dialog box appears informing you of the success or failure of this test.

Network Discovery Tool Parameters

Besides the common fields for configuring an integration source, some fields for a network discovery tool may appear for the specific third party connector. These fields are dependent on the source and will be named differently for different sources.

Connection String

This is the database connector string.

Data Source=GLO2K3SYSQL2005;User ID=DEV_USER;Password=*****;Initial Catalog=SMS_MOD;Persist Security Info=True;Application Name=ASM.

Login ID and Password

These are the user ID and password that will be used to log onto the source database
