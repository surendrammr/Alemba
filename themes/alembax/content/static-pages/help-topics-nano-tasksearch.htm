Searching for Tasks

This topic relates to the Nano interface.

You can search for tasks and/or approvals in the system using any of these methods:

  • Typing the task or approval number into the Quick Search.
  • Entering search criteria into the Task Search window.

On the Task Search screen you can:

  • Use the Standard Search to find open or closed tasks by task number or text search.
  • Use the Advanced Search to find tasks using all task search fields.
  • View the results in table layout or Kanban layout.
  • Configure the columns by adding or removing columns and changing their order.
  • Right-click on a task to perform actions such as Add Note, Action, Review, Email, etc.

Using Quick Search

Before you start

To view tasks, Workflow Analyst must be enabled in the Shared tab of your Workflow Management Security Role.

If you know the number of the item you want to view, use the Quick Search field

  1. Type the number into the Quick Search field.
  2. Calls, requests, tasks, approvals matching that number are displayed below the field.
  3. Select the record you want to view.
  4. The record opens in a new tab in Nano.

Using the Task Search Window

Before you start

To view tasks, Workflow Analyst must be enabled in the Shared tab of your Workflow Management Security Role.

The following settings must be enabled in the Tasks tab of your Workflow Management Security Role to perform the listed actions:

  • To forward tasks from the Search or Outstanding windows, enable Task Bulk Forward.
  • To close tasks from the Search or Outstanding windows, enable Task Bulk Close.

  1. On the menu, select Search > Task Search.

    The Task Search window opens with the Standard Search criteria fields displayed by default.

    2. More advanced search options are available in the Advanced Search.

  2. Enter criteria for your search in the Standard Search fields:
Task Number

Use this field if you know the number of the task you're looking for. Enter the number into the field, then leave the field (press Enter or Tab) to trigger the search. An exact number match is performed across open tasks and closed tasks. If a task with that number exists, and if you have the permissions to view, it will appear in the results table. This search option automatically searches closed tasks and open tasks.

Other search criteria are disabled when this option is used.
Text Search

Use this field to perform a text search. Enter the word(s) into the field, then leave the field (press Enter or Tab) to trigger the search. Returns tasks in which the Description or History contain all the words specified, not necessarily in the same order. This will not retrieve text that has another form of the word or a synonym.

Approvals Select to include approval tasks in the search
Active Tasks Select to include active tasks in the search.
Completed Tasks Select to include completed tasks in the search.
Inactive Tasks Select to include tasks that have not yet been activated in the request workflow.

Advanced Search

Advanced search criteria options are available if the Standard Search does not return the records you are looking for.

  1. Switch to Advanced Search by selecting from the toolbar. The Advanced Search panel opens on the right of the search window.
  2. To minimize the Advanced Search window and retain the search criteria, press . To expand it again press on the toolbar.
  3. To close the Advanced Search window, clear the criteria, and return to the Standard search, select on the toolbar.
  4. Enter the criteria for your search and then press .

Advanced Search criteria (default) options:

Open Searches open tasks (even if they are the responsibility of other analysts)
Closed Searches closed tasks
Approvals Searches approval tasks
Tasks Searches tasks
Active Searches active tasks
Inactive Searches inactive tasks
Authorized Searches tasks in requests that have been authorized by the Request Manager.
Not Authorized Searches tasks in requests that are pending authorization
Template Includes tasks in workflow templates in the search
Partition

This option is only visible if you have access to more than one partition.

Include Deleted Field Values

Selecting this option enables deleted values for list fields to be selected.

If you are searching for a task by Priority and select the Include Deleted Field Values option; the Priority list will be updated to display names of priorities that you created and subsequently deleted. You can then select a deleted value from the list, and conduct your search.

Task Type

From the list, select the Type of the task you are searching for. Depending on the number of tiers configured for your system, you may see one to five tiers. You can select a type from any level.

  • To search for tasks that match only the selected type, select the Equal option.
  • To search for tasks that match the selected type and the tiers below, select Equal + Below.
Priority From the list, select the Priority of the tasks for your search.
Task Status From the list, select the Status of the tasks for your search.
Current Analyst Type part or all of the name of the analyst who currently has the task. Matching results are displayed beneath the field. Select one from the list.
Receiving Analyst

Type part or all of the name of the analyst the task has been forwarded to. Matching results are displayed beneath the field. Select one from the list.
Current Group

Type part or all of the name of the current group of the task. Matching results are displayed beneath the field. Select one from the list.

If a task is forwarded by Group A to Group B, and Group B has not yet taken over the task, the Current Group of the task is Group A. The Current Group changes to Group B after Group B takes over the task.
Assigned Group

Type part or all of the name of the group responsible for the task. Matching results are displayed beneath the field. Select one from the list.

If a task is forwarded by Group A to Group B, the Assigned Group of the task is Group B. (Group A was the Assigned Group on the task before it was forwarded it to Group B, at which point it became Group B.)
Receiving Group

Type part or all of the name of the group the task has been forwarded to. Matching results are displayed beneath the field. Select one from the list.

If a task is forwarded by Group A to Group B, and Group B has not yet actioned the task, the Receiving Group of the task is Group B. After Group B takes over the task, there is no Receiving Group on the task.
