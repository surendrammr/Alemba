Maintaining the ASM password policy

You must ensure that the user account passwords in the User Management Server database follow the same password policy as Alemba Service Manager™.

All Alemba Service Manager™ passwords are six characters in length.

Alemba Service Manager™ cannot synchronize existing users with passwords containing less than six characters to the User Management Server.

To set the password policy for the Dashboard users:

  1. Using User Management Server, in the left-hand menu bar, select to view the Settings.

  2. On the Accounts tab, expand the Password Policy section, then configure the following:

    Parameter Required setting
    Upper Case Turn off by sliding the toggle to the left
    Lower Case Turn off
    Number Turn off
    Special Characters Turn off
    Minimum Length Leave as six in keeping with the ASM password policy
    Does not contain part of name, user name, or email address Turn off
  3. Select Save to save your changes.
