Object Explorer Option

You can attach objects or files from Windows applications to the following entities in ASM Core:

  • Call
  • Request
  • Task
  • Bulletin
  • Knowledge
  • CMDB item
  • Person
  • Organization
  • Contract
  • Script

The Attach Object to Task window has two additional options: Task Objects and Request Objects. Selecting Task Objects displays all the objects attached to the current task. Selecting Request Objects displays all the objects attached to the request. You can link a request object to the current task using the toolbar button.

Attaching an Object

Before you start

To attach an object to a CMDB item, CMDB Item Object Update must be selected in the CMDB Items tab of your Configuration Management Security Role.

To attach an object to a person record, Person Object Update must be selected in the People tab of your Configuration Management Security Role.

To attach an object to an organization, Organization Object Update must be selected in the Organization tab of your Configuration Management Security Role.

To attach an object to a location, Location Object Update must be selected in the Locations tab of your Configuration Management Security Role.

To attach an object to a contract, Contract Object Update must be selected in the Contracts tab of your Configuration Management Security Role.

To attach an object to a subscriber group, Subscriber Group Object Update must be selected in the Subscriber Groups tab of your Configuration Management Security Role.

To attach an object to an agreement, Agreement Object Update must be selected in the Options tab of your SLM Security Role.

To attach an object to a knowledge entry, Object Update must be selected in the Options tab of your Knowledge Management Security Role.

To attach an object to a call, Object Update must be selected in the Options tab of your IPK Management Security Role.

To attach an object to a change calendar, Change Calendar Object Update must be selected in the Shared tab of your Workflow Management Security Role.

To attach an object to a task, Task Object Update must be selected in the Tasks tab of your Workflow Management Security Role.

To attach an object to a request, Request Object Update must be selected in the Requests tab of your Workflow Management Security Role.

To attach an object to a bulletin, Object Update must be selected in your Bulletin Board Security Role.

To enable version control on an attached object on calls, Call Object Version Control must be selected in the IPK Settings of your IPK Management system administration settings.

To enable version control on an attached object on requests, Request Object Version Control must be selected in the Workflow Management Settings of your Workflow Management system administration settings.

  1. Display the <Entity> Details window, if it is not already on screen.
  2. Select if necessary to view the details in edit mode.
  3. Select the Object explorer option. This normally appears within the Data group. Scroll up to locate the group, and expand it if necessary.
  4. The Attach Object to <Entity> window appears. From here, you can do any of the following, using the buttons on the toolbar:

    5. To return to the <Entity> Details window
    To upload a file
    To download a file

    To delete an attachment

    You must have Object Delete selected in the relevant entity's security role to access this option.

  5. Select the button.

    The Open dialog box appears.

  6. Navigate to the folder containing the file, then select it.

  7. Select Open.

    The Upload Object window appears.

  8. Complete the details as follows.

    9. Title

    		 The name of the file you are attaching.

    This is completed with the file you selected in the previous window. Overwrite with another name if you wish it to be seen under a different title by people viewing the entity.
    Version The version number of the file. This can also be overwritten if necessary.
    Description Provide an appropriate description of the object you are adding to the entity in the Description field. Depending on your system settings, this field may be plain text or allow html formatting.

    The description will be displayed in the Attach Object to <Entity> window if Object Version Control for an entity is enabled in the CMDB Settings, , Configuring the Bulletin Board or Configuring IPK Settings, depending on the details screen you are updating. If Object Version Control is not enabled, you will be able to specify a description but the description will not be displayed.

    Visible in Self Service Portal Select to make the description of the object viewable to users via the Self Service Portal (the object itself will always be visible on the Portal).

  9. Select OK to upload the file. The details are displayed in the Attach Object to <Entity> window .

    Data and Object in the Explorer pane change color, although you may not be able to see this while Object is still selected. The file appears in the Upload/Download browse table.

  10. Once the object has been uploaded, analysts with the appropriate permissions can do any of the following:

    11. Check Out

    To check the object out
    Check Out Exec To check out and execute the file
    Check In To check the file back in
    Cancel Check Out

    To cancel the check out

    You must have Object Version Control selected in the CMDB Settings, Configuring the Bulletin Board or Configuring IPK Settings, depending on the entity.
    View History To view the history of changes to the uploaded object
  11. Select from the toolbar or Return To Main at the top of the Explorer pane to return to the Details window.

For more information about checking in and checking out files, see Updating Attached Objects.

  1. Display the <Entity> Details window, if it is not already on screen.
  2. Select if necessary to view the details in edit mode.
  3. Select the Object explorer option. This normally appears within the Data group. Scroll up to locate the group, and expand it if necessary.

  5. If a Windows Event Viewer file (*.EVT) is uploaded, you can view it by selecting it and then double-clicking on it. The file will be displayed in the Windows Event Viewer.

To enable version control on an attached object on calls, Call Object Version Control must be selected in the IPK Settings of your IPK Management system administration settings.

To enable version control on an attached object on requests, Request Object Version Control must be selected in the Workflow Management Settings of your Workflow Management system administration settings.

When a linked object is updated outside ASM Core, the button will be highlighted in red. Click Update to update the existing object with the changed version. To update an object within ASM Core, take the following steps.

  1. Display the <Entity> Details window, if it is not already on screen.
  2. Select if necessary to view the details in edit mode.
  3. Select the Object explorer option. This normally appears within the Data group.
  4. The attached objects are displayed in the Upload/Download table. Select the object you want to update, and then select Check Out.

  5. Complete the details in the Upload Object window, and then select OK.

    The object is downloaded.

  6. Make the necessary changes in the relevant application, then save it.
  7. Return to the Attach Object window in ASM Core, and select Check In.

  8. Add a Check In Comment in the Uploading Object window if necessary, and select OK.

  9. In the Open dialog box, select the file, then select Open to check in the updated object.

Cancelling a Checkout

There may be occasions when an object attached to a call, task, request or item has been checked out for a long time, or needs to be recalled. In such cases, you can cancel the checkout and restore the object to its last checked-in state.

Before you start

To cancel the checkout of an object attached to a CMDB item, CMDB Item Object Cancel must be selected in the CMDB Items tab of your Configuration Management Security Role.

To cancel the checkout of an object attached to a person record, Person Object Cancel must be selected in the People tab of your Configuration Management Security Role.

To cancel the checkout of an object attached to an organization, Organization Object Cancel must be selected in the Organization tab of your Configuration Management Security Role.

To cancel the checkout of an object attached to a location, Location Object Cancel must be selected in the Locations tab of your Configuration Management Security Role.

To cancel the checkout of an object attached to a contract, Contract Object Cancel must be selected in the Contracts tab of your Configuration Management Security Role.

To cancel the checkout of an object attached to a subscriber group, Subscriber Group Object Cancel must be selected in the Subscriber Groups tab of your Configuration Management Security Role.

To cancel the checkout of an object attached to an agreement, Agreement Object Cancel must be selected in the Options tab of your SLM Security Role.

To cancel the checkout of an object attached to a knowledge entry, Object Cancel must be selected in the Options tab of your Knowledge Management Security Role.

To cancel the checkout of an object attached to a call, Object Cancel must be selected in the Options tab of your IPK Management Security Role.

To cancel the checkout of an object attached to a change calendar, Change Calendar Object Cancel must be selected in the Shared tab of your Workflow Management Security Role.

To cancel the checkout of an object attached to a task, Task Object Cancel must be selected in the Tasks tab of your Workflow Management Security Role.

To cancel the checkout of an object attached to a request, Request Object Cancel must be selected in the Requests tab of your Workflow Management Security Role.

To cancel the checkout of an object attached to a bulletin, Object Cancel must be selected in your Bulletin Board Security Role.

To enable version control on an attached object on calls, Call Object Version Control must be selected in the IPK Settings of your IPK Management system administration settings.

To enable version control on an attached object on requests, Request Object Version Control must be selected in the Workflow Management Settings of your Workflow Management system administration settings.

  1. Display the <Entity> Details window, if it is not already on screen.
  2. Select if necessary to view the details in edit mode.
  3. Select the Object explorer option. This normally appears within the Data group. Scroll up to locate the group, and expand it if necessary.
  4. The attached objects are displayed in the Upload/Download table. You can see the name of the person who checked it out, and the date and time it was checked out.

  5. Select Cancel Check Out beneath the table.

    The Upload Object box appears.

  6. Give details of the reason for canceling the checkout, then select OK.

    The previous version of the object is restored in the Upload/Download table.

Object Explorer Option Toolbar Buttons

The following toolbar buttons are displayed on the Attach Object to <entity> windows:

Select To...

Attach an existing object (such as a word processing document, a text document, or an image file).

Download

Permanently delete an uploaded object and all its revisions.

This is only displayed on Task Details windows.

Link a request object to the current task. When you attach an object to a task, it is automatically linked to the request. Therefore, objects from multiple tasks are linked to the request. You can link an object attached to a different task or attached to the request to the current task.

This is only displayed on Task Details windows.

Unlink a request object that is linked to the current task.
