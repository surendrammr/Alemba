Configuring Windows Server 2008 for Alemba Service Manager™

Before you start

Ensure that your server and client machines meet the prerequisites. For more information, see Prerequisites checklist.

Adding ASP.NET as a role service to Windows Server 2008

For information about adding ASP.NET to a Windows Server 2012 machine, see Configuring Windows Server 2012 R2 for Alemba Service Manager™.

To add ASP.NET as a role service to your Windows Server 2008 machine:

  1. Using Server Manager, in the tree panel, expand Roles, then select Web Server (IIS) to view the Web Server (IIS) pane.

  2. Select Add Role Services.

    The Add Role Services window appears.

  3. In the Role Services pane, under Application Development, select ASP.NET.

    If the required role services for ASP.NET are not installed, the wizard prompts you to install them. Select Add Required Role Services.

  4. Select Next, then select Install to add the role service to your server.
  5. After installation completes, select Close to close the Add Role Services wizard.
