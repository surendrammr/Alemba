Clone to Knowledge Explorer Option

Analysts with the appropriate permissions can publish an existing ASM call automatically to the Knowledge Bank.

You must have Author and Clone from Calls selected in the Options tab of your Knowledge Management security role.

  1. Display the Call Details window, if it is not already on screen.
  2. In the explorer pane, expand Resolution, then select Clone to Knowledge.
  3. In the Clone Call to Knowledge Bank popup window, click at Type to select a knowledge bank article type.
  4. To edit the details of the call before publishing to the Knowledge Bank, select Edit Entry Now.
  5. To copy text from the Description and Actions and Solutions fields in the call to relevant fields in the Knowledge Entry (the specific fields will depend on which Knowledge Type you are creating from the call) select Import Call Text.

    6. For KCS knowledge entries, the last entry on the call will be carried across to its Resolution field.

  6. To copy (clone) any objects attached to this call to the Knowledge Bank entry when it is created, select Clone Call Objects.
  7. Click OK.

The Knowledge Bank entry you created appears with the Linked CMDB Items and Type fields updated from the call. The body of the entry displays the problem, solution, workaround, and the call history. You can edit these details if you selected Edit Entry Now in the Clone Call to Knowledge Bank dialog box.
