Workflow Management History Types

Each request and task event is recorded in the Request or Task History window, accessed by selecting the History explorer option from the Request or Task Details window. You can assign a name to any history type on the system that may be more meaningful for Analysts within your organization.

You cannot create or delete history types, or change the system-defined title.

Renaming a History Type

Before you start

You must have Workflow Management Set Up enabled within your General Access Security Role to work with workflow management history types.

  1. Select â‰¡ > Admin > System Administration. The System Administration window appears. In the Explorer pane, expand Workflow Management.
  2. Select the Workflow Management History Types option. The Workflow Management History Types window appears, displaying a browse table listing the system-defined titles and Admin assigned names for the history types . Scroll down to see the full list. You can adjust the column widths if required.
  3. Click in the Name field of the history type and type the name you wish to assign. This will be displayed in the Request or Task History window.
  4. Select to save the changes. Provide the Change Reasons if prompted to do so.
