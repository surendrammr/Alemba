LANDESK Connector

This section of the documentation contains technical specifications about the connector that is implemented to link ASM Core and LANDESK version 8.x, 9.0, and 9.5 systems. It describes the details of the third-party application, that is, LANDESK, including:

  • The version for which the connector has been developed
  • The name of the .NET assembly file
  • Connection methodology
  • The resource and link types that can be discovered on the application
  • The attributes of each resource and link types that can be imported into the ASM Core Configuration Management Database (CMDB)

You should familiarize yourself with the information in Installing Connectors before installing any connectors, and read the Integration topics for more information on how to configure them.

Use Case Scenario

Purpose

An organization uses LANDESK version 8.x, 9.0, and 9.5 to discover and maintain resources present in its organization networks.

Role

The role of this connector is to expose the resources and their relationships in order to allow for population and ongoing consistency checks of the ASM Core CMDB.

Customization

This connector permits extensions to the schema through customization of the configuration file.

The possible customizations are listed below. However, the customizations are not limited to these:

  • New resource types
  • New resource properties
  • New link types

Connector Diagnostics

The connector has the facility to trace information. The data can be obtained through Polling tracing or Application tracing.

Resource Types

This section lists the resource types and their properties that the connector has to discover on LANDESK 8.x, 9.0, and 9.5 systems. The connector later imports the resource types it has discovered into the ASM Core CMDB.

Resource types for LANDESK 8.5

This section lists the resource types and their properties that the connector has to discover on LANDESK v8.5 system. The connector later imports the resource types it has discovered into the ASM Core CMDB.

(Unmanaged) category

The resource types that belong to the (Unmanaged) category are:

  • Unmanaged
  • Unmanaged Computers
  • Unmanaged Infrastructure
  • Unmanaged Other
  • Unmanaged Printers

All resource types above will have the following properties:

  • ID
  • Name
  • IP Address
  • Subnet Mask
  • Physical Address
  • OS
  • Work Group
  • First Scanned
  • Last scanned
  • Device Group
  • Device Group Comment

Device

  • ID
  • Type
  • Device Name
  • Display Name
  • Description
  • NT Domain Controller
  • Last Login ID
  • Last Login Name
  • Primary owner
  • Domain
  • Work Group
  • Manufacturer
  • Model
  • RAM Cache
  • Netware Address
  • Directory Address
  • Owner Phone
  • Owner Email
  • Company
  • Server Type
  • Asset Tag
  • Service Tag
  • Serial No
  • Chassis Type
  • System Version
  • Has Battery
  • IP Address

  • Host
  • Default Gateway Address
