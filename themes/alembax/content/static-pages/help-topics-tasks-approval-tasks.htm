Approval Tasks

You may want to create approvals at various stages of the request and create dependencies such that certain tasks are not released until each approval is approved by the request manager.

Analysts with Approver selected in the Tasks tab of their Workflow Management security role can approve or reject Approval Tasks. If an analyst does not have approval rights, they will be able to view and take action on Approval Tasks but they will not be able to complete them.

Approval tasks are automatically scheduled to run when released in a request workflow.

If you want the user to complete the approval, you should create a user approval task.

Creating an Approval Task

  1. Search for the Request Details window, if it is not already on screen.
  2. Select the Update Dependencies explorer option on the Request Details window, or the button from the toolbar. The Task Dependencies window appears with icons for task entity types displayed below the toolbar.

    3. If the workflow template you have used to create the request already has tasks defined in it, you will see these tasks here. You can open them and change their details such as dates, assignment, or expenses. You can add new tasks or delete them from the current request without affecting the template.

  3. Choose the task icon from the palette, and drag it to the window. Alternatively, right-click in the window, and from the menu that appears, select New and then the Approval task entity type.
  4. On the dependency diagram, double-click the Approval task icon to view the Approval Details window.
  5. If needed, take action on the approval in order to edit the details.

    6. The Task Type field displays values that have been defined for approval types on the Approval Types window.

  6. Complete the common task fields on this task.
  7. Complete the Resource Management details, if appropriate. See Approval Tasks for more details.
  8. After you have completed the fields, save or defer the task. If you are creating the task as part of a workflow template, assign it to another analyst or manager and initiate the appropriate action necessary for completion of the approval.

    9. Approvals cannot be approved or rejected when the associated request is being actioned by another analyst and you have modified fields in the request.
