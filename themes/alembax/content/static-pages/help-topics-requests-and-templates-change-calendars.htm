Change Calendars

You can use ASM Core workflow management to define and manage Change Calendars. Change Calendars enable you to specify when changes to a CMDB item can be scheduled by way of a request.

Change Calendars, in essence, consist of Change Windows – these are units of time used to specify when a CMDB item is unavailable for the system, in order to apply a change as part of a request. A Change Calendar can consist of multiple Change Windows.

With the appropriate security permissions as part of your Workflow Management security role, you will be able to:

  • create Change Calendars and the Change Windows it encompasses
  • perform searches for Change Calendars
  • view, edit and delete Change Calendars
  • allocate Change Windows to CMDB items linked to a request

Creating a Change Calendar

Before you start

You must have New Change Calendar selected in the Shared tab of your Workflow Management security role.

  1. Select ≡ > Admin > Change Calendar.

    The Change Calendar Details window appears.

  2. Select Add New at the bottom of the left panel to display the input window.
  3. Complete the details.

    4. Title Type a name for the Change Calendar
    Description Type any relevant information about the Change Calendar, the Change Windows within it, and the CMDB items that will be linked

  4. On the Change Windows tab, select Add New.

    A new row is added to the Change Windows List, with the Start Date and End Date columns automatically set to display the current date and time. Complete the details.

    5. Change Window ID Type a unique name for the Change Window
    Start Date Specify the required start date and time
    End Date Specify the required end date and time
    Portal Access Select to make this Change Window visible to users of the Self Service Portal
  5. Click OK to save the details of the Change Calendar.

Viewing Change Calendars

Within ASM Core workflow management, you can not only create but also search for, view, modify and delete Change Calendars.

Before you start

You must have View Change Calendar selected in the Shared tab of your Workflow Management security role.

  1. Select ≡ > Admin > Change Calendar.

    The Change Calendar Details window appears.

  2. Specify the search criteria. To add more, select at the top of the search pane.
  3. Select Search. All Change Calendars matching your search criteria are displayed in the browse table on the right.
  4. To view the details of a Change Calendar, double-click its name in the browse table.

Updating a Change Calendar

Before you start

You must have Update Change Calendar selected in the Shared tab of your Workflow Management security role. To override any lock that an analyst may have on a change calendar, you must also have Cancel Change Calendar Lock selected in the same tab. If you wish to work with the objects attached to the change calendar, ensure that you have the necessary permissions in the same tab for these also.

  1. Display the change calendar.
  2. Select the button on the toolbar.
  3. Modify the details as appropriate.
  4. Select OK to save your changes and close the Change Calendar Details window.

Deleting a Change Calendar

Before you start

You must have Delete Change Calendar selected in the Shared tab of your Workflow Management security role.

  1. Select ≡ > Admin > Change Calendar.

    The Change Calendar Details window appears.

  2. Specify the search criteria. To add more, click at the top of the search pane.
  3. Click Search. All Change Calendars matching your search criteria are displayed in the browse table on the right.
  4. Select the Change Calendar you want to delete.
  5. Click on the toolbar. A prompt appears, and confirm the deletion.

Allocating Change Windows to CMDB Items

Change Windows are time periods during which changes to CMDB items linked to a request can be scheduled. Each Change Window is part of a Change Calendar that is defined when the Change Calendar is created.

To link CMDB items to a Change Window, use the Change Windows Explorer option on the Request Details window.

Change Calendar Explorer Options

You can carry out further tasks on the change calendar using the explorer options.

Object explorer option

An analyst can use the Object explorer option on the Change Calendar Details window to attach objects or files from Windows-based applications to the current Change Calendar.

Before you start

You must have Change Calendar Object View selected in the Shared tab of your Workflow Management security role to view objects attached to the change calendar. You must have Change Calendar Object Update to change the details; Change Calendar Object Cancel to cancel a lock added by another analyst; and Change Calendar Object Delete to delete objects attached to a change calendar. These all appear in the same tab.

To attach a file:

  1. Display the Change Calendar Details window.
  2. Select the Object explorer option. The Attach Object to Change Calendar window appears.
  3. Click .
  4. Select the object you want to attach to the Change Calendar.
  5. Click Open.
  6. Add a title and/or a description for the object in the Upload Object window.
  7. Click OK.

View Linked Requests Explorer Option

The View Linked Requests Explorer option enables you to view all requests that are linked to the current Change Calendar.

  1. Display the Change Calendar Details window.
  2. Select the View Linked Requests explorer option. The Linked Requests window appears.

Audit Explorer Option

The Audit Explorer option enables you to view an audit trail of the changes made to the current Change Calendar.

  1. Display the Change Calendar Details window.
  2. Select the Audit explorer option. The Calendar Audit Trail window appears, which displays the details of the changes made to the Change Calendar in a browse table.
