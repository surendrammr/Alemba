Forms Explorer Option

Forms are specially designed windows attached to specific entities in ASM Core (such as calls, requests, tasks, or CMDB items). They enable Analysts to capture general, non-reportable, non-searchable information about the entity. Forms can be created in the ASM Designer, for example to include combo boxes, drop-downs, options. ASM Core provides a default form for each entity.

The Forms explorer option can be accessed on various ASM Core entity Details windows, as well as the Call, Request, Task and Agreement windows.

Before you start

Your system administrator must have selected Forms in the System Settings.

If the form relates to calls, you must have access to it in the Forms tab of your IPK Management Security Role. If you have Read Only selected in the Access column, you can read the information on the form, Read/Write allows you to open and edit forms of the selected type.

If the form relates to a CMDB item such as a Person, You must have access to the relevant form in the Forms tab of your Configuration Management Security Role, depending on the nature of the form. If you have Read Only selected in the Access column for the form, you can read the information on the form, Read/Write allows you to open and edit forms of the selected type.

Completing the Form

  1. From the Details window, select the Forms explorer option. This normally appears within the Data group. Scroll up to locate the group, and expand it if necessary.
  2. If there are forms available for the entity, the option will appear in orange text with a small arrow, indicating that there are forms within the option .
  3. Select the arrow to expand the group and display the forms within it.
  4. Select the form you wish to work with.
  5. The <Entity> Form window appears.
  6. Complete all necessary details. You can copy and paste text using the and buttons.
  7. Select from the toolbar or Return To Main at the top of the Explorer pane to return to the Details window.
