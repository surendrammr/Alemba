Configuring the Alemba API connection

Each dashboard you create connects to data via a data source. The connection requires the Alemba API Client ID. You must also configure the Alemba API client to contain an Allowed Redirect Uri.

To find the Alemba API Client ID and configure the Allowed Redirect Uri:

  1. Log into Alemba API Explorer.

    By default, the Alemba API Explorer is here: http://ServerName/VirtualDirectory/alemba.web/alemba/admin, where ServerName is the server name or IP Address. For more information, see Logging In to the Alemba API Explorer.

  2. In the left-hand menu, select API Clients.

    A list of API Clients appears.

  3. Select Dashboards.

  4. In the Allowed Redirect Uri field, type the following:

    http://ServerName/DashboardServerName/webdesignerservice/v1.0/oauth/agent, where ServerName is the server name or IP Address, and DashboardServerName is the name you gave the Dashboard Server.

    For example: http://localhost/AlembaDashboardServer/webdesignerservice/v1.0/oauth/agent

  5. Make a note of the Client ID.

    When you create an Alemba dashboard data source, you connect to the Alemba Account using the Alemba API Client ID. For more information, see Creating a new data source.

  6. Select OK to save your changes.
