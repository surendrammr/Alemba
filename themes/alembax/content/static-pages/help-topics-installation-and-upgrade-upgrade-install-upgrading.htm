Upgrading your system

You can upgrade your system using the Alemba Service Manager™ Patch Tool. We recommend that you read this section fully before applying a patch.

Prior to upgrading a live Production system, we recommend you replicate the Production system to a lower environment and perform the upgrade on the replicated system. For more information, contact Alemba Support.

Before you start

The Patch Tool will not create a backup. Before upgrading, create a backup of your databases, including your system files and registry settings. Ideally, create a snapshot or backup of the virtual web server, enabling a swift and simple rollback should the need arise.

Close the ASM Server Console and ensure that you are not logged into the database before running the upgrade, otherwise an error will occur.

If you are performing an upgrade from a version lower than 9.2.0, it will invalidate your existing license key. You must have an updated license key to run Alemba Service Manager™. Contact Alemba® Support for more details if necessary.

Locating and running upgrade files

From 10.1.0, you can find upgrade files here by default: C:\Program Files\Alemba\ASM\WebServer\Config\Version, where:

  • WebServer is the Web Server you are using
  • Version is the upgrade version number that is used as a folder name.

For example, for 10.2.1: C:\Program Files\Alemba\ASM\System1\Config\100201

Depending on which version you selected, the upgrade script files are either SCP files, or use the new XML format.

Creating a Backup

For virtual servers

Create a snapshot or backup of the virtual web server and the virtual database server.

For physical servers

  1. Stop the ASM Administrative Service, which will stop all ASM Services.

  2. Stop the ASM application pools and websites.

  3. Create a backup of the system database.
  4. Create a backup of the ASM registry keys (e.g.: export to file HKEY_LOCAL_MACHINE\SOFTWARE\Alemba\ASMCore).
  5. Create a folder in the same location as the extracted zip file.

    6. Create new folder C:\Alemba\944Backup

  6. Back up the files in the ASM directory, as well as the system directory, for example, C:\Program Files\Alemba\ASM\WebServer, where WebServer is your web server.

Rolling Back the Upgrade

You may need to roll back the upgrade and return your system to its previous version.

For virtual servers

Restore the web server and database server to the snapshot or backup you took prior to applying the upgrade.

For physical servers

  1. Stop the ASM Administrative Service.

  2. Stop the ASM application pools and websites.

  3. Restore the system directory and files from the backup location you created prior to upgrading.
  4. Restore the database from the backup.
  5. Restore the registry keys from the backup.
  6. Restart the ASM Services.
  7. Restart the ASM website and application pool.
  8. Restart IIS.

Considerations before upgrading to 9.3 or above

If you used the features and functions listed below prior to ASM 9.3, then you must perform these extra steps after upgrading:

  1. Have the correct licenses to use the Integration Connectors. See Licensing integration connectors.
  2. Reconfigure the Jira connector if upgrading from ASM 9.8 or 9.9 to ASM 9.10 or higher. See Upgrading the Jira connector.
  3. Configure your Reminder columns if upgrading from 9.5 or below. See Configuring Reminder columns on Calls Outstanding.
  4. Reapply search fields on Self Service Portal screens if upgrading from 9.2 or below. See Redesigning search fields on Self Service Portal screens.
  5. Reconfigure personalized column selections on search windows if upgrading from 9.2 or below. See Reconfiguring personalized Column configurations.
  6. If using a SAM connector, ensure you have the correct Java installation, and your environment is configured correctly. For more information, see Upgrading your system.

Licensing integration connectors

Your existing integration settings will continue to function after upgrading to v10.1, with no additional steps required.

If you want to add a licensed connector to your integration platform, contact your Sales Manager to purchase a license key.

Upgrading the Jira connector

The Jira connector was introduced in 9.8 and updated in 9.10.0 to support a single integration source for multiple projects. If you are upgrading from 9.8 or 9.9, to 9.10 or above, you need to remove the Jira Integration Source(s) and recreate them using the new connector. You also need to update any tasks previously set up to use Jira outbound actions, and reconfigure them. Refer to the 9.10 Release Notes for information about setting up and using the Jira connector.

Configuring Reminder columns on Calls Outstanding

If you are using Reminder columns and are upgrading from 9.5 or below, to 9.6 or above, remove and re-add them to your column configuration after upgrading.

Redesigning search fields on Self Service Portal screens

If you are upgrading from 9.2 or below, the following fields may stop working on Self Service submissions screens:

  • Multi-value search fields that reference External Suppliers
  • Search fields that reference SLA, OLA, and UC Agreements

After the upgrade, use ASM Designer to remove these fields and reapply them to the screen.

Reconfiguring personalized Column configurations

If you are upgrading from 9.2 or below, to 9.3 or above, you must reconfigure personalized column selections on the following search windows:

  • Organizations
  • Locations
  • Subscriber Groups
  • Contracts
  • Cost Centers
  • Jurisdictions
  • Knowledge Admin
  • Agreements
  • Bulletin Administration

From 9.3.0, column selections on these search windows are reset to default as a result of an enhancement, which extends the Configurable Column and Advanced Search functionality to these searches.
