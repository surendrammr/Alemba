Viewing Outstanding Tasks

The Tasks Outstanding window displays all the tasks that you have been assigned. This window does not display closed tasks. There are a number of filtering options available on the Tasks Outstanding window that enable you to view your tasks as well as other analysts’ tasks, escalated tasks, and suspended tasks.

  1. Access the Tasks Outstanding window in one of the following ways:
  • From the Work menu, select Tasks Outstanding.
  • From the View menu, select Tasks Outstanding.

By default, the Tasks Outstanding window displays all of your outstanding tasks. You can view outstanding tasks for other analysts by selecting their name.

  • You can view your tasks or another analyst’s outstanding tasks by Workflow Process. Simply select the required process from the Workflow Process list. The window will be refreshed to display open requests for the selected workflow process and analyst. Choose the tasks you want to view.

    Additional filters are displayed for Your Group's Tasks

    Current Group’s Tasks displays the tasks assigned to the group selected in the Group drop-down list or owned by the members of the selected group
    Forward to Group

    This filter displays the tasks that have been forwarded to the group(s) to which you belong but that have not yet been actioned.

    When a task or approval is forwarded to a workflow group, it does not show up on the Tasks Outstanding window of the authorizer or the request manager unless the Forward to Group option is selected. This task or approval appears on the Tasks Outstanding windows of the other members of the group even if the Forward to Group option is not selected. However, the Forward to You option must be selected.
    Forward Internally This filter displays the tasks that you have forwarded internally to an analyst or group but that you still have ownership of
    Escalated Tasks

    Escalated tasks are those tasks that have exceeded the service goals of the organization and thus have been brought to the attention of the appropriate analysts.

    Additional filters are displayed for Escalated Tasks

    Resolution Breach displays the tasks that have not been resolved within the time specified by the agreement(s) applied to the task
    Resolution displays the tasks for which a resolve escalation alert has been activated
    Action Breach displays the tasks which have not been actioned by the proper group or external supplier within the set period
    Action displays the tasks for which an action escalation alert has been activated
    Suspended Tasks

    Suspended tasks have been forwarded to you or your group, or been forwarded internally by you. A task is classified as suspended if an analyst selected the Suspended option when forwarding the request associated with the task. The Status column shows the action you should expect to take on the item.

    Any tasks to which you are assigned that are linked to a suspended request will be displayed in the Suspended Tasks view, regardless of whether the task is currently active, inactive, authorized or unauthorized.
    Delegated Tasks

    You must be nominated as a Delegate to see this filter. If you are nominated as a delegate by more than one analyst, you can select from the drop down list.

    Additional filters are displayed for delegated tasks

    Delegator Tasks displays the tasks that are owned by the delegator
    Forward to Delegator displays the tasks that have been forwarded to the delegator
    Forward Internally by Delegator displays the tasks that have been forwarded internally by the delegator
    Forward to Delegator Group displays the tasks that have been forwarded to groups of which the delegator is a member

    To close one or more of the tasks in the search results, use the Bulk Close function. Select the record(s) you want to close, using the Shift key to select consecutive records, or the Ctrl key to select non-consecutive. Then click on the toolbar.

    Taking action on a task

    Take Over Tasks must be selected in the Tasks tab of your Workflow Management security role to allow you to work on another analyst's tasks.

    Select a task in the Results table and click to action the task and work on it. If you open a task that belongs to you, you can work on it straightaway.

    If you action a task that belongs to another analyst, you can still work on it. If you double-click a task that belongs to another analyst, the task opens in Review mode. You will have to click Take Over to take ownership of the task and work on it.

    Using the Close Selected Task toolbar button to close a task from the Tasks Outstanding view will not validate any mandatory fields set on the Task Details window.
