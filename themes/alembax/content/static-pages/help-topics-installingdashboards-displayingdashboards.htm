Displaying a dashboard in ASM

After you have installed and configured the dashboard platform, you can configure the dashboard menu in ASM. For example, the following dashboard menu appears in ASM Core:

Alemba supply five dashboard placeholders each for the dashboard menus in ASM Core, Nano, and Self Service Portal. These placeholders are labelled Dashboard 1, Dashboard 2, and so on. You must configure each placeholder to display a specific dashboard.

Before you start

Before you configure the Alemba Service Manager™ dashboard menus:

  • Ensure you have correctly installed the dashboard platform. For more information, see Installing the dashboard platform.

  • The dashboard must exist in Dashboard Server. For information about creating dashboards, see the online help.

  • You must have the following privileges to see the dashboard menu in ASM:
    • For ASM Core and Nano, analysts must have the General Access Security Role set to All on their Person Details screen, and at least one IPK Management or Workflow Management Group selected.
    • For Self Service Portal, users must have a Self Service Portal Role selected in the User Details section of the Person Details screen, which enables access to the Self Service Portal.

Finding the dashboard placeholder .HTM files

You configure the placeholders by editing supplied .HTM files, which are installed when you install ASM. You can find them here, by default:

C:\Program Files\Alemba\vFire\System1

ASM Core, Nano, and Self Service Portal each have separate dashboard placeholder .HTM files, labelled 1 through 5. Search for the following in the System1 folder:

  • For ASM Core, search for InDashboard*_Frame
  • For Nano, search for NanoDashboard
  • For Self Service Portal, search for LiteDashboards

The returned results list the five relevant .HTM files. For example, for Self Service Portal: LiteDashboards1, LiteDashboards2, LiteDashboards3, LiteDashboards4, and LiteDashboards5.

Linking an ASM dashboard placeholder to an existing dashboard

To configure an ASM dashboard placeholder to open a specific dashboard:

  1. Search for a product-related dashboard .HTM file.

    For more information, see Finding the dashboard placeholder .HTM files.

  2. Right-click the .HTM file you want to configure, then select Open with > Notepad in the shortcut menu.

    For example, if you want to configure the ASM Core Dashboard 1 menu, open the InDashboardDashboard_1_Frame.htm file.

  3. Using Notepad, search for class="df-dashboard".

  4. Replace data-service-url='http://localhost/DashboardService/DashboardService.svc'

    data-dashboard-path='http://localhost/infra.web/dashboards/MasterFilter.sydx'

    with:

    data-dashboard-path='Category/DashboardName'

    where Category is the category where the dashboard is grouped, and DashboardName is the name of the dashboard in Dashboard Server.

  5. If you are editing Nano or Self Service Portal .HTM files, then adjust the height as follows:

    style="height: 800px;"

    For example:

  6. Save your changes.

  7. Using a command prompt, perform an iisreset to restart IIS.

  8. Clear the internet browser cache.
  9. Log into the Alemba Service Manager™ product where you have configured the dashboard menu.

  10. Select the dashboards placeholder you configured.

    For example, in ASM Core, select ≡ > Dashboards > Dashboards > Dashboard n, where Dashboard n is the same number as the .HTM file you recently edited.

    The dashboard opens.
