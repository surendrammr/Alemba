Server Time Zone

You can set a default time zone for the ASM Core server. All time zone calculations are based on comparing the Analyst’s time zone with this default.

The Admin login is always assumed to be in the server’s time zone. Other Analysts can be assigned a time zone in their Person Details.

Before you start

You must have System Set Up selected in the Admin tab of your General Access security role.

The Time Zones checkbox must be selected in System Settings to enable the Server Time Zone option.

  1. Select ≡ > Admin > System Administration.

    The System Administration window appears.

  2. In the Explorer pane, expand System, then select Server Time Zone. The Server Time Zone window appears, displaying a list of time zones and their corresponding locations.
  3. Select the time zone you want to use as the default server time zone, scrolling down if necessary.
  4. Select to save the changes. Provide the Change Reasons if prompted to do so.
