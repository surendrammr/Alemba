Configuring Single Sign On using SAML for the Alemba API

Alemba API Clients can be configured to support SAML based Authentication.
This authentication type can only be used in conjuction with the Authorization Code grant flow and is not supported by the Implicit Grant flow.

To enable SAML based Authentication, the Single Sign On checkbox must be enabled in the API Client settings page.

Single Sign On must also be configured using ASM Core admin. The list of available User Interfaces now includes each Alemba API client.

For more information, see Single Sign-On using SAML.
