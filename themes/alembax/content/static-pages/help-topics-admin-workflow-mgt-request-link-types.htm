Request Link Types

Analysts can link requests to one another in a relationship using the Linking explorer option. You can define the titles that describe the nature of the link or relationship, as well as link styles, colors, and sizes for Request Link Types.

New or existing calls can also be linked to a request. However, link types are not required for this.

Creating a Request Link Type

Before you start

You must have Workflow Management Set Up enabled within your General Access Security Role to work with request link types.

As some parameters are read from the database at application start-up, you should always make changes to settings when there are no Analysts logged on to the system.

  1. Select ≡ > Admin > System Administration. The System Administration window appears. In the Explorer pane, expand Workflow Management.
  2. Select the Request Link Types option. The Request Link Types window appears, with a browse table listing the existing link types and the line style to be used in the linking diagram. You can adjust the column widths if required.
  3. Select . A blank row is inserted in the browse table.
  4. Type the name of the link type in the Name column.
  5. In the Style column, click in the cell and select the line style to be used in the linking diagram.
  6. In the Color column, click in the cell and select the color to be used for the link. This color is also displayed in the linking diagram.
  7. In the Size column, select a weight for the link style. This is the thickness of the lines in the linking diagram.
  8. Select to save the changes. Provide the Change Reasons if prompted to do so.

Updating a Request Link Type

Before you start

You must have Workflow Management Set Up enabled within your General Access Security Role to work with request link types.

As some parameters are read from the database at application start-up, you should always make changes to settings when there are no Analysts logged on to the system.

  1. Select ≡ > Admin > System Administration. The System Administration window appears. In the Explorer pane, expand Workflow Management.
  2. Select the Request Link Types option. The Request Link Types window appears, with a browse table listing the existing link types and the line style to be used in the linking diagram. You can adjust the column widths if required.
  3. Make the necessary adjustments. You can:
    • Overtype the name of the link type in the Name column.
    • Click in the cell in the Style column and select a new line style to be used in the linking diagram.
    • Click in the cell In the Color column and select a new color to be used for the link. This color is also displayed in the linking diagram.
    • Select a different weight for the link style in the Size column. This is the thickness of the lines in the linking diagram.
    Select to save the changes. Provide the Change Reasons if prompted to do so.

Deleting a Request Link Type

Before you start

You must have Workflow Management Set Up enabled within your General Access Security Role to work with request link types.

You cannot delete the “Unspecified” Request Link Type, which is the default used if no other link is specified.

If you delete a link type, you cannot reverse this procedure and will have to re-create it if needed. Even if you use the same name, existing links using this link type will not be restored.

  1. Select ≡ > Admin > System Administration. The System Administration window appears. In the Explorer pane, expand Workflow Management.
  2. Select the Request Link Types option. The Request Link Types window appears, with a browse table listing the existing link types and the line style to be used in the linking diagram. You can adjust the column widths if required.
  3. Select the link type you want to delete. Select . If you attempt to delete the system default, you will receive a warning message.
  4. Select to save the changes. Provide the Change Reasons if prompted to do so.
