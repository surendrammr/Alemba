General Access Security Roles – Admin Tab

Before you start

You must have Security Roles setup selected in the Admin tab of your own General Access security role before you can assign or remove permissions for any security roles.

  1. Display the General Access Security Role details window, if it is not already on screen.

    Select ≡ > Admin > System Administration.

    The System Administration window appears.

    In the Explorer pane, expand Security Roles, then select Configuration Management. All existing Configuration Management security roles are displayed.

    The pre-defined role All typically has all role options selected. However, some of the options are not selected by default and you should explore the permissions set before assigning the role.

    If you want to create a role, select and assign the role a name in the Role Name field. To view or update a role, select the role in the browse table and select .

  2. On the Admin tab, adjust the following permissions to suit your needs:

    3. System Setup Analysts with this role can configure System Settings. Select to enable the options in the Systemgroup in the System Administration Explorer pane.
    Security Setup Analysts with this role can configure Security Settings. Select to enable options in the Security group in the System Administration Explorer pane
    Security Roles Setup Analysts with this role can create, edit, and delete security roles Select to enable the options in Security Roles in the System Administration Explorer pane.
    IPK Setup Analysts with this role can configure IPK Management settings. Select to enable the options in the IPK Management group in the System Administration Explorer pane.
    Workflow Management Setup Analysts with this role can configure Workflow Management settings. Select to enable the options in the Workflow Management group in the System Administration Explorer pane
    CMDB Setup Analysts with this role can configure CMDB settings. Select to enable the options in the CMDB group in the System Administration Explorer pane
    SLM Setup Analysts with this role can configure Service Level Management settings. Select to enable the options in the Service Level Management group in the System Administration Explorer pane.
    Availability Setup Analysts with this role can configure Availability Management settings. Select to enable the options in the Availability group in the System Administration Explorer pane.
    Bulletin Board Setup Analysts with this role can configure the Bulletin Board. Selecting this option will enable the Bulletin Board Settings option on the Bulletin Board group in the System Administration Explorer pane.
    Override Browse Limits Analysts with this role can override the browse limits set in the Browse Limits option within the System menu.
    Knowledge Bank Setup Analysts with this role can configure the Knowledge Bank. Select this to enable options in the Knowledge Bank group in the System Administration Explorer pane.
    Self Service Portal Setup Analysts with this role can configure the Self Service Portal. For more information, see Self Service Portal. Select this to enable options in the Self Service Portal group in the System Administration Explorer pane.
    Forum Management Setup Analysts with this role can configure the Forum Management settings. Selecting this option will enable the IPK Rules to Workflow Mapping and Forum Management Settings options in IPK Workflow Rules.
    Depreciation Setup Analysts with this role can configure the Depreciation settings. Select this to enable the options in the Depreciation group in the System Administration Explorer pane.
    Text Retrieval Setup Analysts with this role can configure text retrieval settings. Select to enable options in the Text Retrieval group in the System Administration Explorer pane
    Integration Setup Enables Analysts with this role can configure the Integration module. Select to enable the Integration option on the Admin menu
    Configuration Portability Analysts with this role can set up their Configuration Portability module. Select to enable the Configuration Portability option on the Admin menu
    AI Ops Enables the Using AI Ops option on the Admin menu
    ASM Designer Enables the Using ASM Designer option on the Admin menu
    Delegation Setup Analysts with this role can assign their action items to someone else. Enables the Delegation Setup option on the Admin menu
    API Lock By default, when a call, request or task is updated via the API, the API Analyst takes ownership. Select this option to prevent the API Analyst from taking ownership.

    If the option is not selected, there will be two history records, showing both Analysts. If it is selected, there is only one history record.
    Manage Searches Admin

    Analysts with this role can access the search center. This is accessible via Saved Searches > Manage Searches and lists all the saved searches the user has access to, sorted by type. You can use it to save and share searches.
    Nano Setup

    This setting applies only to Nano.

    Analysts with this role can access the Nano group in the System Administration Explorer pane.
    Alemba API Explorer Analysts with this option selected can access the Alemba API Explorer. The standard security as set in their Person Details will then apply.
    Nano Analyst Analysts with this role are redirected to Nano when attempting to log into ASM Core. Analysts with this role cannot log into ASM Core.
    Restricted Nano Analyst When using Nano, Analysts with this role can only see calls and requests they are working on or that have been assigned to them; meaning they must be the 'Current Analyst' or 'Forwarded to Analyst' of the call or request.
  3. Select to save the changes and close the window. Provide the Change Reasons if prompted to do so. Alternatively, select another tab, if appropriate.
