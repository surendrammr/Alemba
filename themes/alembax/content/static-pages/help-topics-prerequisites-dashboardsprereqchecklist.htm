Dashboards prerequisites checklist

You install the dashboards platform separately from Alemba Service Manager™ software.

For improved performance, we recommend that you install the Dashboard software on a separate Application Server. You must also install the Dashboard SDK software on the ASM Application Server.

You can use virtual machines to run your ASM System if the guest operating system is supported by Alemba® and the virtual machine vendor.

The recommended minimum requirements for dashboards are summarized below.

If your site has a large volume of data, you should consider using hardware with a higher specification.

Each Dashboard application has its own prerequisites. If you install these applications on the same server, then the prerequisites are cumulative, and are also in addition to the prerequisites for ASM. For example, if you choose to install the User Management Server on the Dashboard Application Server, you will need a recommended 8 GB RAM for memory. For more information about ASM prerequisites, see Prerequisites checklist.

Configuring Windows Server for Dashboards:

  • For Windows Server 2012, see Configuring Windows Server 2012 R2 for Alemba Service Manager™.
  • For Windows Server 2016, see Configuring Windows Server 2016 for Alemba Service Manager™.
  • For Windows Server 2019, see Configuring Windows Server 2019 for Alemba Service Manager™.

Additional considerations:

  • Ensure that the CPUs are not over-committed.
  • We recommend optimizing the web server for background service performance.

  • If required, you can configure multiple web servers under a load-balancer.

    Contact Alemba® Support regarding the best configuration for your requirements.

  • It is advisable to install a supported web browser – see Client requirements for more information.

  • If you use Unicode databases, install the relevant language packs associated with any non-English characters that your Analysts use. 

    This ensures that the indexing service on the web server performs indexing correctly.

Dashboard database server

We recommend that you install the dashboard database on a separate dedicated server from the ASM database to improve performance. If the Dashboard database is installed on the same server as the ASM database, it must have its own database and schema.

Alemba® does not support SQL Server Express editions.

Requirement Check?

Supported versions:

  • SQL Server 2012 (64-bit)
  • SQL Server 2014 (64-bit)
  • SQL Server 2016 (64-bit)
  • SQL Server 2017 (64-bit)
  • SQL Server 2019 (64-bit)

Minimum hardware requirements:

  • Consult your database vendor’s documentation for the minimum recommended hardware configuration required for the database version you intend to deploy.

  • 500 MB free space for the database.

  • We recommend that you install the Dashboards and Reporting software on a separate server to the database server.
  

Dashboard database user account permissions

For installations where a new SQL Server database is used, we recommend that you create a single database user account for administering the Dashboards database and performing operations such as running upgrade scripts on the database. You can restrict this database user account to access the Dashboard database only.

Networking

Requirement Check?

The network location of the database server to the Dashboards server to the Alemba Service Manager™ server will impact performance. In your system design, you must consider network switching and firewall configurations.  While Alemba® are not responsible for your system performance and can only provide very limited guidance in this area, we can recommend third-party consultants to provide specialist advice.

  

SMTP Exchange Server

The Dashboard and Reports software uses SMTP for sending scheduled dashboards and reports.

Requirement Check?

Supported versions:

  • Exchange 2010
  • Exchange 2013
  • Exchange 2016
  • Exchange 2019
  • Exchange 365

Email Protocols:

  • Outgoing Email: SMTP

If you are using SMTP over SSL, ensure your web server can communicate with your SMTP server.

Other email servers like Novell GroupWise and IBM Lotus Notes are untested and not supported but may also work for the SMTP protocol.
