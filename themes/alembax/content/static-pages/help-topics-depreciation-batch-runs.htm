Batch run parameters

Batch runs are depreciation schedules that run automatically. They make scheduled depreciation calculations against items in your CMDB and you can filter them for specific CMDB items, users, organizations or locations. The ASM Core polling service activates the depreciation schedule in a batch run.

Defining the Depreciation batch run parameters

Before You Start

Enable the Depreciation Setup within your General Access Security Role in order to configure any Depreciation administration settings.

You must first configure your Depreciation Schedule and Depreciation Methods before configuring batch runs.

You must have system administrator privileges to configure batch runs.

  1. Select ≡ > Admin > System Administration.

    The System Administration window is displayed.

    In the Explorer pane, locate the Depreciation group. You may need to expand the group to see the options within it.

  2. Select Batch Run Parameters to display the window.
  1. Complete the details, as follows:

    2. Depreciation Schedule Name Select the depreciation schedule for the batch run from the list. You can only run one schedule at a time as part of a batch job. You can create new schedule names in the Depreciation Schedule Names window.
    Batch Number The batch number is automatically set by the system and cannot be edited
    Month Ending The final month and year for this batch run in the field. The depreciation calculation will be performed up to this month. When you click on the right arrow in the calendar, the calendar closes.
    Filter Options
    CMDB Item Type Use the menu button to search for and identify the item type
    User Use the search button to search for and identifya specific User
    Organization Use the search button to search for and identifya specific organization
    Location Use the search button to search for and identifya specific location

    Before you continue, re-check all the parameters you have set because once this process starts, it cannot be canceled.

    Let's assume you have linked CMDB types A and B to your depreciation schedule. In the filter options, you select to filter the batch run to a certain location. However, the only CMDB items at that location are of CMDB type C, which isn’t linked to the schedule. The batch run will be performed, but no write down calculations will occur since CMDB type C is not linked to the particular schedule.

  2. Select . A message will display on the toolbar informing you that the batch run has been scheduled. The actual calculations will be performed the next time the ASM Core polling service is invoked.
  3. To view the results of the batch run, select the Batch Run History option.
