Quick Launch Explorer Option

Quick Launch URLs are shortcuts that let you run a URL through the click of a link. Parameters such as task or call fields, or even fields on a CMDB item linked to the task/call can be passed to a URL. This is useful, for example if you want to easily access a website.

All quick launch URLs, including the parameters used, have been configured on the Quick Launch window in System Administration. For more information, see Quick Launch on page 1.

If you run a quick launch URL which uses CMDB item fields as parameters passed to the URL, ASM will use the currently selected configuration item or service linked to the task. If no configuration item is selected, the one with the lowest reference number is used.

Running a Quick Launch URL

Before you start

Quick Launch is only visible if the quick launch functionality is enabled in System Settings.

You must have access to quick launch URLss on the Task Quick Launch tab of your Workflow Management Security Role.

 

  1. In the Explorer pane, expand Quick Launch to view a list of all the quick launch URLs to which you have access.
  2. Select the URL you want to run.

    3. All quick launch URLs run outside the ASM browser, meaning that existing organization IT policies regarding running URLs will still be enforced. Therefore if your IT policy does not allow you to run certain URLs, you will not be able to run them even if they are displayed on the Quick Launch tab.

  3. If this quick launch URL has been configured to update the task history, the task history will be updated with the entry saying “Quick Launch Application Selected” plus the name of the URL run from the request, and any parameters that have been passed to the URL.

Receiving output or errors from a quick launch application

ASM will try to update the user of any output it receives from a quick launch URL that has been run from a request. Any output received can be regular output, or an error.

  • If ASM receives regular output from a quick launch URL, an informational popup dialog will display with the output message. If the quick launch URL has been configured to update the task history, an entry titled “Quick Launch Output” will be added to the task history. This entry will include the name of the URL from which an output has been received, plus any configured parameters and the output received from the quick launch URL.
  • If ASM receives an error from a quick launch URL (for example, incorrect parameters has been passed to the quick launch URL), a warning popup dialog will display with a generic error message “Unable to launch application. Please contact your system administrator”, plus the details of the error message received.

Not all URLs will return an output to ASM. While ASM will display any output received from a URL, you may not see any response in the cases where an URL does not provide an output.
