Contract Transactions

ASM has two Application Programming Interfaces (or APIs), which enable you to develop programs that automate many common ASM transactions. This section of the documentation deals with the Classic API. For more information on the RESTful Alemba API, see here.

The ASM web service provides basic contract transactions such as create, update, delete and retrieve. In addition to this, the web service can also be used to link and unlink a contract to a Configuration Item or External Contact.

The Classic API MessageStakeholders transaction also lets you send an email message to stakeholders linked to a contract. For more information, see MessageStakeHolders Transaction.
