Viewing the Request History

Once a request is created, any analyst with the appropriate rights can view its history. The request history shows in detail all stages of the request, including any actions carried out for its resolution and notifications sent to users about the current status of the request.

  1. On the Request Details window, select one of the following:
    • on the toolbar
    • History explorer option
    • View History in the Actions & Solutions section.
  2. The Request <number> History window appears. The request history shows actions carried out for the request at previous stages, including both the notes entered in the Actions & Solutions field by handling analysts and automatic entries generated by the system.

    3. The Unspecified security profile is applied to any system-generated request history entries.

    Each entry includes:

    1. The type of action taken
    2. The analyst who initiated the action (if applicable)
    3. The date and time when the action was taken on the request. Request action times are displayed according to the local server time, regardless of the time zone in which the analyst is located.

      4. If the Portal symbol appears next to a history entry, it means this entry can be viewed by a user on the Self Service Portal

  3. To view the details of all actions on the Request History window, select on the toolbar.
  4. To view the details of a specific action, click the right-pointing arrow in front of the history entry.
  5. By default, only entries for active tasks, and the request are displayed in ascending chronological order. To change the order in which the history events are displayed, click on the toolbar to display the events in decreasing chronological order. The request history can again be displayed in ascending chronological order by clicking .

    6. The request history also displays information about:

  • Details of any Service Level Agreement escalation or breach events
  • Calls or requests linked to the current request
  • Objects attached to the request, recorded as a hyperlink. Click the link to open the attachment
  • Task history entries
  • The request history is filtered to only show history tasks that were active when the history was created and preset history types. To change the filter, click on the button in the toolbar. Select the filter options you require and click Apply. Your settings will be saved, and used whenever you open the request history screen. To view history for tasks that were inactive when the history was created, you will need to check the Show All History Messages checkbox.
  • You must select at least one filter from either option. If no option is selected for the filter, the filter is ignored and all items are displayed.
  • To hide the filter options click on the button. The expanded state of the filter is remembered when navigating away from the history window.
