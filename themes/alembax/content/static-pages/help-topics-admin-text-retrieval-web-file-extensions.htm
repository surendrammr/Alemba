Web File Extensions

When you are indexing across external web sites (URLs) to build your knowledge bank, you can specify the file types which are required to be searched. For example, you may only want to search across HTML files, or you may also want to search TXT files.

Many web pages are in fact generated ‘on the fly’ by server-side scripting languages, such as PHP, ASP and JSP, so you may want to include these in your file extension list. ASM Core provides a standard list of common web file extensions by default. You can modify this list by adding or removing file extensions according to your requirements.

You cannot specify web file extensions per domain.

Adding a Web File Extension

Before you start

You must have Text Retrieval Setup selected in your own General Access security role before you can add a web file extension.

  1. Select ≡ > Admin > System Administration.The System Administration window appears.

  2. In the Explorer pane, under Text Retrieval, select Web File Extensions.

    A browse table appears, with all of the extensions currently held.

  3. Select . A blank row is inserted in the browse table.
  4. At Extension, type the extension you want to add to the list.
  5. Use exactly the characters of the extension (that is, do not type in ‘Text’ if you want to search across text files, rather type in the letters TXT, which is the extension for text files).
  6. Repeat for each extension you want to add to the list.
  7. Select to save the changes. Provide the Change Reasons if prompted to do so.

Renaming a Web File Extension

Before you start

You must have Text Retrieval Setup selected in your own General Access security role before you can rename a web file extension.

  1. Select ≡ > Admin > System Administration.The System Administration window appears.
  2. In the Explorer pane, under Text Retrieval, select Web File Extensions.
  3. In the browse table, select the web file extension you want to edit.
  4. Overtype with the new characters you want to assign for the selected extension.
  5. Repeat for each extension you want to rename.
  6. Select to save the changes. Provide the Change Reasons if prompted to do so.

Deleting a Web File Extension

Before you start

You must have Text Retrieval Setup selected in your own General Access security role before you can delete a web file extension.

You cannot reverse this procedure. If you delete an extension by mistake, you have to re-create it by adding the extension to the list.

  1. Select ≡ > Admin > System Administration.The System Administration window appears.
  2. In the Explorer pane, under Text Retrieval, select Web File Extensions.
  3. In the browse table, select the web file extension you want to delete.
  4. Select . 
  5. Repeat for each file extension you want to delete.
  6. Select to save the changes. Provide the Change Reasons if prompted to do so.
