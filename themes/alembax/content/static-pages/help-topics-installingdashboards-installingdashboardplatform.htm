Installing the dashboard platform

Use the ASM Support Service Catalog to request the Dashboard Platform software. Speak to your Account Manager for more information.

Before you start

Before you install the dashboard platform:

  • Back up your servers.
  • Ensure your system meets the minimum technical requirements and is configured correctly. For more information, see Dashboards prerequisites checklist.
  • Ensure you have installed the correct service pack for your operating system, and the correct version of MMC.
  • Installing the Dashboards platform will stop IIS and any dependent services. Ensure that you install Dashboards at a time when IIS and its dependent services are not needed.

To install the dashboard platform:

  1. Extract the installation package to the Dashboard Application Server.

    We recommend you extract the installation package to a folder on your c:\ drive. For example: c:\Temp

  2. Using Windows Explorer, right-click AlembaDashboardInstall.bat, then select Run as administrator on the shortcut menu.

  3. At the prompt Is vFire installed on the same server?, type either:

    y – if the current machine has ASM installed. 
    n – if ASM is not installed on the current machine.
  4. Press the Return key.

  5. At the prompt Dashboard Server – Proceed with installation?, type either:

    y – to install the Dashboard Server, deploy PantomJS, and configure the Dashboard Server Web Applications.
    n – to skip Dashboard Server installation.
  6. Press the Return key.

  7. At the prompt User Management Server – Proceed with installation?, type either:

    y – to install User Management Server and configure the UMS Web Applications.
    n – to skip User Management Server installation.

  8. Press the Return key.

  9. At the prompt Dashboards Platform SDK – Proceed with installation?, type either:

    y

    – to install the Platform SDK, deploy PhantomJS, deploy CeFSharp, and host the Dashboard Platform SDK Service in IIS.

    If you are installing on multiple servers, ensure you install the Dashboards Platform SDK on the same server as the ASM application.
    n – to skip Dashboards Platform SDK installation.
  10. Press the Return key.
  11. At the command prompt, perform an iisreset to restart IIS.
