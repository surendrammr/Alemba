Event Management Source Parameters

Besides the common fields for configuring an integration source, more fields are available when you configure a source for an event management connector:

Max Number of Transaction Attempts

Specify the number of attempts for ASM Core to establish a connection with the source before declaring a failure to connect. This applies to each event management transaction that ASM Core attempts to send to the source

Time between Transaction Attempts

The length of time between each connection attempt

Asynchronous Transaction Timeout

The maximum length of time given to the connector to notify ASM Core as to whether a transaction has completed or failed. This is a mandatory field.

Synchronous transactions are transactions that complete or fail immediately when they are executed. Asynchronous transactions, on the other hand, are transactions that do not complete or fail immediately: the connector notifies ASM Core within a specified time as to whether the transaction has completed or failed. The timeout is the maximum time given to the connector to notify ASM Core. If the transaction is not completed or failed within this period, the transaction is declared “timed out” and the attempt considered failed.

This setting applies to individual transactions for a particular event – not to the event itself. For example, an event has an unlimited amount of time after the CREATE event to send ASM Core a RESOLVE or a DELETE transaction. However, each individual UPDATE or NOTIFY transaction related to that event is bound by this setting.

Send Fault Notifications

Select this option to send notifications to specified parties when a transaction fails to be sent to the source. When this option is selected, the fields below are enabled. Use these fields to configure notifications and the recipients who need to be notified.

Always

sends notifications to the specified recipients after each failed attempt

After

sends notifications to the specified recipients only after a defined number of failed attempts. Type the number of attempts in the Failed Attempt(s) field
