About the Request Details window

You can use the Request Details window to specify and view the details of a request, and also perform actions on it such as forwarding to another analyst for review or further action. You can also specify request implementation dates and tasks

 

The Request Details window appears when you:

  • log a request
  • select a request from the Requests Outstanding window
  • select a request from the results browse table of a Request Search window.

The Request Details window consists of:

  • The Request Details Explorer in the left frame, which contains links to other windows related to this request. The Service Window in the lower part of the explorer pane displays any SLM events relating to the request as a result of an SLA being applied to the Request.
  • The main Request Details window with details fields which contains all the information about the request.
  • The Information Panel on the right hand side of the window displays important request information such as analyst, SLA, and request status details.
  • Tabs in the lower part of the Request Details window displays information relating to the current status of tasks linked to the request, including an overview of information relating to the tasks linked to a request, as well as the current budget, time and description statuses of these tasks.

The specific fields on the Request Details window will depend on your system, the configurations your administrator has applied, and any configurations to field names. Requests are based on workflow templates that have been set up for a workflow process in your system. Workflow templates are created using the Workflow Template Details window, which contains the same fields as a Request Details window but is saved as a template.
