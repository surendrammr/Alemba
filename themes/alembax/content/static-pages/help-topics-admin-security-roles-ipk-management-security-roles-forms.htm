IPK Management Security Role: Forms Tab

Forms is an aspect of ASM Core that has largely been superseded by ASM Designer.

The Forms tab on IPK Management Security Role window lets you control access to call forms. You can set which forms Analysts with this IPK role can access, and specify particular types of access for each form.

Before you start

You must have Security Roles setup selected in the Admin tab of your own General Access security role before you can assign or remove permissions for any security roles.

To use the options on this tab, you need to select the Forms option in System Settings.

To set permissions relating to CMDB forms, use the Forms tab within the Configuration Management Security Role.

  1. Display the IPK Management Security Roles details window, if it is not already on screen.

     

    Select ≡ > Admin > System Administration.

    The System Administration window appears.

    In the Explorer pane, expand Security Roles, then select the type of role you want to view. All existing roles of this type are displayed. Now you can create, update or rename a security role of this type.

    The pre-defined role All typically has all role options selected. However, some of the options are not selected by default and you should explore the permissions set before assigning the role.

    If you want to create a role, select and assign the role a name in the Role Name field. To view or update a role, select the role in the browse table and select .

  2. Select the Forms tab.
  3. Select Form Security to give Analysts with this role access to particular forms. Deselect the option to give the role access to all IPK forms.
  4. To enable Analysts with the role to access a form, select the form in the Select Form table and click Add to List.
  5. Set the access type for the form, by clicking in the Access column for the form and selecting Read Only which enables Analysts to view the forms or Read/Write to allow Analysts with this role to open and edit forms of the selected type.
  6. To remove a form from a role, select the form in the Selected Form table, and click Remove.
  7. Select to save the changes and close the window. Provide the Change Reasons if prompted to do so. Alternatively, select another tab, if appropriate.
