Configuring the ASM app

The ASM app is designed for end users and analysts to access the ASM System through a mobile device. Users can access their calls, approvals, orders and asset lists. They can log calls, add notes, view the history of the call, action approvals, and view orders and assets. Analysts can view their call lists and details of calls assigned or forwarded to them. They can forward the call to another analyst or group, update the call notes, review call histroy, change the call status or close the call.

First, check that your system meets the required prerequisites. Then ensure that your users have the access permissions required to allow them to log calls, and access appropriate call templates, services and assets.

For information about the ASM app prerequisites, see Prerequisites checklist.

Logging into the ASM app

Analysts and users must exist in ASM Core. They will use these credentials to log into the ASM app.

When users or analysts first attempt to log into the ASM app, they must supply the following information:

Server Name The server IP address or server name
System Name The system name, as defined in the Server Console, under Virtual Directory
HTTP Secure Choose whether you want to use this setting

The ASM app supports windows authentication. For more information, see Authentication.

ASM app and partitioning

The ASM app uses the same partitioning rules as the Self Service Portal.

Access permissions

The ASM app uses the security role and other permissions set for the user or analyst in ASM Core.

ASM app user permissions

Users can log calls, and view their calls, approvals, orders and assets using the ASM app. ASM app user permissions are determined by their Self Service Portal role settings.

To configure ASM app users to... Select the following permissions on their Self Service Portal role... 
Log calls Options tab – Log Calls. Calls logged using the ASM app have a Portal logging method.
Access call templates when logging calls Call Templates tab – ?. Call template access is also determined by the stream and status of the call.
See their own services* CMDB tab – Review Own Services
See all/their organization's services* CMDB tab – Review All Services
See their own assets* CMDB tab – Review Own CIs
See all/their organization's assets* CMDB tab – Review All CIs

* The logged in user must appear in the in the User field on the Service Details window to see the relevant services, or in the User field on the CI Details window to see the relevant assets.

ASM app analyst permissions

ASM app analysts can view their Viewing Call Lists in the ASM app (Analyst login), and details of Viewing and Updating Calls in the ASM app (Analyst login)assigned or forwarded to them. They can Forwarding a Call Internally to another analyst or group, Updating the Call Notes, review Viewing Call History, change the call status or Closing a call.

ASM app analyst permissions are determined by their Self Service Portal role settings.
