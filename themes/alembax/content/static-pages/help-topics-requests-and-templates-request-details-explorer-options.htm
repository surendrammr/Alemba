Request Details Explorer Options

The Explorer enables the user to view or record additional information about the request (for example, attach files), initiate action on the request, or access features such as the audit trail of changes made to the request.

Options that are dimmed are unavailable either because the functionality has not been enabled by the system administrator and is dependent on another setting, or because you first need to save and reopen the Request Details window to access this option.

Within the Details group, you can select:

  • SLM to view any SLA (Service Level Agreement) that is linked to the current request
  • History to show all stages of the request, including any actions carried out for its resolution and notifications sent to users about the current status of the request.
  • Authorization to authorize the current request so it can start being implemented. Authorizing a request releases all tasks and approvals in the request.

    • You can pre-authorize requests by authorizing the workflow template.

  • Update Dependencies to view the workflow of tasks and approvals that are associated with the request, as well as their interdependencies. It also enables you to modify this workflow by adding or removing tasks on the dependency diagram.

Within the Data group, you can select:

  • Object to attach screenshots and files from Windows applications to the current request. These files can, for example, provide additional information about the request to contribute to its implementation
  • Attributes to specify additional request information relating to categories defined by the system administrator
  • Forms to attach a form to the current request. Forms are specially designed to capture general, non-reportable, non-searchable information about a request. ASM Core is provided with a default (blank) form for requests. You can use the ASM Designer to add fields to the default form or create new forms.
  • Events. If the current request was logged as a result of an incoming event (such as a network issue) received from an external system, this option enables you to view the transactions exchanged between ASM Core and the external system in relation to this event. You can view event information for the root node, transaction history, transactions and transaction audit trail.
  • Inbound Actions. If the current request was logged as a result of an Inbound Action received from an external system, this option enables you to view the messages exchanged between ASM Core and the external system in relation to this Inbound Action. The window displays the source, type and reference IDÂ of the inbound action, as well as the date and time the transaction was sent, its origin and type, details and status.
  • Audit to display an audit history for the request. The audit trail must be enabled in the Workflow Management Settings window.
  • Transactions to initiate Asset Management transactions from the current request. Asset Management transactions include receiving, updating and deleting asset orders, and allocating assets to people and/or CMDB items.
  • Transaction History to search on transactions added to a software product or inventory through a request. Every time a new transaction is performed on an inventory or software product, for instance to add a new purchase of items/licenses or allocate a quantity of items/licenses, a new record is added to the transaction history. This window contains filter options for the software product or inventory details and a browse table with a list of the transactions added to the software product or inventory for the current request.

  • The number of transactions shown in the browse table is limited to the search browse limit set by the system administrator.

  • Change Windows to view and/or add Change Windows for CMDB items linked to the current request.
  • Outages to view the outages planned for any CMDB items linked to the request
  • Create MS Project to transfer the details of a request, as well as its associated tasks and approvals to Microsoft Project
  • Gantt Chart to display a comparison of the planned and actual times taken to complete not only the current request, but the associated tasks within.
  • Knowledge to search for Knowledge Bank entries and link them to the current request, and create knowledge entries from the request

The Quick Launch option allows you to launch URLs from within ASM Core. Only Quick Launch URLs you have access to as part of you Workflow Management Security Role settings are displayed.

Within the Communications group, you can select:

  • Add Note to add a note to the current request. This is a useful functionality when a request has been forwarded to another person and you do not currently own the request, or do not want to take ownership of it, but you wish to add some details to it.
  • Email to inform and update Users and other analysts of developments and issues of interest from within ASM Core.
  • Letter to create letters (documents) directly from the Request Details windows, using applications such as Microsoft Word, and attach those letters to the request. The letters are pre-formatted, containing details from the current request.

Within the Workflow group, you can select:

  • Defer enables an analyst to postpone action on a request they currently own. This is useful if, for example, you want to record any action you have taken on the current request but are not yet ready to complete it or forward it to another analyst. Before deferring the request, you can specify the current status of the request while it is deferred, suspend the request, set a reminder, and indicate that the user has been notified about the current state of the request.
  • Save to defer the request while keeping the Request Details window open
  • Complete to indicate that the state of the current request is no longer open. It may be because the request has been implemented or because it has been cancelled
  • Internal to forward the current request to another analyst or group within the organization
  • Screen Set to change the request screen, by selecting another screen set which contains your custom request details screen.
  • Linking to link calls or requests for change that relate to the request that is open
  • Stakeholders to link stakeholders to the current request so that they are informed of developments while the request is being implemented
