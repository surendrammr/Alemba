Configuring the User Management Server

To customise the Dashboard Server:

  1. Navigate to the following location in your browser: http://localhost/AlembaUMS

    If you have already logged into the Dashboard Server, you are logged in automatically. For more information, see Configuring the Dashboard Server

  2. In the left-hand menu bar at the bottom, select to view the Settings.

  3. On the Site tab, configure the following general settings:

    Setting Description
    Organization Name

    User Management Server name that appears in the browser title bar and at the top of the default email template.

    Company Name User Management Server
    Site URL If required, you can change the User Management Server URL. Consult Support before you change this setting.
    Welcome note

    Text up to 70 characters long.

    Welcome to Company Name User Management Server

  4. Configure the following Images:

    Setting Description
    Log In Screen We recommend this image is 240 x 120 pixels.
    Header Logo We recommend this image is 40 x 40 pixels.
    Favicon The favourite icon that appears alongside the browser title. We recommend this image is 16 x 16 pixels.

  5. Configure the Display settings as required.

  6. Disable Show powered by Syncfusion by moving the toggle to the left.
  7. Disable Show copyright information by moving the toggle to the left.
  8. Leave Show server version enabled to assist with support.
  9. Select Save to save your changes and update the settings.
  10. In the left-hand menu bar, select to open Application Management.

  11. To the right of the Syncfusion Dashboard Server application, select > Edit.

  12. Change the Application name to Company Name Dashboard Server.

  13. If you want to configure external access to the Dashboard Server, select Add Url, then type the IP Address and path of the Company Name Dashboard Server.

    For example: http://192.0.2.0/alembadashboardserver

  14. Ensure you are using the correct protocol. If required, select the http dropdown list to change the protocol to https.

    For more information, see Configuring the web protocol.

  15. Leave Allow access to all users enabled.
  16. If required, select the Icon, then select to browse for and replace the icon.
  17. Select Update to apply your changes.
  18. Select to view the Settings.
  19. On the Accounts tab, expand Password Policy.

  20. Disable all the options by moving every toggle to the left.

    You cannot disable Minimum Length, which is set to six in accordance with the ASM password policy. See the online help for more information.

  21. Select Save to save your changes.
  22. Select to refresh the Application Management menu and see your changes.

What's next?

If you want users to have external access to the servers, see Configuring external access to the Dashboard Server.

If you want to enable User Management Server for Active Directory, see Enabling User Management Server for Active Directory authentication.
