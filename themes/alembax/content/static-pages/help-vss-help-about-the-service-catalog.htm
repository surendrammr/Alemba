About the Service Request Catalog

The Service Request Catalog is a listing of all services that are available to you. The Service Request Catalog menu option provides a search facility that enables you to search for and view available services. You can search for services and log an Incident against a service or request a service.

Within Self Service Portal, Services have already been linked to workflows. This means that when you order Services, your system has already been configured to execute the appropriate request(s) that will be used to deliver your Service. You only have to complete and submit your order of Services; the appropriate request(s) will be logged to ensure appropriate Service Delivery.

Each service in the catalog will display the following information:

Where there is no information against a field for a service, this field will not be displayed.

Service Ref Reference number assigned to the service
Description of Service Describes the service in detail, and includes what the user who is subscribing to the service will get as part of the service offering
Date Service Introduced This is the date from which the service has been available to users
Date Service Withdrawn This is the date on which the service was taken off the Service Request Catalog, and since has been unavailable to users.
Availability/ Service Times This is the duration for which the service will be available to use/access. For example, if a service provides online banking facility, the service would typically be available 24 x 7, which means 24 hours a day and 7 days a week.
Target Audience This field identifies the type of users most likely to subscribe to the service. For example, an organization may provide a service for creating employee accounts. The target audience in this scenario would be all of the organization’s employees who need a new account created.
Support Arrangement This field provides details of the analyst/department that is responsible for providing service support to users
Service Targets This field displays the terms and conditions under which the service will be provided.
Training This field describes any training related to this service, which may be of interest to subscribers.
Charging Arrangements This field displays any costs associated with the provision of this service.
Approval Required This field provides the details of the approval required for subscribing to a service.
Access This field contains details of the users/departments that have access to this service.
Backup and Recover Arrangements This identifies the backup arrangements at times when the service is unavailable.
Hyperlink This field displays the address (link to) of the web site that provides details about this or a related service.

There are 3 types of searches you can generate:

  • Your Services
  • Your Organization’s Services
  • All Services

For services that are currently active and available in the Service Request Catalog, you have the option of logging an Incident for these services. For services that are enabled for viewing against your user role but are not currently activated, you will have an option of logging a request for those services.

Viewing the services within the Service Request Catalog

  1. From the My Options menu, select the Service Request Catalog option, and then Browse the Service Request Catalog.
  2. The Service Request Catalog page appears, with a list of promoted or popular items displayed on the home page.

    3. Your system administrator will populate this page with selected items.

  3. You can find the item you wish to order in a number of ways.
    • You can filter your display using the button. Select the button. A drop-down list of the top tier portal display categories appears, with arrows indicating those categories that have further sub-categories. Hover over the relevant category to see the contents of the lower tiers. Then click to select the item you wish to view . You can move back up the levels at any point by clicking on the category name . An "x" appears in the Select a Category button , which you can use to clear the filters and return to the full menu.
    • You can type some or all of the item name in the text field, and then select . All items matching the criteria will be displayed.
    • Click on the button to display the Advanced Search page, in which you can search on title, ref, class, type, and so on.
  4. To view any of the item details, click on the item name. The details are displayed, and you can order from this page, or return to the summary page.

    5. If you are using Internet Explorer and have scripting enabled, you will not be able to enter any negative or non-numeric quantities. Otherwise, any invalid quantities entered into the Quantity field will be checked after you click on the Order button. In this case, if you have specified any invalid quantities, an error message will display.

    There is no maximum number of Services which you can order. You should however consider the effect on performance when ordering a large number on services, as a submission form must be filled out for each Service instance in your order.

  5. When you have identified the items you want to order, click on the button.
  6. Depending on your system configuration, you may be prompted to give a business justification for the order. You can also use this page to attach any files that support your order. You may find that some orders require additional information. When they are completed, the Your Order page appears, listing all of your ordered items.
  7. You can use this opportunity to make any changes, before selecting Submit Order.

    8. If an error message appears and you are unable to log a request for service, contact your system administrator. You may get an error if the call prototype for logging a request for service is not configured correctly in ASM Core.

  8. Your ordered items will be listed on your home page.
