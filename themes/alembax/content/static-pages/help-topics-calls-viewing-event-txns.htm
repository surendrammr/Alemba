Viewing Event Transactions from a Call or Request

When an event triggers a call or request in ASM, analysts can manage the transactions relating to the event from the Events explorer option in the Call Details or Request Details windows.

If event transactions are defined for the call or request, an orange square indicator appears next to the Events option.

Select the Events Explorer option.

The External Events window displays information about the external system that generated the event for which the call or request was logged (the name of the source, event type, mapping and instance of the event), as well as a summary of transactions exchanged between ASM and the external system in relation to this event.

The browse table lists the transactions between ASM and the external system, including:

Date/Time

the date and time the transaction was sent
Origin

the name of the system that sent this transaction (either ASM or the external source)
Transaction Type

the type of transaction that was sent (acknowledge, update, notify, close, and so on)
Details

the contents of the transaction (if any)
Status

the status of the transaction (Succeeded or Failed)
