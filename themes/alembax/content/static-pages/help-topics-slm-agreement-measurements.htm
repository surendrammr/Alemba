Agreement Measurements

You can record measurements against an agreement. As part of the standard application, measurements are just text fields stored against the agreement. They do not interact with any other component of SLM or ASM Core. Measurements can be integrated with other components of the application as part of a consulting project.

Before you start

You must have Agreement New selected in the Options tab of your Service Level Management security role to create agreements. You must have Agreement Update selected in the same tab to update the information.

  1. Display the Agreement Details window, if it is not already visible on screen, by searching for the agreement.
  2. In the Explorer pane, expand Data, then select Measurements.
  3. The Agreement Measurement window consists of the main measurements table, the Measurement Description area, and the Penalty Description area. Complete the details.
    4. Agreement Title This is a read-only field displaying the title of the agreement.
    Agreement Reference This is a read-only field displaying the reference for the agreement.
    The main table contains of the following columns for Name, Unit, Amount and Time. The function of these columns will vary depending on the particular setup for measurements you are using on your system
    Measurement Description Type a description of the measurement as it relates to the particular agreement.
    Penalty Description Type a description of any penalties that might apply as a result of breaching the agreement.

    Not all agreements have penalties. This will depend on the nature of the agreement and the needs of your business.
