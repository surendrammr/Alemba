Service Link Types

You can define service link types that Analysts can use when linking a service to an agreement from the Linked Service Items explorer option of the Agreement Details window, or the location detail window.

Creating a Service Link Type

Before you start

You must have CMDB Setup enabled within your General Access security role in order to configure any CMDB administration settings. Depending on if or how your system is partitioned, ensure that you are working in the correct partition.

  1. Select ≡ > Admin > System Administration.

    The System Administration window appears.

    In the Explorer pane, expand CMDB.

  2. Select Service Link Types to open the window. Existing service link types are displayed in alphabetical order.
  1. In the CMDB Type, select either Location or Agreement from the drop-down list, depending on the link type you wish to create.
  2. Select . A row is added to the browse table.
  3. Type the name of the link type.

    4. This link type will only apply to the CMDB type currently selected (Agreement or Location).

  4. Select to save the changes. Provide the Change Reasons if prompted to do so.

Once you select , the browse table displays the entries in alphabetical order.

Renaming a Service Link Type

Before you start

You must have CMDB Setup enabled within your General Access security role in order to configure any CMDB administration settings. Depending on if or how your system is partitioned, ensure that you are working in the correct partition.

  1. Select ≡ > Admin > System Administration.

    The System Administration window appears.

    In the Explorer pane, expand CMDB.

  2. Select Service Link Types to open the window. Existing service link types are displayed in alphabetical order.
  1. In the CMDB Type, select either Location or Agreement from the drop-down list, depending on where the link type you wish to rename is held.
  2. Select the link type you want to rename in the browse table.
  3. Overwrite with the new name you want to assign to the selected Service Link Type.
  4. Select to save the changes. Provide the Change Reasons if prompted to do so.

Once you select , the browse table displays the entries in alphabetical order.

Deleting a Service Link Type

Before you start

You must have CMDB Setup enabled within your General Access security role in order to configure any CMDB administration settings. Depending on if or how your system is partitioned, ensure that you are working in the correct partition.

You cannot reverse this procedure. If you delete a link type by mistake, you have to re-create the entry by creating a new link type.

  1. Select ≡ > Admin > System Administration.

    The System Administration window appears.

    In the Explorer pane, expand CMDB.

  2. Select Service Link Types to open the window. Existing service link types are displayed in alphabetical order.
  1. At CMDB Type, select either Location or Agreement, depending on where the link type you wish to delete is held.
  2. Select the link type you want to delete in the browse table.
  3. Select . If you attempt to delete the system default, you will receive a warning message.
  4. Select to save the changes. Provide the Change Reasons if prompted to do so.
