Viewing Integration Log

The Integration Activity window enables you to view all integration scheduling activity that has run on sources.

Before you start

The ASM Core Integration Polling Service must be running for scheduled scans to run on Federated CMDB sources in the defined period. Event Management events are detected through the polling service and are not dependent on integration scheduling.

You must have Integration Setup selected in the Admin tab of your General Access Security Role.

  1. Select â‰¡ > Admin > Integration.

    The Integration Platform Settings window appears.

  2. Select the Activity option from the explorer pane .
  3. Complete the details:

    4. From and To

    Select the start and end date to define the period in which you want to view activity.

    Connector

    select the connector for which you want to view activity

    Source

    select the source for which you want to view activity
  4. Select the Log view to view all event messages and warnings recorded in the period. To view the log details for a particular activity error or warning, double-click the entry in the browse table. The following types of integration activity may appear in the Log tab:
    1. Activities that relate to the Federated CMDB, such as errors while importing resources, connection errors, errors when the CMDB is not updated, when resources are unlinked. Depending on the integration connector and source, this may include directory server integration events, such as adding or deleting analysts and unforeseen errors such as network outages and security errors. Situations where person records could not be imported for reasons such as when their login ID is already in use by another person is also reported here. Details will include the person using the login ID and the person that could not be imported.
    2. Event Management events such as the details of any attempted communication to a third party system, any errors generated when sending or receiving a transaction, or configuration errors.

  5. The activity log only records warnings and errors for Federated CMDB activity. If a person record has been successfully matched from a directory server integration source, this information will be recorded in the Federated CMDB Administration window, and in the Audit Trail for the person details record.

    6. The following information is recorded in the synchronization log:

    Source

    The name of the source scanned or on which events were detected

    Level

    The type of log entry, such as Error or Warning

    Logged At

    The date on which the log was created

    Message

    The details of the error message

    Data

    The database details for the activity or event

  6. This information also appears in the Federated CMDB Administration window when you click the Synchronization Log view.
