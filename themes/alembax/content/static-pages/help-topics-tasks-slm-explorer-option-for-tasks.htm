SLM Explorer Option for Tasks

The Task Service Level Management window shows any agreements that are currently in effect against the request and task, and enables you to select agreements for the current task.

Only Organizational Level Agreements (OLAs) can be linked to tasks. OLAs are agreements between departments in an organization, while UCs are agreements with external suppliers. UCs can only be linked to External Supplier tasks. For more information, see Service Level Management on page 1

An agreement is normally applied automatically to a task based on the workflow selection criteria specified at the level of each agreement. Manually assigned agreements can only be changed manually.

  1. Select the SLM Explorer option from the Task Details window.

    2. Whenever an OLA and/or UC is applied to a task, the SLM option displays an orange square.

  2. This window displays the following fields:

    3. Request SLA The name of the Service Level Agreement that is applied to the request associated with the task. You can select an alternative agreement by keying it in.
    OLA The name of the Operational Level Agreement that is applied to the current task. You can select an alternative agreement by keying it in.
    UC

    The name of the Underpinning Contract that is applied to the current task

    You can view the details of a linked agreement by clicking the button next to the agreement you wish to view
    Planned Start Date Exceeded

    This option is only enabled if Planned Date is selected on the Workflow Management Settings window

    Select this option to enable notifications to be sent out to the request manager if the task does not start on the specified planned start date.

    The planned start date is indicated in the Task Planned field as read-only.
    Target Date Exceeded

    This option is only enabled if Target Date is selected on the Workflow Management Settings window

    Select this option to enable notifications to be sent out to the request manager if the task does not end on the specified target date
    Events

    A table listing all agreement events for this task. If you want to view only active (that is, current) events for the task, leave the Include Closed Events option unselected. To include events for the request, select Request Events.

    It is possible to hide selected SLM events from appearing in SLM reports.

    You must have SLM Task Exclusions selected in the Tasks tab or your Workflow Management Security Role to exclude events from SLM reports. If exclusion reasons are enabled in the SLM Settings window (by selecting SLM Reporting Exclusions), you will also be prompted to choose a reason for excluding the event.

    To exclude selected SLM events, select the checkbox in the Exclude column and click Apply.

    1. Select the checkbox in the Exclude column.
    2. Click Apply.

Any notifications about missed task dates will be sent to the Request Manager.  If the Escalation Log option is selected in the Workflow Management Settings, an entry about the escalation will also be added to the Task History.
