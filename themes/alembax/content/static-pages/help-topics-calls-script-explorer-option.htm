Script Explorer Option

A script takes the form of a question (or prompt) and various responses (options). Each option has detailed advice and may also have a question that leads to another level of options beneath it. You should ask the user the question and then select the option that corresponds to their answer.

Whenever Scripting is configured, a check is made after the issue type is selected on the Call Details window to see whether a script is available for that Type.

As you navigate through the 3-tiered problem type structure, if there is a script available for a specific type, the Script explorer option will turn red.

Therefore, as you browse through the Type hierarchy you can see immediately whether or not there is a script available for a specific type.

The Script explorer option is only enabled if the scripting is configured in the IPK Settings window.

  1. Display the Call Details window, if it is not already on screen.
  2. In the explorer pane, expand Resolutions, then select Script.

    3. While the Call Script window is open, you can switch back to the Call Details window, simply by moving the cursor across and clicking on any part of the inactive window. You will probably need to drag the Call Script window off to one side while it is still active, to view the Call Details window.

  3. In the Call Script window, at Script select the required script. Only those scripts that are linked to the issue type of the current call are displayed in this list.

    4. The appropriate script should be displayed automatically, otherwise check each one from the Script dropdown list until you find one that looks likely. Ask the user the top level of questions and, using a single click on an option, review their answers against the contents of the Possible Solutions/Required Actions field. To view the question relating to an option, double-click on the option in the list box. The question relating to that option appears in the Questions field at the top of the window and the advice appears in the Possible Solutions/ Required Actions field. You may be able to access an object linked to that advice. When you double-click an option, it is listed on the Session Dialog. Remember that as this information can be saved to the call, a large amount of text will be built up as you view the different options, and this will then be added to the call Actions and Solutions on the Call Details window. You can also view the possible solution for each option by highlighting the option and keeping the current question displayed. Often you will just narrow down the range of possible causes and the advice will simply be to go on to the next question.

  4. Use the selected script details to assist you as needed:

    5. Questions The question that helps define the script is automatically displayed in the Questions field. When any script option is selected using a double-click, the prompt for that option is then displayed here.
    Possible Solutions/Required Actions The Possible Solutions/Required Actions field displays any advice that has been defined for the selected option. This can be viewed by a single click on the option in which you are interested. There may be an object linked to advice, such as a spreadsheet, documentation and so on. This can be viewed using the Attached Object button. Note that this is an object linked to the script, and not the call. Hence, this object cannot be view on the Objects window.
    Session Dialog The Session Dialog displays the first entry as the script title and main question. Record the questions (prompts) and answers (options) by double-clicking on the option name in the list box. The Possible Solutions/Required Actions for an option are not recorded in this field, but any text you add in the Actions Carried Out field is (when you double-click the option name in the Script list box).
    Actions Carried Out

    Add any actions you may recommend or carried out while using the script in this field. Text added here is saved against the selected option into the Session Dialog, when the option is double clicked.
  5. Click Save to add the Session Dialog into the call Actions and Solutions on the current Call Details window. The Session Dialog field is cleared of text when this is done. Click Clear Session in the Call Script window to clear the contents of the Session Dialog.
