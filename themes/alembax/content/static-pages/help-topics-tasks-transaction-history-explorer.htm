Transaction History Explorer Option

The Transaction History explorer option enables you to search on transactions added to a software product or inventory through a call, task or request.

Every time a new transaction is performed on an inventory or software product, for instance to add a new purchase of items/licenses or allocate a quantity of items/licenses, a new record is added to the transaction history.

  1. Expand the Data tab on the explorer pane and select Transaction History
  2. This window contains filter options for the software product or inventory details and a browse table with a list of the transactions added to the software product or inventory for the current task. You can filter on CI Type, Item, Transaction Date From, Transaction Date To, Cost Center, Organization, Allocated to/Reserved for CMDB Item, Allocated to/Reserved for Person
  3. Select and specify the details by which you want to filter the transaction records.

    4. If you want to view transactions for a particular cost center, select Cost Center and in the next field specify the name of the cost center and press Enter. Click Refresh. The records are updated in the table.

  4.  

  5. Most of the columns in the table are self-explanatory. A description of some noteworthy columns is provided below:

    Source This column shows where the transaction came from. If the transaction was added through a call, it will state “Service Desk” or if through a Manage CMDB task, it will state “Polling Service”
    Source Item Type This column indicates if the transaction was initiated from a call, request, task, or CMDB item
    Source Item Ref: This column indicates the call, request, or task number, or the entity reference, for the call, request, task, or CMDB item from which the transaction was initiated
  6. To show or hide the columns in transaction history table, click Filter Columns.

The number of transactions shown in the browse table is limited to the CMDB Search browse limit set by the system administrator.
