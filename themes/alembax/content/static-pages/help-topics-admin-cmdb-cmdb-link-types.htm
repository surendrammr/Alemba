CMDB Link Types

You can define the types of links that can be used to link CMDB items, specifying which CMDB item types may be linked, as well as the type of relationship the two items can have. You can also set the impact relationship for the link type.

The link types you define between CMDB Item Types will be inherited by any sub types configured for a CMDB Item Type, and will display graphically on the CMDB Linking Diagram.

You can configure more than one link type between the same CMDB Types by using different link type categories. These include Peer to Peer, Parent-Child, Container-Component, and so on. For example, you can set up Peer to Peer link, and a Parent-Child link between two CMDB item types. Each link type must be associated with a particular link type category.

You can also restrict which item types can be linked. For example, you can allow Servers to be linked to Routers, but stop Routers from being linked to Keyboards. You can also set the impact relationship for the link type.

ASM Core provides the Link Types below by default. You cannot delete these link types. However, you can rename them or edit the details:

Service Bundle to Service Action links a service action to a service bundle so that a User ordering this service bundle must order the linked service action.
Service Bundle to Service Action (optional) links a service action to a service bundle so that a User ordering this service bundle does not have to order the linked service action.
Service to Service Action links a service action to a service so you can create an impact relationship between the service and any linked service actions.

Specific link types are targeted at Cloud Provisioning scenarios and are meant to be used in the Integration Platform when importing resources from the VMware vCloud Director. The names of these links all have the same “(vCloud)” prefix.

Creating a Link Type

Before you start

You must have CMDB Setup enabled within your General Access security role in order to configure any CMDB administration settings. Depending on if or how your system is partitioned, ensure that you are working in the correct partition.

  1. Select ≡ > Admin > System Administration.

    The System Administration window appears.

    In the Explorer pane, expand CMDB.

  2. Select CMDB Link Types to open the window. A browse table of all existing link types appears. You can adjust the column widths if required.
  1. Select .
  2. Specify the CMDB Link Type Details.

    3. Name Name of the link type
    Category

    A drop down list of link types to define the relationship between CMDB item types.

    Container-Component This can be used when you have a CMDB item that encapsulates other CMDB items.

    This category is useful when you want to abstract lower CMDB items away from the higher level CMDB linking diagram view, but still want to view them on the CMDB linking diagram when necessary. Take, for example, a Server Farm is the container which is connected to Servers (the component). On the CMDB linking diagram, the Configuration Manager will just see the Server Farm as the CMDB item displayed, although they can drill down further and see the configuration of the servers within.

    Parent-Child This can be used when two CMDB items have a logical parent-child relationship in the CMDB. For example, Email Service can be the parent of the Email Server.
    Peer-to-Peer This can be used when there is no hierarchical representation between two CMDB items. For example, a Server connected to a Network Printer.
    Summary This can be used when you have CMDB items that you would like to have entered in the CMDB, but do not really need to see on the CMDB linking diagram. They are listed in summary view on the CMDB linking diagram Info pane. For example, a Desktop CI is connected to Microsoft Office.

    The category that you select here will determine the item type fields which are displayed on this page. If you select the Summary category, the Summarized tab will display in the linking diagram information pane if the item is selected.
    Audit Check this box to enable auditing for this link.
    Impact Can Be Overriden Manually Check this box to allow analysts to override the impact relationship set by the system administrator for this link type. Analysts can modify the impact relationship when accessing a link on the CMDB Linking Diagram.
    Container The first CMDB Item Type in the Container-Component relationship.

    This field only displays if "Container-Component" has been selected under Category.
    Component

    The CMDB item type to which you want to link the Container item in the Container-Component relationship.

    This field only displays if "Container-Component" has been selected under Category.
    Parent

    The first CMDB Item Type in the Parent-Child relationship.

    This field only displays if "Parent-Child" has been selected under Category.
    Child

    The CMDB item type to which you want to link the Parent item in the Parent-Child relationship.

    This field only displays if "Parent-Child" has been selected under Category.
    Peer A

    The first CMDB Item Type in the Peer-to-Peer relationship.

    This field only displays if "Peer-to-Peer" has been selected under Category.
    Peer B

    The CMDB item type to which you want to link the Peer A item in the Peer-to-Peer relationship.

    This field only displays if "Peer-to-Peer" has been selected under Category.
    Principal

    The first CMDB Item Type in the Summary relationship.

    This field only displays if "Summary" has been selected under Category.
    Summarized

    The CMDB item type to which you want to link the Principal item in the Summary relationship.

    This field only displays if "Summary" has been selected under Category.
    Impacts

    This checkbox specifies whether CMDB items using this link type will impact the other linked CMDB items. The impact determines if an outage against this CMDB item will also cause an outage against the linked CMDB item.

    The label of these checkboxes depends on the type of category selected.

    Role A description for this component’s part in the relationship. This will be displayed on the CMDB Linking Diagram.

    This can be defined for Container, Component, Parent, Child, Peer, Principal and Summarized fields. The Item Types displayed depend on what was selected in the Category field.

    Link Styles Styles will be used on the CMDB Linking Diagram. If necessary, click on the title to expand the section.

    Specify the Line Width (in pixels) or thickness of the linking lines. The line width can be set to a number greater than 1 only when the line type is solid. For other line types, even if you set a thickness greater than 1, it will revert to 1 when you save the link type details.

    Link Styles is unavailable for Container-Component or Summary link types because CMDB items linked using one of these link types are represented by an icon rather than a configurable line on the CMDB Linking Diagram.

    Description Optional description of the Link Type
  3. Repeat for each link type you wish to create.

  4. Select to save the changes and close the window. Provide the Change Reasons if prompted to do so.

Updating a Link Type

Before you start

You must have CMDB Setup enabled within your General Access security role in order to configure any CMDB administration settings. Depending on if or how your system is partitioned, ensure that you are working in the correct partition.

  1. Select ≡ > Admin > System Administration.

    The System Administration window appears.

    In the Explorer pane, expand CMDB.

  2. Select CMDB Link Types to display the window. A browse table of all existing link types appears. You can adjust the column widths if required.
  1. Select the link type you want to edit.
  2. Update the CMDB Link Type Details as necessary. You may want to rename, assign a different line style, line color, or line size.

  3. Select to save the changes and close the window. Provide the Change Reasons if prompted to do so.

Deleting a Link Type

Before you start

You must have CMDB Setup enabled within your General Access security role in order to configure any CMDB administration settings. Depending on if or how your system is partitioned, ensure that you are working in the correct partition.

You cannot delete the default entries, Service Bundle to Service Action, Service Bundle to Service Action (optional), and Service to Service Action.

  1. Select ≡ > Admin > System Administration.

    The System Administration window appears.

    In the Explorer pane, expand CMDB.

  2. Select CMDB Link Types to display the window. A browse table of all existing link types appears. You can adjust the column widths if required.
  3. Select the link type entry you want to delete.
  4. Select . Select to save the changes. Provide the Change Reasons if prompted to do so.

Restoring a Deleted Link Type

Before you start

You must have CMDB Setup enabled within your General Access security role in order to configure any CMDB administration settings. Depending on if or how your system is partitioned, ensure that you are working in the correct partition.

  1. Select ≡ > Admin > System Administration.

    The System Administration window appears.

    In the Explorer pane, expand CMDB.

  2. Select CMDB Link Types to display the window. A browse table of all existing link types appears. You can adjust the column widths if required.
  3. Select Show Deleted to display the deleted link types as well as the active ones.
  4. Select the deleted entry you want to restore.
  5. Select . Deselect the Show Deleted checkbox to see the link type in the active list.
