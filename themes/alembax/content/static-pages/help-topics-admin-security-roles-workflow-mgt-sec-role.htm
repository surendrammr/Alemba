Workflow Management Security Roles

You can use the Workflow Management security role to assign and restrict permissions for the workflow (change, release, configuration management) request processes in ASM Core.

Before you start

You must have Security Roles setup selected in the Admin tab of your own General Access security role before you can assign or remove permissions for any security roles.

  1. Select â‰¡ > Admin > System Administration.

    The System Administration window appears.

    In the Explorer pane, expand Security Roles, then select Workflow Management. All existing Workflow Management security roles are displayed.

    2. The pre-defined role All typically has all role options selected. However, some of the options are not selected by default and you should explore the permissions set before assigning the role.

  2. If you want to create a role, select and assign the role a name in the Role Name field. To view or update a role, select the role in the browse table and select .
  3. The permissions for Workflow Management security roles are available on several tabs. Select the appropriate one for your needs:
Shared To set permissions relating to workflow Analysts, recipients, budgets and monitors, as well as options relating to Change Calendars and CMDB transactions
Requests To set permissions relating to request management, such as creating, viewing and editing requests and objects attached to them. This tab includes permissions that should be given to Request Managers
Tasks To set permissions relating to managing tasks and approvals, such as creating, viewing and editing tasks and approvals, and objects attached to them
Processes To enable access to workflow processes such as change and release management
Groups To assign workflow groups to this role and set the default group. Any analyst with this security role is considered a member of the group or groups linked to this role
Req. Screen Sets To select the request windows to which the role has access
Task Screen Sets To select the task windows to which the role has access
Appr. Screen Sets To select the approval windows to which the role has access
User Appr. Screen Sets To select the User approval windows to which the role has access
Ext. Supplier Screen Sets To select the external supplier task windows to which the role has access
Closure Task Screen Sets To select the closure task windows to which the role has access
Forms To control access to forms and assign access levels to the forms
Template To assign access to workflow templates the role can use for creating requests.

If the role has Template Admin permissions, Analysts will also be able to edit the templates to which they have access.

Request Quick Launch To set access to quick launch applications the role can access when taking action on Requests.

This tab is only enabled if Quick Launch is selected in the System Settings window.
Task Quick Launch To set access to quick launch applications the role can access when taking action on Tasks.

This tab is only enabled if Quick Launch is selected in the System Settings window.
