Configuring external access to the Dashboard Server

You access the Dashboard Server and User Management Server through URLs. If you want users to have external access to the servers, you must set up the external location on both the Dashboard Server and the User Management Server.

You can use either the http or https protocol to access the servers. For more information, see Configuring the web protocol.

Ensure that you follow the order outlined below so that you can continue to access the Dashboard Server and User Management Server while you update the URLs.

To configure external access to the servers:

  1. Using User Management Server, in the left-hand menu bar, select to open Application Management.
  2. To the right of the Company Name Dashboard Server application, select > Edit.
  3. Alongside Application URL, select Add Url.

  4. Ensure you are using the correct protocol. If required, select the http dropdown list to change the protocol to https.

    For more information, see Configuring the web protocol.

  5. Type the server name or IP Address and path of the Company Name Dashboard Server.

    For example: 192.0.2.0/AlembaDashboardServer

  6. Select Update to apply your changes.
  7. In a separate browser tab, using Dashboard Server, in the left-hand menu bar, select to open Settings.

  8. On the Site tab, change the Site URL to the external URL you added to the User Management Server.

    For example: http://192.0.2.0/AlembaDashboardServer

  9. Select Save.

    The Dashboard Server reloads with the updated URL.

  10. Using User Management Server, in the left-hand menu bar, select to open Settings.

  11. On the Site tab, change the Site URL to the external URL.

    For example 192.0.2.0/AlembaDashboardServer

  12. Select Save.

    The User Management Server reloads with the updated URL.

  13. Using Dashboard Server, in the left-hand menu bar, select to open Settings.

  14. On the User Directory tab, edit the UMS URL to include your chosen protocol and the IP Address of your external server.

    For example: http://192.0.2.0/AlembaUMS

  15. Select Test Connection to ensure that you can connect to the UMS URL.
  16. Select Save to save any changes.

What's next?

If you want to enable User Management Server for Active Directory, see Enabling User Management Server for Active Directory authentication.
