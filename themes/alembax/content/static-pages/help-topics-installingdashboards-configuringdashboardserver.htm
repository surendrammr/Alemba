Configuring the Dashboard Server

When you first launch the Dashboard Server, you must connect to the User Management Server, then configure the database and Dashboards system administrator account.

To configure the Dashboard Server:

  1. Navigate to the following location in your browser: http://localhost/AlembaDashboardServer

  2. On the Welcome to Syncfusion Dashboard Server page, type the URL for the User Management Server: localhost/AlembaUMS

  3. Select Proceed.

  4. On the Welcome to Syncfusion User Management Server page, select SQL Server from the drop-down list.

  5. Type in the server name.
  6. Type in your SQL Server administrator credentials.

  7. If you are using a separate database for the Dashboard and Report Platform, select New Database.

    You must give your database a suitable name.

  8. If you selected New Database, type AlembaDashboards.

  9. If you are using an existing database, select Existing Database, then select the database name in the drop-down list.

    We recommend that the dashboards have a dedicated database. Do not use the ASM database for dashboards.

  10. Select Proceed.

  11. Select the File Storage option button.

  12. Select Proceed.

  13. Create a Dashboards system administrator account, for example:

    Username CompanyNameAdmin
    First Name CompanyName
    Surname Admin
    Email admin@CompanyName.com
    Password

    ******

    Store this password in a password repository. If you forget the password, you will have to reinstall and reconfigure your system.

  14. Select Proceed to create the Dashboards system administrator account and register the Dashboard Server in the User Management Server.

    You are redirected to the Dashboard Server.

  15. Select the Use same database credentials of UMSÂ check box.

    The User Management Server database information prepopulates the fields.

  16. Select the Existing Database option button.

  17. Select the database you created for the User Management Server in the drop-down list.

    For example, select AlembaDashboards.

  18. If required, edit the prefix that distinguishes the Dashboard tables.

    In this example, the prefix is SyncDS_.

  19. Select Proceed.
  20. Select the File Storage option button.

  21. Select Proceed.

    The Dashboard Server login page appears.

  22. Use the Dashboards system administrator credentials you created to log into the Dashboard Server.

    For example, AlembaAdmin.
