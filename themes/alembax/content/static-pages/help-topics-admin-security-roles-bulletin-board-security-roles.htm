Bulletin board security roles

You can use the Bulletin Board Security role to assign and restrict permissions for viewing, creating and editing Bulletins.

Before you start

You must have Security Roles setup selected in the Admin tab of your own General Access security role before you can assign or remove permissions for any security roles.

  1. Select ≡ > Admin > System Administration.

    The System Administration window appears.

    In the Explorer pane, expand Security Roles, then select the type of role you want to view.

    A browse table appears, displaying all of the existing roles of this type. You can create, update or rename a security role of this type.

    The pre-defined role All typically has all role options selected. However, some of the options are not selected by default and you should explore the permissions set before assigning the role.

  2. If you want to create a role, select and assign the role a name in the Role Name field. To view or update a role, select the role in the browse table and select .
  3. The permissions for Workflow Management security roles are available on several tabs. Select the appropriate one for your needs:
  4. Select the levels of permission you want to assign to the role:
Read

Analysts with this role can view existing bulletins, enabling the Bulletin Board option on the Admin menu.
Create/Edit Own

Analysts with this role can create bulletins as well as publish a call to the Bulletin Board and edit any messages they have posted.

Select it to enable the Bulletin option in the Bulletin Board menu, and the Clone to Bulletin option from the Call Details window.

Read must be selected to enable this option.
Create/Edit All

Analysts with this role can create bulletins and edit messages they and other Analysts have posted.

Create/Edit Own must be selected to enable this option.
Publish to Self Service Portal

Analysts with this role can publish the current bulletin to the Self Service Portal. If this option is cleared, the Self Service Portal and Ext Supplier Portal options on the Bulletin Details window will be cleared and disabled.

Create/Edit Own must be selected to enable this option.
Cancel Lock

Analysts with this role can cancel the lock placed on an object attached to a bulletin that another Analyst has open for editing. If an Analyst has Cancel Lock rights, a button on the dialog gives them the ability to override the other Analyst and put the object into Edit mode. If that happens, and the first Analyst who had the object open attempts to save changes to that object, the item will go into Edit mode and a warning message appears on the Details window: “Locked by Lastname, Firstname”.

Create/Edit Own must be selected to enable this option.
Object View

Analysts with this role can download and view an object attached to a bulletin.
Object Update

Analysts with this role can update an object attached to a bulletin.

Object View must be selected to enable this option.
Object Delete

Analysts with this role can delete an object attached to a bulletin.

Object Update must be selected to enable this option.
Object Cancel

Analysts with this role can cancel the lock placed on an object attached to a bulletin that another Analyst has checked out.

Bulletin Object Version Control must be selected in the Bulletin Board Settings to enable this option.

  1. Select to save the changes and close the window. Provide the Change Reasons if prompted to do so.
