Security Settings (Partitioned)

You can define the call template to be used for the automatic call that is logged when the account for an Analyst or User has been disabled due to failed logins. The template selection is based on the current partition and different templates can be defined per partition. For more information see Call Template Administration.

IPK Workflow Rules picked up will override the assigned Analyst or group specified on the call template.

Assigning Call Templates for Disabled Accounts

Before you start

You must have Security Setup selected in the Admin tab of your General Access security role to access the Security options within the System window.

  1. Select ≡ > Admin > System Administration. The System Administration window is displayed. In the Explorer pane, expand Security.
  2. Select Security Settings (Partitioned). The Security Settings (Partitioned) window appears.
  3. If your system is partitioned, select the Partition for which you want to assign call templates from the drop down list.
  4. Complete the details:
Account Disabled Call – Full

Using the drop-down list, select the call template you want to use when a Person record is disabled due to failed log in attempts into ASM Core.
Account Disabled Call – Self Service Portal

Using the drop-down list, select the call template you want to use when a Person record is disabled due to failed log in attempts into Self Service Portal.
Account Disabled Call – Supplier Portal

Using the drop-down list, select the call template you want to use when a Person record is disabled due to failed log in attempts into ASM External Supplier Portal.
  1. Select to save the changes. Provide the Change Reasons if prompted to do so.
