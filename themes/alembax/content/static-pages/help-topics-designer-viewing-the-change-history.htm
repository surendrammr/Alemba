Viewing Designer Change History

You can view a history of the changes made in ASM Designer.

Details shown in the history include the name of the Analyst who made the change, the date and time of the change, and the type of action which led to the change.

The following is a list of actions recorded in the history:

  • Load from file and Saved: The screen was loaded from an external file and saved in ASM Designer.
  • Save to file: The screen was saved to an external file.
  • Saved: The screen was edited and saved in ASM Designer.

If auditing is turned on (that is if Designer is selected in the Auditing window), the change reason also appears in the history. In this case, you can select the history entry to expand it and view the change reason.

  1. Display the ASM Designer window if it is not already visible on screen.

    Select ≡ > Admin > Designer.

    This window consists of three tabs that display different views:
    Find a Screen Can be used to search for and work with specific screens, forms or message templates, as well as create new in-house screens
    View screens Displays details of currently selected screens and is used to modify screens
    Skins Enables you to create and modify skins for your Self Service Portal(s)
  2. There are two ways to open the history of a screen from here:
    • From the Find a Screen view, find a screen and select View History.
    • From the View Screens view, ensure you have a screen open and select Open History. The information messages which appear in the left panel of the View Screens tab are purely informational and can be ignored for the purposes of viewing the history of a screen. These are the standard information messages which display if no screens are open in this view or you are viewing the history of a screen.
  3. The history details appear in reverse chronological order with the most recent entry at the top. If the screen has been previously loaded from an external HTML file and then saved, a warning appears stating “This screen has been loaded from an external file. Any modification made outside of ASM Designer may result in loss or undesirable changes to functionality."
  4. From here, you can do any of the following:

    5. Open a screen Select Open Screen. The screen opens in a separate window in the View Screens view.
    Export the history to Excel Select Export History. In the Windows Save As dialog box, save the history.

    If auditing is turned on, you will also see a Change Reasons window. You may need to expand the change heading to see the reason for change.

  5. Select Close. You are returned to one of the other open tabs in the View Screens view.
