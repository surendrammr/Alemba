License Corrals

Corralling allows you to reserve a subgroup of licenses for specific group members, to ensure that they can log in, even if all of the other available licenses are in use.

When all the unrestricted licenses have been used, the corralled licenses will be reserved for the specified subgroup of Analysts.

Say, for example, your organization has many analysts using ASM Core but you have a service desk in which at least two Analysts must always have access. With corralled licenses it is possible to reserve two licenses to be shared amongst the service desk only.

Corralled licenses do not restrict Analysts named in that corral when there are other licenses still available. They are only reserved when the remaining general licenses have been used.

Defining License Corrals

Before you start

You must have System Set Up selected in the Admin tab of your General Access security role.

Your system must have either a concurrent Analyst license, or a named Analyst license with a number of additional Analysts defined as concurrent Analysts to enable the License Corrals option.

  1. Select ≡ > Admin > System Administration. The System Administration window is displayed. In the Explorer pane, expand Security.
  2. Select the License Corrals option from the Explorer pane. The License Corrals window appears. The top part of the window shows the concurrent licenses available:

    3. Maximum Concurrent Licenses The total number of concurrent licenses available for your system. This is a read only field
    Default The current number of licenses assigned to the default corral. This is a read only field and the value will dynamically increase and decrease depending on the number of licenses allocated to non-default corrals.

    When the sum of the licenses assigned to all Analyst-defined corrals matches the maximum number of concurrent licenses available for the system, the following warning message appears on this window:

    This means, any Analysts that belong to the Default corral will be unable to login to the system.

    The number displayed will be based on your system’s configuration.

 

Beneath this there is a browse table listing the existing corrals.

  1. Select .
  2. Complete the details:

    3. Corral The name or description of the new license corral you want to define. This will appear in the License Corral drop down in the Person Details.
    Allocated The number of licenses you want to assign to the defined license corral
    Corral Timeout Select to specify a period after which inactive sessions within the corral will be timed out.

    If you do not specify a corral timeout, the default session timeout defined in the Server Console will be used.
    Timeout Period (hh:mm) The timeout period in hours and minutes
  3. Repeat for each license corral you want to create.
  4. Select to save the changes. Provide the Change Reasons if prompted to do so.. The Default field is automatically updated to display the number of licenses remaining.
  5. Now you can assign Analysts to the relevant corrals in their Person Details

Modifying Details of a License Corral

You can rename corrals, reallocate license numbers and alter or remove the timeout period.

Before you start

You must have System Set Up selected in the Admin tab of your General Access security role.

  1. Select ≡ > Admin > System Administration. The System Administration window is displayed. In the Explorer pane, expand Security.
  2. Select the License Corrals option from the Explorer pane. The License Corrals window appears. The top part of the window shows the concurrent licenses available. Below this is a browse table listing all of the current corrals.
  3. Highlight the information in the corral you wish to update.
  4. Update the details by overwriting with new information.
  5. Select to save the changes. Provide the Change Reasons if prompted to do so.If you have amended the number of allocated licenses for a corral, the Default field is automatically updated to display the number of licenses remaining.

Deleting a License Corral

Before you start

You must have System Set Up selected in the Admin tab of your General Access security role.

You should remove the Analysts from the corral in their Person Details, reassigning them as needed.

You cannot restore a corral once you have deleted it. You must recreate it if necessary. Deleted corrals will appear in linked analyst details with an asterisk beside the name.

  1. Select ≡ > Admin > System Administration. The System Administration window is displayed. In the Explorer pane, expand Security.
  2. Select the License Corrals option from the Explorer pane. The License Corrals window appears. The top part of the window shows the concurrent licenses available. Beneath this is a list of existing corrals.
  3. Highlight the corral you wish to delete.
  4. Select in the toolbar. The entry is deleted. The Default field is automatically updated to display the number of licenses remaining.
