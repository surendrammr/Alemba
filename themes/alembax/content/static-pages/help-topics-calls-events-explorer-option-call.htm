Events Explorer Option - Call

When a call is logged by an incoming event (such as a network issue), you can view the event transactions exchanged between ASM and the system in which the event originated.

  1. Display the Call Details window, if it is not already on screen.
  2. Select the Events explorer option, expanding the Data group if necessary.
  3. The External Events window appears. Select an option from the event navigator (to the left of the window):
    • Root node
    • Transaction History
    • Transactions
    • Transaction Audit Trail
  4. View the event information in the browse table (to the right of the window). The table displays the following information about each event transaction:
    • Date/Time: The date and time the transaction was sent.
    • Origin: The name of the system that sent the transaction (either ASM or the external system)
    • Transaction Type: The type of transaction sent (such as ACKNOWLEDGE, UPDATE, NOTIFY)
    • Details: The content of the transaction, if any.
    • Status: The status of the transaction, that is, Succeeded or Failed.
