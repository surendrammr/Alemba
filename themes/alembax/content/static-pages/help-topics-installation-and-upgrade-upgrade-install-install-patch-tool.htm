Installing the Patch Tool

Before you can run an upgrade, you have to download the Alemba Service Manager™ Patch file and install it on your system.

Before you start

Ensure your system meets the minimum technical requirements and is configured correctly. For more information, see Prerequisites checklist.

  1. Log in to the Alemba® portal and browse the Service Request Catalog. Order the items from the catalog.
  2. When you receive an email, use the link to access the portal download page.
  3. Download the Alemba Service Manager xx Patch file (where xx is the appropriate version number).
  4. Extract the contents.
  5. Copy the Alemba Service Manager xx Patch.msi file to a temporary location, such as your desktop, and double-click on it to start the Patch Installer wizard.
  6. The License Agreement window appears. Read the details and accept the terms, before selecting Next to continue.
  7. On the User Information window, enter your name and organization.
  8. Select Next to continue.
  9. On the Setup Type window, you can select the type of setup you want:
    • Select Complete to install the upgrade files in the installation directory. The directory will depend on server configuration and which version was previously installed. An example of an old installation directory would be C:\Program Files (x86)\VMware\Service Manager.
    • Select Custom to change the directory in which you wish to install the upgrade files.
  10. Select Next to continue.
  11. In the Ready to Install the Program window, select Install.
  12. If you receive a confirmation box, select Yes to continue.
  13. Select Finish when the installation is complete.

You can now perform a screen pre-upgrade, as explained in Screens Pre-Upgrade, or the full upgrade, as explained in Running the ASM Patch Tool.

Oracle databases are not supported.
