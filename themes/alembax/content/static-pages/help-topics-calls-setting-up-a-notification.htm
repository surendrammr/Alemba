Setting Up a Notification Workflow

The IPK Notification Workflow Setup window opens with the Workflow Activity field set to Notification.

  1. Complete the details.

    2. Title Type a name for the workflow you are creating
    Workflow Description Type a brief description of this workflow.
  2. You can now select the analysts or groups to whom you want to send a notification.

  3. If your system is partitioned, you can only add analysts or groups within the partitions to which you belong. Analysts may be able to see other analysts/groups on the workflow that do not belong to any of their partitions, for example when another analyst in a different partition has added these analysts/groups to the workflow.

  4. Select the tab for the activity at which you want the notification to be sent:

    5. Notify on Log/Update

    Set up the notification to be sent when the call is logged, or updated to meet the criteria.

    Notify on Log/Update occurs only once per IPK rule per call; notifications are not sent each time the call is updated.
    In order to send a notification every time the call is updated, create a Forum.
    Notify on Resolve Set up the notification to be sent when the call enters the Resolve phase. This is when the call will no longer be updated and is of state Open-Resolve or Open-Unresolve when there are closure groups set up in the system or Close-Resolve or Close-Unresolve when there is no closure groups set up in the system.
    Notify on Close Set up the notification to be sent when the call is closed.
  5. In each tab you want to set up notifications, select the checkbox to Notify on.... This will enable the Add Recipient and Remove Recipient buttons.
  6. To select the recipient(s) for notification, click Add Recipient. Find the recipients.
  7. All of the identified recipients are displayed in the browse table. Complete the remaining columns for each recipient.

    8. Email

    This column is only displayed if Email is selected in the Messaging window.

    Select the checkbox in the column to send the notification by email
    Email Recipient Type

    This column is only displayed if Email is selected in the Messaging window.

    Use the drop-down list to choose between TO, CC and BCC
    Pager

    This column is only displayed if Pager is selected in the Messaging window.

    Select the checkbox to send the notification to a pager
    Internal Message

    This column is only displayed if Internal is selected in the Messaging window.

    Select the checkbox to send an internal system message for notification
    Template

    This column is only displayed if Email is selected in the Messaging window.

    Use the drop-down list to select the email template to be used for the required recipient analyst or group. If the template you want to set for notification is not displayed in this list, ensure that your system administrator has added the template using the Message Types window. The required template must be added to the IPK Workflow â€“ Notification message type (message code 12000).

    If your system is partitioned, you can use a different email template for each partition using IPK Workflow Rules. To specify a different template for different partitions, create a new workflow rule and link the notification workflow to the rule. In essence, you are setting up a condition for a particular template to be used for notification based on the partition selected in your rule.

    For example, if partition=West, use email template User Email Template for your notification workflow.

    Pager Template

    This column is only displayed if Pager is selected in the Messaging window.

    Use the drop-down list to select the pager template to be used.
  8. Click . The IPK Notification Workflow window appears with the workflow you just created listed in the browse table.

    9. If a Notification workflow is run on call logging or update, the specified analysts and groups will receive notification that the call has been logged or updated. In the case of a call resolution, the specified analysts and groups will receive notification that the call has been resolved. If notification is set to occur on call closure, the specified analysts and groups will receive notification that the call has been closed.

  9. To view the details of a workflow, simply select it in the browse table and click .
  10. To delete a workflow, select it in the browse table and click .
