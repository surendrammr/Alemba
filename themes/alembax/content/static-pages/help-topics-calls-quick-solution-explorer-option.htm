Quick Solution Explorer Option

Quick Solutions are defined by the system administrator for a type of call that is logged often, and for which you enter similar details every time. You can then to select one from within the Call Details window.

Quick Solutions must be selected in the IPK Settings (Partitioned) window.

  1. Display the Call Details window, if it is not already on screen.
  2. In the Explorer pane, expand Resolution, then select Quick Solution.
  3. If you selected a Type on the Call Details window which has a Quick Solution already defined for it, the Quick Solution explorer option will display an orange box next to it indicating you can click it to access the Quick Solution for that type. If a Quick Solution is not defined for this type, click the Quick Solution explorer option to view a list of all Quick Solutions.
  4. If only one quick solution has been defined, it will run immediately.
  5. If multiple quick solutions apply or no quick solutions apply, the Quick Solutions popup window appears. Select the quick solution and then click OK.
  6. A new Call Details window is created, and details are automatically entered into the following fields:
    • Description
    • Type
    • Reason
    • Actions and Solutions.

If any information exists on the Call Details window before you select the Quick Solutions explorer option, the ‘quick’ details will be added to the existing information on the new call.

If Disable Checks is selected in the IPK Settings (Partitioned) window, a call to which a quick solution has been applied can be closed immediately, bypassing the validation checks, such as Customer Required, CI Required, Service Required and Type Required. This means that even if mandatory fields are left incomplete, the call can still be closed.
