Scripting

You can create call scripts that Analysts can use when responding to a User’s call. A script can provide a step-by-step approach to help an analyst resolve a call. Scripts can also be made available on the Self Service Portal through the Self Help menu.

Before you start

Configured must be selected under Scripting Options in the IPK Settings window. This also enables the Scripting explorer option on the Call Details window. Select the Type option in the same window if you want scripts to be linked to issue types.

You must have IPK Setup enabled in the Admin tab of your General Access Security Role.

You must have Script Admin selected in the Options tab of your IPK Management security role.

Creating a Script

Before you start

Configured must be selected under Scripting Options in the IPK Settings window. This also enables the Scripting explorer option on the Call Details window. Select the Type option in the same window if you want scripts to be linked to issue types.

You must have IPK Setup enabled in the Admin tab of your General Access Security Role.

You must have Script Admin selected in the Options tab of your IPK Management security role.

  1. Select ≡ > Admin > System Administration.

    The System Administration window appears.

    In the Explorer pane, expand IPK Management.

  2. Select Scripting to open the window. A browse table displays the existing call scripts, with checkboxes indicating whether they are available in the full application and/or the Self Service Portal. You can adjust the column widths if required.
  1. Select . A row is added to the browse table.
  2. In the Script column, type the name of the script you want to create into the field.
  3. To enable Analysts to access the script in ASM Core, select the checkbox in the Full Application column.
  4. To enable Users to view the scripts on the Self Service Portal, select the checkbox in Self Service Portal column.
  5. Select to add the Script details.
  6. The script’s name displays in the Title field and cannot be edited in this window.
  7. From the Script box, showing the layout of the script as a tree with different options, select Add to add a new option. This opens the New Option window.
  8. Type a title for the script option. Select Ok. The option displays in the script tree layout. (If you want to remove the option from the script, select the option in the Script box and select Remove .)
  9. If the selected option is a prompt, that is, a question the Analyst is likely to ask the User, type it in the Prompt field (for example: “Is it a problem with Word?”).
  10. If the option is an answer to a question, that is, a piece of advice for dealing with a particular issue, type the advice you would like to use in the Advice field.
  11. Repeat these steps to create another option. 
  12. To attach a file to the script or the currently selection option in the script, select Attach Object and upload an object.
  13. At Linked Types, select to choose an issue type. In the multi-tier type list, double-click the issue type to select it.
  14. Select Add to link the current script to the issue type. To remove an issue type from a script, select the issue type and select Remove.

    15. The type you link to a script is only used to filter scripts when an Analyst selects a call type in the Call Details window within the main application. If a User logs a call from a script through the Self Service Portal, the linked type is not allocated to the call.

  15. Select to save the changes. Provide the Change Reasons if prompted to do so.

Once you select , the browse table displays the entries in alphabetical order.

Renaming a script

Before you start

Configured must be selected under Scripting Options in the IPK Settings window. This also enables the Scripting explorer option on the Call Details window. Select the Type option in the same window if you want scripts to be linked to issue types.

You must have IPK Setup enabled in the Admin tab of your General Access Security Role.

You must have Script Admin selected in the Options tab of your IPK Management security role.

  1. Select ≡ > Admin > System Administration.

    The System Administration window appears.

    In the Explorer pane, expand IPK Management.

  2. Select Scripting to open the window. A browse table displays the existing call scripts, with checkboxes indicating whether they are available in the full application and/or the Self Service Portal. You can adjust the column widths if required.
  1. In the Script column, select the name of the script you want to rename.
  2. Type the new name for the script over the current name.
  3. Select to save the changes. Provide the Change Reasons if prompted to do so.
