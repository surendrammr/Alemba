Task Action Types

Task Action Types are used to break a task down into its separate activities, such as Evaluating, Specifying, or Writing Report. These are selected by an Analyst when adding an item to the task.

Creating a Task Action Type

Before you start

You must have Workflow Management Set Up enabled within your General Access Security Role to work with task action types.

  1. Select ≡ > Admin > System Administration. The System Administration window appears. In the Explorer pane, expand Workflow Management.
  2. Select the Task Action Types option. The Task Action Types window appears, displaying existing task action types .
  3. Select . A blank row is inserted in the browse table.
  4. Type a name for the task action type you want to create.
  5. Select to save the changes. Provide the Change Reasons if prompted to do so.

Updating a Task Action Type

Before you start

You must have Workflow Management Set Up enabled within your General Access Security Role to work with task action types.

  1. Select ≡ > Admin > System Administration. The System Administration window appears. In the Explorer pane, expand Workflow Management.
  2. Select the Task Action Types option. The Task Action Types window appears, displaying existing task action types .
  3. Select the task action type you wish to update and overtype the existing name with the new one.
  4. Select to save the changes. Provide the Change Reasons if prompted to do so.

Deleting a Task Action Type

Before you start

You must have Workflow Management Set Up enabled within your General Access Security Role to work with task action types. If you delete an action type currently associated with a task, the task will revert to the system default task type of "unspecified".

You cannot restore a deleted task action type; you would have to re-create it if needed.

  1. Select ≡ > Admin > System Administration. The System Administration window appears. In the Explorer pane, expand Workflow Management.
  2. Select the Task Action Types option. The Task Action Types window appears, displaying existing task action types .
  3. Select the action type you want to delete. Select . If you attempt to delete the system default, you will receive a warning message.
  4. Select to save the changes. Provide the Change Reasons if prompted to do so.
