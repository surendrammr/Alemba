Service Level Management Security Role: Options Tab

You can set security permissions relating to agreements for the SLM Security Role.

Before you start

You must have Security Roles setup selected in the Admin tab of your own General Access security role before you can assign or remove permissions for any security roles.

  1. Display the SLM Security Roles details window, if it is not already on screen.

    Select ≡ > Admin > System Administration.

    The System Administration window appears.

    In the Explorer pane, expand Security Roles then select Service Level Management. All existing SLM security roles are displayed.

    The pre-defined role All typically has all role options selected. However, some of the options are not selected by default and you should explore the permissions set before assigning the role.

    If you want to create a role, select and assign the role a name in the Role Name field. To view or update a role, select the role in the browse table and select .

  2. On the Options tab, choose the settings you wish to assign to the security role:
Agreement Read

Analysts with this role can search for and view agreements
Agreement New

Analysts with this role can create an agreement.

Agreement Read must be selected to enable this option.
Agreement Update

Analysts with this role can edit and update agreements created by other Analysts.

Agreement Read must be selected to enable this option.
Agreement Delete

Analysts with this role can delete existing agreements.

Agreement Read must be selected to enable this option.
Agreement Cancel Lock

When not selected, this option prevents two Analysts updating an agreement at the same time by locking the agreement. To disable the lock, this option can be selected

Agreement Update must be selected to enable this option.
Agreement Action by Default

Analysts with this role can open agreements in Edit mode directly; they do not need to use the Edit button.

Agreement Update must be selected to enable this option.
Monitor

Analysts with this role can create monitors for SLM, providing real time graphing of SLM statistics and information.

This option does not give or deny access to the overall monitor application, just the part relating to agreements

Agreement Object View

Analysts with this role can download and view objects attached to an agreement
Agreement Object Update

Analysts with this role can modify a file attached to an agreement.

If you have Advanced Object Management on your system, you will need to check the object out and back in

Agreement Object View must be selected to enable this option.
Agreement Object Cancel

Analysts with this role can cancel and ‘override’ a check out and restore the object to its last checked-in state.

Agreement Object View must be selected to enable this option.
Agreement Object Delete

Analysts with this role can delete an attached object.

Agreement Object Version Control must be selected in the Service Level Management Settings to enable this option.
Agreement Stakeholder Read Analysts with this role can view stakeholders linked to agreements.
Agreement Stakeholder Write

Analysts with this role can edit the details of stakeholders linked to agreements.

Agreement Stakeholder Read must be selected to enable this option.
  1. Select to save the changes and close the window. Provide the Change Reasons if prompted to do so. Alternatively, select another tab, if appropriate.
