Creating dashboards using the designers

You can create a dashboard using either the web designer or the Alemba Dashboard Designer.

You need to define an Alemba data source for the dashboard in order to access your ASM database. You can either create a new data source, or use an existing one. For more information about data sources, see ???

Dashboards contain one or more widgets – a collection of mini-reports that present data in different styles, including simple numeric metrics, tables and charts. You define the widget parameters within the dashboard where they are displayed. For information about the different widgets, see ???.

Using web designer to create dashboards

To create a new dashboard using web designer:

  1. Using Dashboard Server, at the top-right corner, select > Create > Dashboard.

    A new untitled dashboard appears.

  2. Select (IMAGE) to configure a data source.

    The Data Sources panel appears.

  3. Select Create New to add a new data source.

    For information about using an existing data source, see ???.

    The Connection Type panel appears. Four data connection types are available: Alemba; Amazon Redshift; Microsoft SQL; and PostgreSQL. This procedure explains how to set up an Alemba connection type.

  4. Select to set up an Alemba connection.

    The New Data Source panel appears. If you selected a different connection type, different connection options become available.

  5. Complete the connection details, including a data source Name that will identify the data source within the dashboard designer.

    If you are connecting to the vFire data source, you need the Alemba API Client ID. For more information, see Configuring the Alemba API connection in the Alemba Service Manager™ 9.12 Prerequisites and Installation Guide.

  6. Select Connect.

    The data design view appears.

  7. Expand the tree view to see the table schema.

  8. Drag one or more tables into the data design view to set the data parameters for the dashboard.

    (IMAGE)

    Note the available information in the Data Preview section beneath the data design view. You can filter this information, change the data settings (such as column names), and hide columns without affecting the underlying data.

    If you drag more than one table into the data design view, the join editor appears to define the relationship between the tables.

  9. Select Save to save your changes.

  10. Drag the relevant widgets onto the dashboard.

    You can change the size and position of the widgets. For more information, see Changing the size and position of a widget on a dashboard.

  11. At the top-right corner of each widget, select to configure the widget Settings.

    For more information about assigning data to the widget, see ???.

    For more information about the widget properties, see ???

  12. Select > Save to save the dashboard.

    The Save As Dashboard page appears.

  13. Select which Category you want to assign to the dashboard.

    For information about categories, see ???.

  14. Type a suitable Name for the dashboard.
  15. Type a short Description for the dashboard.

  16. Select a Privacy Settings option:

    Private displays the dashboard to those users with the appropriate permission. You must sign in to see private dashboards.

    Public displays the dashboard to anyone with a link to them. No sign-in is required.

    Unlisted displays the dashboard to anyone with a link to them. No sign-in is required. Only dashboard owners can see unlisted dashboards in their dashboards list.

  17. Select Save to save the dashboard.

    A confirmation box appears.

  18. Select either Yes to preview the dashboard, or No to return to editing the dashboard.

When the dashboard is complete, you must publish it so that you can use it from within Alemba Service Manager™. For more information, see Publishing a dashboard.

Using Alemba Dashboard Designer to create dashboards

To create a new dashboard using web designer:

  1. Launch Alemba Dashboard Designer.

    A new untitled dashboard appears.

  2. Select Add to add a new data source.

    For information about importing an existing data source, see ???.

  3. On the New Connection window, type a Data source name to identify the data source within the Dashboard.

  4. Select a Connection type from the drop-down list.

    Various data connections types are available, for example: Microsoft Excel; Microsoft SQL Server; JSON; Hive, and so on. Depending on your connection type, different connection options become available.

  5. Complete the connection details.
  6. For server-based data connection types, select Test Connection to validate the connection details.

  7. Select Connect.

    The data design view appears.

  8. Expand the tree view to see the table schema.

  9. Drag one or more tables into the data design view to set the data parameters for the dashboard.

    (IMAGE)

    Note the available information in the Data Preview section beneath the data design view. You can filter this information, change the data settings (such as column names), and hide columns without affecting the underlying data.

    If you drag more than one table into the data design view, the join editor appears to define the relationship between the tables.

  10. Select Save to save your changes.

  11. Drag the relevant widgets onto the dashboard.

    You can change the size and position of the widgets. For more information, see Changing the size and position of a widget on a dashboard.

  12. If required, you can add multiple tabs to the dashboard.

    For more information, see Adding multiple tabs to the dashboard.

  13. If required, you can increase the size of the dashboard.

    For more information, see Increasing the dashboard size.

  14. At the top-right corner of each widget, select to configure the widget Settings.

    Depending on your widget type, different Settings options are available.

    For more information about assigning data to the widget, see ???.

    For more information about the widget properties, see ???

  15. Select > Save to save the dashboard.

    The Save As Dashboard page appears.

  16. Select which Category you want to assign to the dashboard.

    For information about categories, see ???.

  17. Type a suitable Name for the dashboard.
  18. Type a short Description for the dashboard.

  19. Select a Privacy Settings option:

    Private displays the dashboard to those users with the appropriate permission. You must sign in to see private dashboards.

    Public displays the dashboard to anyone with a link to them. No sign-in is required.

    Unlisted displays the dashboard to anyone with a link to them. No sign-in is required. Only dashboard owners can see unlisted dashboards in their dashboards list.

  20. Select Save to save the dashboard.

    A confirmation box appears.

  21. Select either Yes to preview the dashboard, or No to return to editing the dashboard.

Adding multiple tabs to the dashboard

If you are using Alemba Dashboard Designer, you can add multiple tabs to a dashboard. While there is no limit to the number of tabs you can add, we recommend that you limit yourself to five tabs. A large number of tabs and widgets affects the dashboard performance.

To add a new tab to the dashboard:

  1. With the dashboard open, on the right-hand side in the DASHBOARDS panel, select (IMAGE +) to add a dashboard.

    A new dashboard tab appears. The multiple tabs are listed in the DASHBOARDS panel, with the active tab highlighted in blue.

  2. At the top left, select (IMAGE) to change the dashboard tab title.

  3. If required, import an existing dashboard.

    For more information about importing an existing dashboard, see ???.

Importing an existing dashboard

You can include an existing dashboard in your current dashboard. If the imported dashboard contains multiple tabs, each of the tabs is added to the current dashboard.

  1. With the dashboard open, on the right-hand side in the DASHBOARDS panel, select (IMAGE +) to import a dashboard.

    The Import Dashboard dialog box appears.

  2. Select where the file you want to import is stored.

    You can import from your local machine, a server, or the Recent dashboard list.

  3. If you import from your local machine, browse for and select the file, then select Open to import the file.

  4. If you import from a server, select Login to connect to the server using the Server URL and your credentials, or an external provider.

    Dashboards are listed according to category.

    1. Expand the category.
    2. Select the relevant dashboard.
    3. Select Import.
  5. If you import from the Recent list, select the dashboard check box, then select Import.

If the imported dashboard uses the same data source as the current dashboard, an alert appears to warn you that the data source already exists.

  1. Select Yes to use the existing data source.
  2. Select No to import the data source that the imported file uses.

Increasing the dashboard size

If you are using Alemba Dashboard Designer, you can add rows to a dashboard. There is no limit to the number of rows you can add. A scroll bar appears so that users can access the entire page.

To add rows to a dashboard:

  • Drag or resize widgets to the bottom of the last row.

    An extra row appears. (Do more than one row appear at a time?)

Changing the size and position of a widget on a dashboard

When you place a widget on the dashboard, the resize handles are visible:

  • Drag a resize handle to change the widget size.

  • Drag the selected widget to move it.

If you resize or move a widget, a blue outline shows that the widget will fit into the new space. A red outline shows that the widget will not fit the new space. You can only resize or move the widget into empty spaces on your dashboard.
