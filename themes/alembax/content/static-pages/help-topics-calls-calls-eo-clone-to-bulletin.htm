Clone to Bulletin Explorer Option

Analysts with the appropriate permissions can publish an existing ASM call automatically to the Bulletin Board.

You must have Create/Edit Own selected in your Bulletin Board security role.

  1. Display the Call Details window, if it is not already on screen.
  2. Expand the Resolution group, if necessary, and select the Clone to Bulletin option.
  3. Edit the bulletin details in the Bulletin window as needed.
  4. Expand the Details section to view the call details copied to the bulletin.

    5. The Title field is updated from the Call Number and Description fields in the format <Call Number>: <Description>. The body text of the Bulletin Board message appears as follows:

    An issue has occurred.

    Service: <value of call Service field>

    Config Item: <value of call Configuration Item field>

    Type: <value of call Type field>

    Priority: <value of call Priority field>

    Current Analyst: <value of call Current text field>

    Description

    <value of call Description field>

  5. Click Save to store the message.

When you clone the call into a Bulletin Board entry, the first line will say, “An issue has occurred.” This is by design, as calls relate to issues within the organization and any Bulletin Board entries based on a call should highlight that it is an issue.
