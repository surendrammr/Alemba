Knowledge Management Security Role: Options Tab

You can set permissions relating to the Knowledge Bank using the Options tab in the Knowledge Bank Security Role. This includes permissions for viewing, creating and updating knowledge bank entries.

Before you start

You must have Security Roles setup selected in the Admin tab of your own General Access security role before you can assign or remove permissions for any security roles.

  1. Display the Knowledge Management Security Role details window, if it is not already on screen.

    Select ≡ > Admin > System Administration.

    The System Administration window appears.

    In the Explorer pane, expand Security Roles, then select Knowledge Management. All existing Knowledge Management security roles are displayed.

    The pre-defined role All typically has all role options selected. However, some of the options are not selected by default and you should explore the permissions set before assigning the role.

    If you want to create a role, select and assign the role a name in the Role Name field. To view or update a role, select the role in the browse table and select .

  2. Select the Options tab .
  3. Choose the settings you wish to assign to the security role:
Search

Analysts with this role can search and view knowledge bank entries
Clone from Calls

Analysts with this role can create a knowledge entry from a call. 

Search must be selected to enable this option.

The Analyst does not need Author rights to do this. The knowledge entry is created from the call text immediately.
Author

Analysts with this role can create and update knowledge entries. They can also take over unclaimed entries which have been created from calls.

Search must be selected to enable this option.
Editor

Analysts with this role can update and delete other Analysts’ knowledge entries.

Author must be selected to enable this option.
Review

Analysts with this role can mark a knowledge entry as reviewed. They can also change the Next Review Date for entries provided they are not the contributor for those entries.

Editor must be selected to enable this option.
Review Own

Analysts with this role can mark knowledge entries they have authored or contributed to as reviewed.

Review must be selected to enable this option.
Cancel Lock

Analysts with this role can cancel the lock on a knowledge entry when another Analyst has it open for editing.

Author must be selected to enable this option.

Object View

Analysts with this role can read and download objects (files) attached to the knowledge entry.

Search must be selected to enable this option.
Object Update

Analysts with this role can update objects attached to the knowledge entry.

Author and Object View must be selected to enable this option.
Object Delete

Analysts with this role can delete objects attached to the knowledge entry.

Object Update must be selected to enable this option.
Object Cancel

Analysts with this role can cancel the lock placed on an object when another Analyst has it checked out.

Knowledge Object Version Control must be selected in Knowledge Bank Settings and Object View must be selected in the role to enable this option.
  1. Select to save the changes and close the window. Provide the Change Reasons if prompted to do so. Alternatively, select another tab, if appropriate.
