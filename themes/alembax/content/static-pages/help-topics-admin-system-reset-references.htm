Reset References

You can set the base numbers used in the Reference fields throughout ASM Core (such as the Call Ref field on the Call Details window). This is typically done by companies who are using two or more separate ASM Systems with databases which they may want to combine at a later date. In such cases, Site A could use reference numbers 0 – 5000 and Site B could use reference numbers 5001 - 10000, etc. This greatly facilitates the data conversion when the databases are merged.

Call Ref, Request Ref and CI Ref cannot be manually set, but are automatically incremented by one on performing a reset.

Before you start

You must have System Set Up selected in the Admin tab of your General Access security role.

  1. Select ≡ > Admin > System Administration. The System Administration window appears.
  2. In the Explorer pane, expand System, then select Reset References. The window displays all of the fields that can be reset. Scroll down if necessary to see the one you want.
  3. The Value to its right shows the current number being used. Select the Value you want to edit.
  4. Type the number to which you want that reference set. If you enter a value that is outside the acceptable range, you will receive an error message.

Resetting the Values in Reference Fields

Before you start

You must have System Set Up selected in the Admin tab of your General Access security role.

When you reset values, all of the values will reset. You cannot reset them individually.

  1. Select ≡ > Admin > System Administration. The System Administration window appears.
  2. In the Explorer pane, expand System, then select Reset References. The window displays all of the fields that can be reset. Scroll down if necessary to see the one you want.
  3. Select to reset the references to overwrite the value/s you have assigned.
  4. Select to save the changes. Provide the Change Reasons if prompted to do so.
