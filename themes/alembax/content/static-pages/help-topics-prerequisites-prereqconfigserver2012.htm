Configuring Windows Server 2012 R2 for Alemba Service Manager™

Before you start

Ensure that your server and client machines meet all the prerequisites. For more information, see Prerequisites checklist.

If you are using Server 2016, see Configuring Windows Server 2016 for Alemba Service Manager™.

Alemba® does not support Oracle databases for ASM v10.1.

To configure Windows Server 2012 R2 for ASM Core:

  1. Using Server Manager, select Dashboard, then select Add roles and features to start the Add Roles and Features Wizard.
  2. On the Before you begin page, ensure that you have completed the listed tasks.
  3. Select Next.
  4. On the Select installation type window, select Role-based or feature-based installation, then select Next.
  5. On the Select destination server window, select the server that you will be using, then select Next.
  6. On the Select server roles window expand Web Server(IIS) and then Web Server.
  7. Within Application Development, select Application Initialization and WebSocket Protocol.
  8. Select the Application Server and Web Server (IIS) check boxes as a minimum requirement.
  9. When you select Web Server (IIS), you are prompted to add Management Tools.
  10. Select Add Features to add the Tool and return to the Select server roles window.
  11. Select Next to continue.

  12. In the Features window, ensure that the following options are selected:

    • .NET Framework 3.5 Features
      • HTTP Activation
      • Non-HTTP Activation
    • .NET Framework 4.5 (or higher) Features
      • .NET Framework 4.5 (or higher)
      • ASP.NET 4.5 (or higher)
      • WCF Services
        • HTTP Activation
        • TCP Activation
      • TCP Port Sharing

You are notified if any additional features are required for HTTP Activation. Click Add Features to add the features and return to the Select Features window.

  1. When you have finished adding the Features, select Next
  2. to display the Application Server window
  3. Read through the details to ensure you are familiar with them, then select Next.

    4. You may not see this window if you are upgrading.

  4. On the Select role services window, select the following role services for the application server:

    • .NET server 4.5 (or higher)
    • Web Server (ISS) Support, which displays a popup requesting to add required features for Web Server (IIS).
  5. Select Add Features to add the features and return to the Select role services window.
  6. Select Next to continue.
  7. On the Web Server Role (IIS) window, read the information provided, then select Next.

  8. On the Select role services window, ensure that you have selected all of the following roles:

    • Web Server
      • Common HTTP Features
        • Default Document
        • Directory Browsing
        • HTTP Errors
        • Static Content
        • HTTP Redirection
      • Health and Diagnostics
        • HTTP Logging
        • Logging Tools
        • Request Monitor
      • Performance
        • Static Content Compression
      • Security
        • Request Filtering
        • Basic Authentication
        • Client Certificate Mapping Authentication
        • Digest Authentication
        • IIS Client Certificate Mapping Authentication
        • IP and Domain Restrictions
        • URL Authorization
        • Windows Authentication
      • Application Development
        • .NET Extensibility 3.5
        • .NET Extensibility 4.5 (or higher)
        • Application Initialization
        • ASP
        • ASP.NET 3.5
        • ASP.NET 4.5 (or higher)
        • ISAPI Extensions
        • ISAPI Filters
        • WebSocket Protocol
      • Management Tools
        • IIS Management Console
        • IIS Management Scripts and Tools

    9. Alemba Service Manager™ provides its own Dynamic Content Compression, which performs slightly better than IIS Dynamic Content Compression. You can choose to disable IIS Dynamic Content Compression.
    If you disable IIS Dynamic Content Compression, the Web Server (IIS) Support Role is also disabled. We therefore recommend you review the Role Services enabled by the Web Server (IIS) Support Role and manually re-enable those required for your environment.

  9. Select Next to continue.
  10. On the Confirm installation selections window, confirm your choices.
  11. Select Install.
  12. When installation completes, select Close to close the wizard.
