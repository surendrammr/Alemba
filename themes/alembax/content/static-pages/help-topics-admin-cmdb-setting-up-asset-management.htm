Setting Up Asset Management

System administrators can set up the asset management module in the following way.

  1. Configure the CMDB.
  2. Set up and define the CMDB Item Types provided for managing your asset CMDB items. These include:
  • Software Product (for adding your software licenses)
  • Inventory (for adding your bulk inventory items)
  • Packages (an abstract entity type you can use to group related Software Product and Inventory items, for example, for a software deployment)

When you are defining a CMDB item type, you can flag it as an asset, enabling analysts to search for CMDB items designated as ‘assets’.

  • Specify which information is mandatory on asset transactions. This is set up in the CMDB Settings.
  • Assign specific values or list options for use in Software Product Licenses, Inventory Details and Transaction Details using other options on the CMDB administration menu:
    • Asset Transaction Statuses
    • Asset Lifecycle Statuses
    • License Types
  • Assign analyst access to License or Inventory Transactions through the CMDB Items tab of their Configuration Management Security Role settings. This includes access to the transactions from calls, requests and tasks.

    • Analysts will be able to perform transactions (such as allocate an inventory item to a person/CMDB item) from a call, request, or task (for example, through a configuration item field) even if they do not have permission to view people/CMDB items in their security role.

  • Create Cost Centers against which analysts can make asset transactions.
  • Create Jurisdictions and allocate these to analysts to define which Cost Centers they can make asset transactions against.
