Creating an Assignment Workflow

The IPK Assignment Workflow window lists all the IPK Assignment Workflows that have been defined in ASM and enables you to create Assignment Workflows, that is, workflows that assign a call to an analyst or a group. You can create Assignment Workflows to be run when a call is logged or updated, and/or resolved.

Before you start

The following settings must be in place before you can build a rule:

  • IPK Workflow Rules Builder must be enabled in IPK Settings .
  • IPK Workflow must be selected in the IPK Management Role: Options Tab of your IPK Management Security Role.
  1. Select Menu and then Admin. From the submenu, select IPK Workflow Rules. The IPK Workflow Rules window appears. The explorer pane on the left of this window shows three groups: IPK Workflow Rules, Forum Management and Auto Close Configuration.
  2. From the IPK Workflow Rules group, select IPK Assignment Workflow.
  3. On the IPK Assignment Workflow window, click .
  4. In the IPK Assignment Workflow Setup window, set up the Assignment workflow.
