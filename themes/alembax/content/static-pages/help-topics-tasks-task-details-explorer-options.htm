Task Details Explorer Options

The Task Details explorer consists of the following groups and options within those groups:

Not all of the options are displayed for all tasks.

Within the Details group, you can select:

  • SLM to show any agreements that are currently in effect against the request and task, and select agreements (OLA and UC) for the current task
  • History to view the current task's history, such as, who created it, its description, or who owns the task
  • Display Request to view the Request Details window. You can update the information the request, if it is required
  • View Dependencies to view the workflow dependency diagram for the current request. The dependency diagram is opened in Review Only mode, and cannot be updated in this mode. You cannot add new tasks to, or delete existing tasks from the dependency diagram. However, you can double-click on a task and open its Details window, and update it if you own the task. You can also set display options for the dependency diagram.

Within the Data group, you can select:

  • Object to attach objects or files from Windows applications to the current task
  • Attributes to access the Task Attributes window
  • Forms to access the form attached to the task and specify the required details. Forms can be used to capture general, non-reportable, non-searchable information about the task.

    • This option is only enabled if Forms are enabled on the System Settings window. Your ability to access a particular form is set in the Forms tab of your Workflow Management Security Role.

  • Events. If the current task is linked to a request that was logged as a result of an incoming event (such as a network issue) received from an external system, you can view the transactions exchanged between ASM Core and the external system in relation to this event.
  • Inbound Actions. If the current task is linked to a request that was logged as a result of an Inbound Action received from an external system, you can view the messages exchanged between ASM Core and the external system in relation to this Inbound Action
  • Outbound Actions to view the outbound action messages exchanged between ASM and the external system relating to the Outbound Action triggered on a task
  • Audit to display an audit history for the task. The audit trail must be enabled in the Workflow Management Settings window
  • Budgets to specify the estimated time and cost for the current task
  • Knowledge to search for and link knowledge bank entries to the current request through the current task
  • View Change Windows to see a list of all the change windows for the current requests. You cannot modify the details of the change window here.
  • Outages to view planned outages created against this task, and create outages.
  • Transactions to view or perform transactions on a software product or inventory item from the Task Details
  • Transaction History to search on transactions added to a software product or inventory through a task

Within the Quick Launch group, you can select the Quick Launch option, which allows you to run any quick launch URLs to which you have access.

Within the Communication group, you can select:

  • Add Note to add a note to a task. This is useful where you have already forwarded a task to another person and wish to add some details to it without having to take action on the task. The note is added to the history of the item where it can be selected and viewed. Notes can be added to only tasks that are open. If you try to add a note to a task that has been completed, ASM will display a message stating that the task cannot be updated.
  • Letter to create letters (documents) directly from the Task Details window, using applications such as Microsoft Word

Within the Workflow group, you can select:

  • Defer to defer a task. The Defer Task <task number> window appears. At Approval/Task Status, select a status for the deferral and click OK. The Request Details window appears with the task listed in the task browse table with a status of Current. A history entry is also added.
  • Save to store the details of the current task. Selecting this option saves the task, but does not close the Task Details window. You can also save a task by using the Save button on the Task Details window.
  • Complete to close the task after it has been implemented or if you do not want to continue with the task. Clicking this option closes the task.  The task is no longer displayed on your Tasks Outstanding window and its status is updated on the request manager’s Request Details window. (This option is only enabled if the task is active.)
  • ReOpen to refresh the Task/Approval Details window so that the Internal, Defer, Save and Complete buttons are enabled. From here, you can continue to perform the necessary actions to complete the task or approval.
  • Internal to forward or assign a task to one or more internal recipients
  • Screen Set to change the task or approval screen, by selecting another screen set which contains your custom task/approval details screen

The following additional explorer options are displayed in the Create Request Task Details window when the task is added to a workflow template, and you associate the workflow template with the Create Request task:

  • Request Params to set the values for some fields on the request to be created
  • Request Assign to assign the request
