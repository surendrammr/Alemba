Running an AI Ops Rule

A rule will only run if:

  • All the mandatory information has been specified in the AI Ops Wizard and it has been activated.
  • The ASM AI Ops Polling Service is running on the ASM server (enables events to be logged automatically by the system).

The following outlines what happens when an AI Ops rule runs.

  • If the rule runs and does not meet the threshold and conditions within the running period, no call/request is logged for this rule.
  • If the rule runs and the threshold and all other conditions are met, a call or request is logged. Furthermore, if a Grouping Criteria has been used, multiple calls or requests may be triggered for each member of the group (provided each member meets the threshold), when the rule runs. The call or request will be forwarded to the appropriate analyst or group specified on the Call Template or Workflow Template specified for the rule.
  • The next time the rule runs and the threshold and all the conditions are met, a new call or request will be logged if:
    • The first call or request is closed, or
    • The first call or request logged is still open, and the Always Create a Call or Request option is selected for the rule in the Logging Output step in the wizard. Since the first call/request logged is still open, the new call/request will be linked as a child of the first call/request.
  • This process will potentially be repeated as the rule runs and meets its threshold again during the running period. In other words, when the rule runs again, it will:
    • Log a new call or request if the previous call or request is closed. The new call or request is not linked to the previous call or request, since it is a new issue.
    • Log a new call or request and link it as a child of the previous call or request if the latter is still open and the logging option Always Create a Call or Request is selected for the rule in the Logging Output step in the wizard.
    • Not log a new call or request, if the previous call or request is still open and the logging option Always Create a Call or Request is cleared for the rule in the Logging Output step in the wizard.

Ultimately, on closing any parent call or request, all the linked child calls or requests can also be closed.

You can control the assignment of a Call or Request logged by an AI Ops rule by configuring the rule to use a specific Call Template or Workflow Template. For Calls, you can also set up IPK Workflow Rules to route the Calls or trigger notifications.

When a call or request is logged through a rule, the following is added to the Description of the call or request:

  • Title and Description
  • Data Set
  • Selection Criteria
  • Grouping Criteria
  • Schedule
  • Logging Output

If there are any outages running against a call or request analyzed, or SLM breach events, this information is not applied to the call or request logged through the AI Ops rule.
