Quick Launch Explorer Option

Quick Launch URLs are shortcuts that let you run a URL through the click of a link from a call, request or task. Parameters such as call fields, or even fields on a CMDB item linked to the call can be passed to a URL. This is useful, for example if you want to easily access a website.

The Quick Launch tab is only visible if the quick launch functionality is enabled in the System Settings, and you have access to quick launch URLs in your IPK Management security role.

You configure quick launch URLs and any parameters they use on the Quick Launch window in System Administration. For more information, see Quick Launch on page 1

  1. Click the explorer option below the Quick Launch explorer tab which corresponds to the URL you want to run.
  1. The URL will launch outside ASM.
  2. If this quick launch URL is configured to update the call history, the call history will be updated with the entry saying “Quick Launch Application Selected” together with the name of the URL run from the call, and any parameters that have been passed to the URL.

All quick launch URLs will be run from outside ASM, meaning that existing organization IT policies regarding running URLs will still be enforced. Therefore if your IT policy does not allow you to run URLs, you will not be able to run them even if they are displayed on the Quick Launch tab.

ASM will try to update the user of any output it receives from a quick launch URL that has been run from a call. Any output received can be regular output, or an error.

  • If ASM receives regular output from a quick launch application, an informational popup dialog will display with the output message. If the quick launch URL has been configured to update the call history, an entry titled “Quick Launch Output” will be added to the call history. This entry will include the name of the URL from which an output has been received, plus any configured parameters and the output received from the quick launch URL.
  • If ASM receives an error from a quick launch URL (for example, incorrect parameters has been passed to the quick launch URL), a warning popup dialog will display with a generic error message “Unable to launch application. Please contact your system administrator”, plus the details of the error message received.

Not all URLs will return an output to ASM. While ASM will display any output received from a URL, you may not see any response in the cases where a URL does not provide an output.
