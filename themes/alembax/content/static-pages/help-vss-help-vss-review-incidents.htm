Reviewing Incidents in Self Service Portal

You can review past and current incidents from the Self Service Portal. Past Incidents are Incidents that were raised some time in the past and have since been closed after resolution. Current Incidents are Incidents that are open and pending resolution.

You must have Review Own Calls, Review Org Calls and/or Review All Calls selected in the Options tab of your Self Service Portal role, depending on your viewing requirements.

  1. On the My Options menu, select Incidents > Review Past/Current Incidents, or select the Search Incidents link in the My Incidents widget.
  2. The Review Past/Current Incidents window appears. Complete as many of the details as necessary/available.

    3. Incident No

    type the number of the incident, if known.

    If you complete this field, details entered in any of the other fields are ignored in the search.

    Incident State

    Select the Incident status.

    • Open Incidents will instruct the system to search for and return results on incidents that are currently open.
    • Resolved < 1 Week will instruct the system to search for and return results on incidents that were closed (resolved) in the last week.
    • Resolved < 4 Week will instruct the system to search for and return results on all incidents that were closed (resolved) in the last 4 weeks.

    Advanced Search Criteria

    		 Use the Advanced Search Options to widen the scope of your search by specifying additional search criteria.
    Text Search Type in key words or details in the Text Search field. This will instruct the system to search and match the typed entry against all searchable text fields of the Incident.
    Service Select the service relating to the Incident/s for which you are searching. You can use the button to search for the service.
    Type Use the drop-down list to select a call/incident type. This may be multi-tiered, in which case you can select from any of the tiers.
    Priority User the drop-down list to select a priority for the call/incident.
    Log Dates Type a Log Date From and/or Log Date To date, in the format YYYY-MM-DD to specify the time period between which the Incident/s you are searching for were logged.
  3. Click Search at the bottom of the window to begin the search.

Troubleshooting

No Search Results Displayed

If your search does not return any results, modify search criteria to make it more general.
Too Many Search Results Displayed

If your search returns too many results, refine your search by providing more information about the Incident/s for which you are searching. You can do this by completing the Advanced Search Criteria options.
