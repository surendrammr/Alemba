Viewing the Request History

The request history shows actions carried out for the request at previous stages, including both the notes entered in the Actions & Solutions field by handling analysts and automatic entries generated by the system.

Â 

  1. Select the Take Action button on the Request Details window if necessary to allow you to work on the request.
  2. Select the button on the toolbar, or the History explorer option, expanding the Details group if necessary to display it.
  3. The History window shows in detail all stages of the request, including any actions carried out for its resolution and notifications sent to users about the current status of the request. Each entry includes:
  • The type of action taken
  • The analyst who initiated the action (if applicable)
  • The date and time when the action was taken on the request

Request action times are displayed according to the local server time, regardless of the time zone in which the analyst is located. The Unspecified security profile is applied to any system-generated request history entries.

The request history also displays information about:

  • Details of any Service Level Agreement escalation or breach events
  • Calls or requests linked to the current request
  • Objects attached to the request, recorded as hyperlinks. Click the link to open the attachment.
  • Task history entries
  • By default, only entries for active tasks, and the request are displayed in ascending chronological order. To change the order in which the history events are displayed, click Sort Descending on the toolbar to display the events in decreasing chronological order. The request history can again be displayed in ascending chronological order by clicking Sort Ascending
  • To view the details of a specific action, click the right-pointing arrow in front of the history entry.
  • The request history is filtered to only show history tasks that were active when the history was created and preset history types. To change the filter, click and select the filter options you require. Your settings will be saved, and used whenever you open the request history screen.

    • You must select at least one filter from either option. If no option is selected for the filter, the filter is ignored and all items are displayed

  • To view history for tasks that were inactive when the history was created, you will need to check the Show All History Messages checkbox.
