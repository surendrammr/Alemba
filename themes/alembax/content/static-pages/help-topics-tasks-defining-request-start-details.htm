Defining Request Start Details

The Request Start Task is automatically added to the dependency diagram when a request is created. You can specify additional information for this task type.

  1. On the dependency diagram, double-click the Request Start task icon to view the Request Start Task Details window.
  2. Complete the fields in this window.

    3. In the Request Information section, the Request Title, Ref and Description are populated automatically from the request and cannot be edited.

  3. Expand the Task Information section if necessary, and complete the details.

    4. Task Title Type a name for the task if required. By default, this field displays the name of the task entity type.
    Task Phase Select the phase in the request at which this task is implemented.
  4. Expand the Task Description section and in the text area, type some details about this task.
  5. Save the changes.

This is a system-defined task, and therefore cannot be deleted. For email messages to be sent for the tasks in a workflow, the Request Start task must be authorized through the request.
