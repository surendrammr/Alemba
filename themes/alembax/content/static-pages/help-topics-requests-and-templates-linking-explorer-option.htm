Linking Explorer Option

The Linking Explorer option enables you to link calls or requests for change that relate to the request that is open. By using this linking function, you can easily access any related calls or requests from the Request Details interface rather than having to search for each call or request separately.

 

There is no limit to the number of requests and calls which you can link to the current request, However, you must link each call and/or request one at a time.

You can link a request as either a parent or child of the current request, but you can only link calls as children to the current request. You can also link a request to another request that already has requests linked to it.

Before you start

Attributes must be selected on the System Settings window and if request attributes have been defined.

 

  1. Select the Take Action button on the Request Details window if necessary to allow you to work on the request.
  2. Select the Linking option from the Workflow group to display the Request Linking window. The Linking window is divided into two panes:
  • The upper area displays the request linking diagram.
  • The lower area shows the details of the selected linked call(s) or request(s) in a tabular format.

The color of the current request in the linking diagram is always displayed in gray, making it easily identifiable for you as you view linked requests and calls.

  • When several calls of the same type (for example: Incidents) are linked to the request, they are grouped into one node, with the total number of calls indicated on the linking diagram. To view the details of each call in the node, select the node. The linked calls are displayed in the lower part of the window.

    • If the abbreviation assigned to Incidents is I, an incident linked to the current request will display I in the linking diagram. If three incidents are linked to the current request, the node will display 3I. Clicking this node will display the three linked calls in the browse table below.

    The nodes in the linking diagram display the abbreviation assigned to the IPK status of the linked calls. The colors assigned to a node are those that are assigned to the IPK status. Linked requests are denoted by R, and the color used for the links is what your administrator has specified on the Request Link Types window.

    However, the current request is always displayed in gray. This is to make it easy for you to identify the current request on which you are working.

  • From the Linking window, you can:
    • log a new request and link it to the current request
    • search for existing requests and link them to the current request
    • log a new call and link it to the current request
    • search for existing calls and link them to the current request.
    • assign a profile to the link between 2 requests
  • When linking a call to a request, select Copy Details to copy details such as type, description and user from the request to the call. Data items such as attached objects, attributes and forms are not copied across.
  • To link an existing call or request to the current request, select the current request in the linking diagram and click one of the toolbar icons:

    • to open a blank Call Details window, specify the details of the new call, and link it to the current request

    You can only link calls as children to the current request.

    to open the Call Search window, search for a call, and link it to the current request

    You can only link calls as children to the current request.

    to open a blank Request Details window, specify the details of the new request, and link it to the current request

    Requests can be linked either as parents or children to the current request.

    to open the Request Search window, search for a request, and link it to the current request

  • When you have located the call or request, select the button to link it as a child, or to link it as a parent. The selected requests are linked and the Request Linking window appears with the linking diagram updated. Requests belonging to a workflow process are grouped in the diagram.
  • Clicking on a node in the linking diagram displays its details, including the current request number, the call number of the linked call, an indicator to display that the call has been linked as a child, the physical status, priority, type, description and last action date of the call, the analyst who owns it and any specified workaround.
  • To unlink a call or request, select it in the linking diagram. If you selected a node, the linked calls in that node are displayed in the lower part of the window. If applicable, select the request you want to unlink in the lower browse table. Select .

    • If you select the request you are currently viewing, the Unlink button is disabled.

  • To assign a link type to linked requests, double-click the linking arrow in the request linking diagram. The Request Link Details window appears. The Request Number and Linked Request Number fields display the request numbers of the linked requests; there are read-only fields. Select the link type you want to assign from the Link Type drop-down list. Optionally, specify a brief description for the link type in the Link Details field.
