Configuring CMDB Selection Priority

Multiple CMDB items can be linked to calls and requests. You can prioritize CMDB items so that the system can select one, and apply the SLA for that one.

When evaluating a call or request to apply an SLA, the system will evaluate the CMDB Selection Priority list after evaluating the Agreement Selection Priority.

  1. Select â‰¡ > Admin > System Administration.

    The System Administration window appears.

  2. In the Explorer pane, expand Service Level Management, then select CMDB Selection Priority.

    All CMDB Item fields defined in the system are displayed. This includes both standard fields and fields added through the Designer .

  3. To set up the CMDB selection priority for calls, leave the Calls option selected. To set up the agreement selection priority for requests, select the Requests option.
  4. In the left-hand browse table, select the CMDB selection priority you wish to make available as agreement selection criteria.
  5. Select Add >> or double-click the priority.

    6. You do not have to include all of these options in your priority list.

  6. Use the Move up andMove Down buttons to specify the order in which you want the criteria to be evaluated when ASM Core searches for a matching agreement to apply.
  7. To remove an added CMDB selection priority, select it in the right-hand browse table. Select Remove.

    8. You should only remove a field from the list if you are sure that you will never want ASM Core to search against that field when looking for an agreement.

  8. Once you have completed your CMDB selection priority, select to apply the changes.
