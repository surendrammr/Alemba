AI Ops Explorer Option

You can link calls or requests created by the same AI Ops rule.

Â 

  1. Display the Call or Request Details window, if it is not already on screen.
  2. On the explorer pane, expand the Workflow group if necessary, and select AI Ops.

    3. This option is only visible if the call or request was triggered by an AI Ops rule.

  3. The browse table on the AI Ops window displays a list of all the calls or requests that resulted in the creation of the current call or request. The current call or request is the call or request generated automatically by the AI Ops rule.

    4. If the AI Ops rule that created the current call or request was set up to analyze calls/requests, then the calls/requests that were analyzed and generated are listed in the browse table.

  4. To link the AI Ops call/request to an originating call or request, select the call or request in the browse table to which you want to link the current call or request.
  5. Click to link the current call or request to the selected call or request as a child.
  6. Click to link the current call to the selected request as a parent.
  7. If the current call or request is already linked to another call or request, a dialog box displays a message to this effect. Click Yesto link the selected call or request instead. No cancels the action.
