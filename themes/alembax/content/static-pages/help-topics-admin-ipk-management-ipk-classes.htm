IPK classes

IPK classes are used internally to classify each call as an Incident, a Problem, a Known Error or a Service Request. These grouping classes are defined in ASM Core. You cannot add or delete IPK classes; you can only rename them.

Renaming an IPK class

Before you start

Ensure you are in the correct partition before applying the settings.

You must have IPK Setup selected in the Admin tab of your General Access Security Role.

  1. Select ≡ > Admin > System Administration.

    The System Administration window appears.

    In the Explorer pane, expand IPK Management.

  2. Select IPK Classes to open the window. The classifications are displayed, although you may find that one or more are already renamed.
  3. Highlight the entry that you want to rename.
  4. Type the name you want to assign for the selected IPK report grouping class.
  5. Select to save the changes. Provide the Change Reasons if prompted to do so.
