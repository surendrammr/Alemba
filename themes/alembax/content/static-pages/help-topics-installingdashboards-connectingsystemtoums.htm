Connecting Alemba Service Manager™ to the User Management Server

When you connect Alemba Service Manager™ to the User Management Server, all the Alemba Service Manager™ users, groups and categories are imported into the Dashboard database that you configured. These Dashboard database tables have a prefix that you assigned when you installed the Dashboard platform. For more information about the database, see Configuring the Dashboard Server.

If you want to use Active Directory authentication for your users and passwords, you must first synchronize the User Management Server with Active Directory before you connect ASM to the User Management Server. For more information, see Enabling User Management Server for Active Directory authentication.

To connect ASM to the User Management Server:

  1. Using ASM Core, select ≡ > Admin > System Administration.

    The System Administration window appears.

  2. In the Explorer pane, expand Dashboards, then select User Management Server Settings.

    The User Management Server Settings window appears.

  3. Select Enabled to enable the Reporting Service.

  4. If required, use the Service Poll Interval to adjust how often the Reporting Service polls the User Management Server.

  5. In Application URL, type the User Management Server URL in the following format: http://ServerName/AlembaUMS/, where ServerName is the server name or IP address.

    For example, http://localhost/AlembaUMS/

  6. In User ID, type the Dashboard system administrator you set up.

    For example, AlembaAdmin.

    For more information, see Configuring the Dashboard Server.

  7. In User Password, type the Dashboard system administrator password.

  8. In Client Id, paste the User Management Server client ID.

    For more information about sourcing the client ID, see Accessing the User Management Server client ID.

  9. Select Run to ensure the connection works and the user import succeeds.
  10. Select to save your changes.
  11. Close, then reopen System Administration to refresh the menu.

  12. Using the command prompt, perform an iisreset to restart IIS.

  13. Use Windows Services to start the Alemba Service Manager™ Reporting Service.

  14. If you want to manually import the users, groups, and categories, on the User Management Server Settings window, select Run.
