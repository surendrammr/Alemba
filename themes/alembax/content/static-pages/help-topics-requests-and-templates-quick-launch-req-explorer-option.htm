Quick Launch Explorer Option

The Quick Launch section on the Request Details Explorer displays shortcuts you can use to launch URLs from within ASM Core. Only Quick Launch URLs you have access to as part of you Workflow Management Security Role settings are displayed.

 

Parameters such as request field values or field values on a CMDB item linked to the request can be passed to a Quick Launch URL. This is useful for example, for automatically filling in a part number on a website for purchasing purposes..

If you run a quick launch URL which uses CMDB item fields as parameters passed to this URL, ASM will use the currently selected configuration item or service linked to the request. If no configuration item is selected, the one with the lowest reference number is used.

Running a Quick Launch URL

Before you start

Quick Launch is only visible if the quick launch functionality is enabled in System Settings. All quick launch URLs, including the parameters used, have been configured on the Quick Launch window in System Administration.

You must have access to quick launch URLs on the Request Quick Launch tab of your Workflow Management Security Role.

  1. Expand the Quick Launch group in the Request Details explorer pane. A list of all the quick launch URLs to which you have access will display.
  2. Select the explorer which corresponds to the URL you want to run.

    3. All quick launch URLs run from outside the ASM main application, meaning that existing organization IT policies regarding running URLs will still be enforced. Therefore if your IT policy does not allow you to run certain aURLs, you will still not be able to run these URLs even if they are displayed on the Quick Launch tab.

  3. The URL opens in a separate browser. If this Quick Launch URL has been configured to update the request history, a new request history entry is added, indicating the name of the URL run from the request and any parameters passed to the URL.

Receiving output or errors from a quick launch URL

ASM will try to update the user of any output it receives from a quick launch URL that has been run from a request. Any output received can be regular output, or an error.

  • If ASM receives regular output from a quick launch URL, an informational popup dialog will display with the output message. If the quick launch URL has been configured to update the task history, an entry titled “Quick Launch Output” will be added to the task history. This entry will include the name of the URL from which an output has been received, plus any configured parameters and the output received from the quick launch URL.
  • If ASM receives an error from a quick launch URL (for example, incorrect parameters has been passed to the quick launch URL), a warning popup dialog will display with a generic error message “Unable to launch application. Please contact your system administrator”, plus the details of the error message received.

Not all URLs will return an output to ASM. While ASM will display any output received from a URL, you may not see any response in the cases where a URL does not provide an output.
