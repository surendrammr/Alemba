Dashboard Quick Start Guide

You will learn how to create dashboards using Web Designer.

This Quick Start Guide presumes you have correctly installed and configured your ASM System and the Dashboard Platform. For more information, see the ASM 10.1 Prerequisites and Installation Guide.

1. Create a dashboard

  1. Using Dashboard Server, select Homepages.

  2. Select > Create > Dashboard.

    An Untitled dashboard appears in the web designer.

2. Create a data source

  1. Select Data Sources.

  2. Select Create New.

  3. Select the Alemba connection type.

  4. Select Connect New Account.

  5. Type the Base URL for your Alemba Service Manager™ system in the format http://ServerName/VirtualDirectory/

    where ServerName is the Server Name or IP address.

  6. Type the Client ID, which you will find in the Alemba API Explorer, in the Dashboards API Client.

  7. Select Connect.

    The Alemba Account login page appears.

  8. Type the Alemba Account credentials, then select Login.
  9. Type a name for the account.

  10. Select Next.

  11. Select Continue.

    The data source appears on the right side of the dashboard.

  12. Select Connect.

    The data design view appears.

3. Configure the data source

  1. In the Tree view, expand Alemba, then expand Tables.

  2. Drag the table that you want to extract data from onto the data design view window.

    All the fields on that table are listed. The data preview is displayed below the table.

    The changes you make here will not affect the underlying data.

  3. On the table, select in the title bar to hide all columns.

    We recommend that you globally hide all columns and only display the data you want to report on so that performance is not impacted. This provides a smaller data set for the system to refresh, and has less impact on performance.

  4. Select alongside each column you want to display.

  5. Select > Change Properties > Name to display the field name.

    The Data Preview updates for that column.

  6. Select to filter the table.

    The Query Filters window appears.

  7. Select Add.

  8. Select a field from the drop-down list, then configure the filter parameters.

  9. If required, select Add to add further filters.
  10. Select OK to set the filter.

  11. If required, add more tables.

    You will need to join multiple tables. For more information, see the Online Help.

  12. Change the data source Name to describe the displayed data.

  13. Select Save to return to the dashboard.

4. Add widgets to the dashboard

  1. Expand or contract the menu to see the widgets easily.

  2. Drag a widget onto the dashboard.

    The Settings menu opens on the right side.

  3. If required, use the resize handles to change the widget display size.

    You can also select and drag the widget to move it.

  4. In the Settings menu, on the Properties tab, change the Name.
  5. Expand or contract the Properties headings to view different adjustable settings.

  6. Change the Properties as required.

    For example, you can change the title text color, and the value alignment.

  7. In the Settings menu, on the Assign Data tab, drag one or more Measures into the Value(s) pane.

    A measure is used to display the primary data. For example, number of calls.

    When you drag a Measure or Dimension into the panels, ensure your cursor points within the Drag & Drop boundary line before you release the item, otherwise it will not adhere to the panel.

  8. Drag one or more Dimensions into the Column(s) panel.

    A dimension divides the primary data into sections. For example, number of calls by Assigned Group.

  9. If required, adjust the primary data using the Settings menu.

    You can change the aggregation, filter the displayed information and format the display.

    The widget updates to display your choices.

  10. Drag more widgets onto the dashboard.

    The available Settings depend on the type of widget.

  11. Select to save the dashboard.

    The Save As Dashboard window appears.

  12. Select a Category from the drop-down list.
  13. Type a Name to identify the dashboard.

  14. Select a Privacy Settings option.

    Private displays the dashboard to those users with the appropriate permission. You must sign in to see private dashboards.

    Public displays the dashboard to anyone with a link to them. No sign-in is required.

    Unlisted displays the dashboard to anyone with a link to them. No sign-in is required. Only dashboard owners can see unlisted dashboards in their dashboards list.

  15. Select Save.

  16. Select Yes to view what the dashboard looks like in a browser.

  17. At any time, you can also select Preview to view the dashboard.

5. Make the dashboard available in the Alemba Service Manager™ Dashboard menu

You can configure the Alemba Service Manager™ Dashboard menu to open specific dashboards. For more information, see the ASM 10.1 Prerequisites and Installation Guide.
