Contract Types

You can define different contract types in order to create different profiles. For example you may want to set up different kinds of Contracts such as on-site, off-site, contract and warranty. You may define as many contract types as you wish.

Creating a Contract Type

Before you start

You must have CMDB Setup enabled within your General Access security role in order to configure any CMDB administration settings. Depending on if or how your system is partitioned, ensure that you are working in the correct partition.

  1. Select ≡ > Admin > System Administration.

    The System Administration window appears.

    In the Explorer pane, expand CMDB.

  2. Select the Contract Types option from the Explorer pane. The Contract types window appears, allowing you to define profiles for different types of contracts.
  1. Select . A new row is created to the browse table.
  2. In the Name list, type the name of the new contract type you want to create.
  3. Repeat these steps for each contract type you want to create.
  4. Select to save the changes. Provide the Change Reasons if prompted to do so.

Renaming a Contract Type

Before you start

You must have CMDB Setup enabled within your General Access security role in order to configure any CMDB administration settings. Depending on if or how your system is partitioned, ensure that you are working in the correct partition.

You can rename the default contract type but it is not recommended.

  1. Select ≡ > Admin > System Administration.

    The System Administration window appears.

    In the Explorer pane, expand CMDB.

  2. Select the Contract Types option from the Explorer pane. The Contract types window appears, allowing you to define profiles for different types of contracts.
  1. Select the contract type entry you want to edit by highlighting it.
  2. Type the name you want to assign for the selected contract type.
  3. Select to save the changes. Provide the Change Reasons if prompted to do so.

Deleting a Contract Type

Before you start

You must have CMDB Setup enabled within your General Access security role in order to configure any CMDB administration settings. Depending on if or how your system is partitioned, ensure that you are working in the correct partition.

You cannot reverse this procedure. If you delete a contract type by mistake, you have to re-create the entry by creating a new contract type.

  1. Select ≡ > Admin > System Administration.

    The System Administration window appears.

    In the Explorer pane, expand CMDB.

  2. Select the Contract Types option from the Explorer pane. The Contract types window appears, allowing you to define profiles for different types of contracts.
  1. Select the contract type you want to delete from the list by highlighting it.
  2. Select . If you attempt to delete the system default, you will receive a warning message.
  3. Select to save the changes. Provide the Change Reasons if prompted to do so.
