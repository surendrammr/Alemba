Logging an Incident in Self Service Portal

To log an Incident in Self Service Portal, follow these steps:

  1. On the menu, select Incidents > Log an Incident.

    2. Your system administrator may have set up different incident logging screens for different types of incident. If so, select the most appropriate one from the list.

  2. Complete the details.

    3. If your system administrator has set up suggested knowledge, you will see proposed solutions for your incident as you complete the details. If you find the answer to your query, you can select it and then click the Solved button to close the call without logging it.

    Service

    Identify the service you want to log a call against.

    Search as you type

    This field performs a search as you type, presenting matching results which adjust as you continue typing in the field. Your system administrator determines which columns from the entity to display in the results popup, which columns to search within for matching values, whether to perform a 'starts with' or 'contains' type search, how many characters must be typed before the results are displayed, and how many results to show at one time.

    You can press Enter to see the first n results in alphabetical order (where n is number of results defined by the Admin); or use the button, which appears after you start typing, to do a search if you wish.
    Config Item

    Identify the config item the call relates to.

    Search as you type

    This field performs a search as you type, presenting matching results which adjust as you continue typing in the field. Your system administrator determines which columns from the entity to display in the results popup, which columns to search within for matching values, whether to perform a 'starts with' or 'contains' type search, how many characters must be typed before the results are displayed, and how many results to show at one time.

    You can press Enter to see the first n results in alphabetical order (where n is number of results defined by the Admin); or use the button, which appears after you start typing, to do a search if you wish.
    Type Use the drop-down tiered list to identify the problem type. There may be several tiers of problem type. You can select from any tier.
    Urgency Use the drop-down list to identify the urgency level.
    Impact Use the drop-down list to identify the impact level.
    Description Type a free text description of the problem.
    Send Email Select the checkbox to send an email to the analyst to whom the Incident is assigned.
    Send Pager Select the checkbox to send a pager message to the analyst to whom the Incident is assigned.
    Attach File If you have screen shots or additional information relating to your Incident contained in one or more files, you can attach them to your Incident. Select Attach File field. Click Browseâ€¦ and select the file(s) you want to attach.
  3. Click Submit. The Incident is forwarded to an analyst or group based on the details given. Once you have submitted your Incident, the system will display an acknowledgment with the automatically allocated unique Incident number and a summary of the Incident that you logged.

The date and time when the Incident was logged is automatically recorded and can be viewed in the Incident details.

If you are an analyst logging an Incident via Self Service Portal, you will automatically be set as the Logging Analyst as well as the Current Analyst of the Incident. If your system is partitioned, the Incident will be logged in the partition specified in the URL of the page you are viewing.

Further options enable you to:

  • Add a Note to Your Incident
  • Attach one or more Object(s) to Your Incident
