Object Explorer Option for Requests

The Object Explorer option enables you to attach screenshots and files from Windows applications to the current request. These files can, for example, provide additional information about the request to contribute to its implementation.

If Object Version Control for requests has been selected by the system administrator on the Workflow Management Settings window, the Attach Object to Request window also displays the following buttons: Check Out, Check Out Exec, Check In, Cancel Check Out and View History.

Attaching a File to a Request

Before you start

You must have Request Object View selected in your Workflow Management security role to view objects attached to the request. You must have Request Object Update to change the details; Request Object Cancel to cancel a lock added by another analyst; and Request Object Delete to delete objects attached to a request. These all appear in the Requests tab.

 

  1. Select the Take Action button on the Request Details window if necessary to allow you to work on the request.
  2. In the Explorer pane, expand Data, then select Object.The browse table shows details of all the objects that are already attached to the request.
  3. Select the button to attach a file, browse to the file, and click Open. Use the Security drop-down list to specify a security profile to restrict access to the attachment.

    4. The Security field is only visible if security profiles are enabled and you have access to more than one security profile.

  4. Add a description for the file in the Upload Object window. Select Visible in Self Service Portal if you want to make the description of the attached file visible to the user through the portal and select OK. The details of the file are displayed in the browse table, including the title of the file, its size, the date it was uploaded, and the security profile that was applied.
  5. When a file is attached to the request, a small orange square appears next to the Object explorer option, and the option name is also orange.

Downloading a File Attached to a Request

Before you start

You must have Request Object View selected in your Workflow Management security role to view objects attached to the request. You must have Request Object Update to change the details; Request Object Cancel to cancel a lock added by another analyst; and Request Object Delete to delete objects attached to a request. These all appear in the Requests tab.

 

When a file is attached to the request, a small orange square appears next to the Object explorer option, and the option name is also orange. To download the file onto your computer, take the following steps.

  1. Select the Take Action button on the Request Details window if necessary to allow you to work on the request.
  2. In the Explorer pane, expand Data, then select Object.The browse table shows details of all the objects that are attached to the request.
  3. Select the object to download and click .
  4. Select the file location where you want the object to be saved and click Save.

Updating a File Attached to a Request

Before you start

You must have Request Object View selected in your Workflow Management security role to view objects attached to the request. You must have Request Object Update to change the details; Request Object Cancel to cancel a lock added by another analyst; and Request Object Delete to delete objects attached to a request. These all appear in the Requests tab.

A file can only be updated if it remains in the same location where it was automatically saved by ASM Core. Saving it in a different directory is treated as a new object rather than the latest version of the attached file.

 

When a file is attached to the request, a small orange square appears next to the Object explorer option, and the option name is also orange.

  1. Select the Take Action button on the Request Details window if necessary to allow you to work on the request.
  2. In the Explorer pane, expand Data, then select Object. The browse table shows details of all the objects that are attached to the request.
  3. Select the file to update and click the Update button on the toolbar.
  4. Make the necessary changes.
  5. After making changes to an attached file, the button turns red. Click Update to update the existing object with your latest version.
