Managing Resource Types

The Integration Resources window displays all the resource types discovered on the selected source. This window allows you to select a resource type in order to set up the mapping between the resource type and CMDB Types and data fields in ASM Core.

Viewing Resource Types

Before you start

You must have Integration Setup selected in the Admin tab of your General Access Security Role.

  1. Select ≡ > Admin > Integration. The Integration Platform Settings window appears.
  2. In the Explorer pane, under Integration, select Resources.
  3. Complete the details.

    4. Filter by Source

    Select a source from the drop-down list.

    Only sources that correspond to connectors with Federated CMDB capability and which are set to be visible are available in this list.
    Mapped Select this option to view resource types for which mappings have been defined or which have inherited mappings from their ancestors.
    Not mapped Select this option to view resource types that do not have any mappings defined or any inherited mappings.
  4. The resource types are listed below the search parameters.

Viewing Resource Type Mappings

Before you start

You must have Integration Setup selected in the Admin tab of your General Access Security Role.

  1. View the Resource Types.
  2. Double-click the resource type. The Resource Type [Name of the resource type] window appears. It lists the existing mappings for the selected resource type and enables you to create new mappings, as well as edit, delete and reorder existing mappings.

    3. Resource mappings apply to a selected source and connector, and resource type. If you define more than one instance of the same source and connector, resource mappings will only apply to the particular instance of the source and connector for which they were defined. They will not apply globally to all instances of the same source and connector.

  3. The following information appears in the Resource Type Mappings window:

    4. Source

    The name of the selected source from which the resource type is derived

    Resource Type

    The name of the resource type

    Map Resources by Group

    This option only displays if you are viewing the resource type mappings for an LDAP-compliant Directory Server connector source.

    In Active Directory, users are organized into groups or Organizational Units (OUs) on the directory server. When the Map Resources by Group option is selected, a Groups browse table is also displayed on this window where you must select a group in order to apply mappings to the resource type group rather than the resource type itself. 

    You can only set up one mapping per group.
  4. Beneath these fields, the Mappings browse table displays the following details about a mapping:

    5. Mapping

    the name of the mapping specified when defining the mapping details

    Import

    if the checkbox is selected, then particular import rules have been specified in the mapping details

    Default

    if the checkbox is selected, then this mapping has been configured as the default mapping. If you define a default mapping, then external resources that do not meet the criteria defined in any mappings can still be imported. If no default mapping is set, then some resources may be overlooked by a scheduled scan and not be imported into the CMDB.

    The Groups table displays the following information about a group.

  5. This table only appears on this window for an LDAP-compliant integration source and connector. For example, on an Active Directory, users are organized into groups (OUs). Resource type groups are defined on the source.

    6. Type

    displays the name of the group type (defined on the source)

    Group

    displays the name of the group (defined on the source)

    Total

    displays the number of mappings that have been defined for the group

Adding a resource type mapping

You can add as many mappings as you need, except in the case of an LDAP-compliant directory server source, for which you can only add one mapping. You can also set one of the mappings as the default for this resource type so that resources that do not meet the criteria for any defined mappings in this window will still be discovered in a scheduled scan, and imported or updated.

Before you start

You must have Integration Setup selected in the Admin tab of your General Access Security Role.

  1. View the Resource Types.
  2. Double-click the resource type. The Resource Type [Name of the resource type] window appears.
  3. Select New from beneath the Mappings table.
  4. You can also clone an existing mapping.
  5. Specify the mapping details.
  6. Select to save the changes.

Updating a resource type mapping

Before you start

You must have Integration Setup selected in the Admin tab of your General Access Security Role.

  1. View the Resource Types.
  2. Double-click the resource type. The Resource Type [Name of the resource type] window appears.
  3. Select the mapping you want to update from the browse table and then select Open.
  4. Update the details as necessary.
  5. Select to save the changes.

Deleting a resource type mapping

Before you start

You must have Integration Setup selected in the Admin tab of your General Access Security Role.

  1. View the Resource Types.
  2. Double-click the resource type. The Resource Type [Name of the resource type] window appears.
  3. Select the mapping you want to delete from the browse table and then select Delete.
  4. Confirm the deletion.

Changing the priority of resource type mappings

ASM Core will apply the mapping and match the external resource to the internal resource in the order in which the mappings appear in the Mappings (and Groups if relevant) browse table. If more than one mapping applies to an external resource, then the mapping that is higher in the list will be applied first, then the second one in the list, and so on.

Before you start

You must have Integration Setup selected in the Admin tab of your General Access Security Role.

  1. View the Resource Types.
  2. Double-click the resource type. The Resource Type [Name of the resource type] window appears.
  3. Select a mapping you want to move, and use the Move up andMove Down buttons to reassign the priority. Repeat for all mappings you want to reorder.
  4. Select to save the changes.

Adding a resource type mapping for a Directory Server group

Before you start

You can only add one mapping per group.

You must have Integration Setup selected in the Admin tab of your General Access Security Role.

  1. View the Resource Types.
  2. Double-click the resource type. The Resource Type [Name of the resource type] window appears.
  3. In the Groups table, select the group for which you want to define a mapping.
  4. To search for a group, click Add.
  5. To delete a group and any mappings defined for it, select the group in the table and click Delete. In the confirmation message dialog box, click Yes to confirm the deletion or No to cancel.
  6. Once you’ve selected a group for which you want to define a mapping, click New in the Mappings browse table.
  7. Specify the mapping details.
  8. Select to save the changes.

Changing the priority of groups

ASM Core will apply the mapping and match the external resource to the internal resource in the order in which the mappings appear in the Groups browse table. If more than one matching mapping applies to an external resource, then the mapping that is higher in the list is used.

Specifying Resource Type Mapping Details

You can define a mapping between external resources and ASM Core CMDB fields. External resources can be network discovery resources or users imported from a directory server.

The Source and Resource Type fields on the Mapping for Resource Type window are automatically populated based on the source and resource type selected to access this mapping. The source is the application or directory server whose resources you want to configure the mapping for. The resource type is the category of the resource you are mapping.

To define a resource type mapping:

  1. Display the Mapping for Resource Type [Resource Type Name] window if necessary.
  2. Select New to create a new resource type mapping, or double click on an existing mapping to update the details.
  3. In the Name field, type a name for the mapping. This name will appear in the Mappings table on the Resource Type window.
  4. Select the Criteria tab if necessary.

    5. Default Mapping Leave the option selected to flag this mapping as the default, . The mapping will be applied whenever no other mapping is applicable. It is useful to define a default mapping so that resources that do not meet the criteria specified in any of the other mappings for that resource type can still be discovered and imported. If no default mapping is set, then some resources may be overlooked by a scheduled scan and not be imported into the CMDB.

    Deselect to specify the conditions for applying this mapping to an external resource and build the conditions using the Rules Builder panel. These conditions are set through defining a rule.

    For example, to only import resources of the Computer resource type that are manufactured by Dell, build the following rule: {Resource:Manufacturer}=“Dell”. This mapping will be used for all resources of the Computer resource type and manufactured by Dell.

    The fields available in the left-hand side of each clause of the rule correspond to the fields associated with the external resource type. They are defined at the level of the connector and cannot be modified. The values available in the right-hand side of the clause are values corresponding to the field selected on the left.  You can only select fields or attributes from the external resource type as the criteria in your rule, since there are the criteria ASM Core will use to select external resources for importing.

  5. Select the Action tab to define the import rules used when this mapping is applied.

    6. Import Leave this option selected to enable the fields on this tab. If you clear this option, external resources will not be imported or updated for this mapping when a scheduled scan runs on the source.
    Import As Use the drop-down list to select the type of CMDB entity type that the external resource will be imported as. By default, Person is selected if the source is a directory server and CMDB Item if the source is a network discovery source. However, you can modify the default selection.
    Using template/Clone from Use this field to select the template to apply to the CMDB records created after the source is scanned for resources. Only the templates for the CMDB entity type selected in the Import As field are available for selection.

    To create a new template from this window, open the Using template menu and select New.
    Apply template to CMDB Item or Person Use the drop-down list to select when the selected template must be applied to the imported resource:
    On creation Select only to apply the template to the imported resource only when the CMDB record is created, that is, only if the resource does not match an existing CMDB item or person record.
    On creation or update Select to apply the template to the imported resource when the ASM Core record is created or when an existing linked CMDB record is updated.
    When template is different to last applied

    Select to apply either (a) the current template that is different from the one last applied to the matching CMDB record, or (b) a modified version of the template last used for the imported resource.

    For a directory server source, users may need to be imported under another Person template if their status has changed on the directory server.
    When mapping is different to last applied Select to apply the template to the imported resource when the field mapping last used is different from this mapping.
    When a resource is discovered Use the drop-down list to select the action to be taken on the CMDB if a new resource or user is discovered on the source:
    Create linked CMDB Item or Person automatically Select to instruct the polling service to discover any new resources on the source. When a new resource is found, ASM Core automatically creates a new CMDB record and then links this CMDB record and the record that is created in the Federated CMDB.
    Add to the Pending Imports list Select to instruct the polling service to take no immediate action on the CMDB. Instead, it will record the detected definitive resource record as a “pending import”, which you can later review in the Pending Imports View on the Federated CMDB Administration window.
    When a resource is deleted Use the drop-down list to select the action to be taken on the CMDB if a resource or user is deleted on the source:
    Delete linked CMDB Item or Person automatically Select to instruct the polling service to discover any deleted resources on the source and delete the linked CMDB items or people immediately.
    Add to the De-activated Resources list Select to instruct the polling service to take no immediate action on the CMDB. Instead, it will record the detected differences for each definitive resource record as a “deactivated resource”, which you can later review in the Deactivated Resources view on the Federated CMDB Administration window.
    Take no action Select to instruct the polling service to make no changes to the CMDB.
    When a resource is restored Use the drop-down list to select the action to be taken on the CMDB if a resource or user is restored on the source:
    Restore linked CMDB Item or Person automatically Select to instruct the polling service to discover any restored resources on the source and apply that information directly to the linked CMDB items or people. The CMDB items or people will be restored immediately.
    Add to the Activated Resources list Select to instruct the polling service to take no immediate action on the CMDB. Instead, it will record the detected differences for each definitive resource record as an “activated resource”, which you can later review in the Activated Resources view on the Federated CMDB Administration window.
    Take no action Select to instruct the polling service to make no changes to the CMDB.
  6. Select the Fields tab to map the fields from the imported external resource with fields from the ASM Core CMDB item or person record.

    7. You can only map fields once you have selected a CMDB item or Person template under the Actions tab.

  7. To add a field for mapping, click Add. The Add Fields window displays all the fields from the template selected under the Actions tab.
  8. Select the ASM Core CMDB field you wish to map and click OK to add the field to the Fields table.
  9. To set a value for this field or select a matching field, click the corresponding cell in the Resource Field column and select . The options available depend on the type of field (string, date and time, profile list, etc.). You must select a resource field or value for each field selected in the Fields table; otherwise, you will receive a warning message when you save the mapping details.
  10. To set the condition for updating the field, select one of the options in the Update column:

    11. Always populates the field automatically whenever this field mapping is applied during a scheduled scan. This option is useful if the external source is the definitive place where this information resides.
    On Initial Population Only populates the field only when a CMDB item or person is created in ASM Core during a scheduled scan. This option is useful if this CMDB record may be updated from integrations with various external sources.
    Only When Blank populates the field only if it is blank. This is useful if the field on the external source is not the definitive place where this information resides and should not have the ability to overwrite the template field value.
    Create Discrepancy Report does not populate the field automatically when the field mapping is applied during a scheduled scan. Rather, an entry is added to the Discrepancy Report view in the Federated CMDB Administration window, allowing you to decide whether the field can be updated or not. This is useful to detect unauthorized changes or errors on the source.
  11. Select the Matching tab to set the criteria ASM Core will use to match a discovered external resource to an existing CMDB record (rather than create a new one).

    12. This step allows you to configure rules for importing and updating resources. This includes defining how users are imported through a directory integration source or other resources through a network discovery connector. For example, by defining a rule using fields from the person details, you can match a new user discovered on a directory server to an existing analyst or User in ASM Core.

  12. Select On discovery match resource to existing CMDB Item or Person to enable the matching of a discovered resource to an existing CMDB record based on a set of criteria. If you leave this option unselected, a new CMDB record will be created for each imported resource rather than a link with an existing CMDB record.
  13. Use the Rules Builder to specify the criteria that will be used to match a discovered resource or user to an existing CMDB record:
    1. Start a statement by clicking the question mark (?).
    2. Click to select a field. You can choose either a field from ASM Core (select CMDB Item) or a field from the external resource (select Resource). Based on your selection, the next tier opens, enabling you to drill down and select the relevant field.
    3. Select an operator for the rule, such as equal to (=).
    4. Select the final part of the statement by selecting .
    5. Set a value for the field or select a matching field. The options available for selection in the multi-tier field selector pop-up depend on the field type you selected in the first part of the rule.
  14. Select what you want ASM Core to do if a matching CMDB record is found:

    15. > 1 matching records are identified Select to add the discovered resource to the Pending Imports list if more than one record in the ASM Core CMDB has been found to match the resource based on the criteria specified above. This option enables you to manually match the resource to one of the matching CMDB records.
    NO matching records are identified Select to add the discovered resource to the Pending Imports list if no record in the ASM Core CMDB is found
    Leave both checkboxes unselected to import the resource directly and automatically link the resource to a matching CMDB record. If more than one CMDB record matches the resource based on the set of criteria you specified and both checkboxes are left unselected, ASM Core automatically selects one of the CMDB records to link it to the resource.
  15. Select to save the changes.You are returned to the Resource Type window, where you can add more mappings for the same resource type if needed.

Mappings are applied only when a resource is imported or updated. Changing the resource mappings will not update existing CMDB records. Updates are made only when ASM Core detects changes to a resource.

Disabling scans for a connector or a specific resource type after a scan has been run will not delete CMDB records linked to the external resources for that connector. However, viewing any of these external resources (which may be linked to CMDB records) will display them as being inactive.

Resources are imported automatically during a Federated CMDB scan, and these resources are checked for any links that correspond to the mapped resource link types. This relationship information is automatically discovered by the scan; it cannot be regulated. However, you can choose not to import the relationship information as links into the CMDB when you import resources with links to other resources.

Field Mappings for Directory Servers

The fields available for mapping a directory server user to a Person entity in ASM Core include the following:

Login ID allows you to link a user from a directory server to a person record in ASM Core using the Login ID details.

When a person record is deleted from ASM Core based on a disabled Active Directory record, their Login ID and Password is deleted within ASM Core and is available for reuse.

Email allows you to link a user from a directory server to a person record in ASM Core using the email details.
First Name and Surname allows you to link a user from a directory server to a person record in ASM Core using the First Name and Surname
