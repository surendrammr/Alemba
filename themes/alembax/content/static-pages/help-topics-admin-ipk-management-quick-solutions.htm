Quick Solutions

You can define macros that automatically update details in the Call Details window and optionally close the call. These "quick solutions" can be defined for commonly logged calls, such as a locked account or forgotten password, where the same details are required every time. Using Quick Solutions speeds up the process of logging common calls.

Analysts can select a quick solution by selecting the Quick Solutions explorer option on the Call Details window, and selecting the required solutions from the displayed Select Quick Solution window. The Quick Solutions displayed will be the ones linked to the type they have chosen. If there are no Quick Solutions linked to the call’s issue type, all the Quick Solutions in the system will be displayed. You can define multiple Quick Solutions for a type.

Creating a Quick Solution

Before you start

Configured must be selected under Quick Solutions in the IPK Settings (Partitioned) windows. (If you select Disable Checks in the same window, a call to which a quick solution has been applied can be closed immediately, bypassing validation checks such as User Required, CI Required, Service Required and Type Required. This means that even if mandatory fields are left incomplete, the call can still be closed.)

The issue types must be defined in the Type Tiers window.

  1. Select ≡ > Admin > System Administration.

    The System Administration window appears.

    In the Explorer pane, expand IPK Management.

  2. Select Quick Solutions to open the window. A browse table at the top of the window shows all existing quick solutions. To view any that have been deleted, select the Deleted radio button.
  1. Select Add. A row is inserted in the browse table. 
  2. In the Name field, type a descriptive title for the quick solution (that is, the issue commonly logged).
  3. To close the call to which the quick solution is applied immediately on applying the quick solution, select the Close checkbox.
  4. At Type, select to link the quick solution to an issue type. This will be recorded as the issue type for the call to which this quick solution is applied.  By default, the default type (Unspecified) is selected.
  5. If you selected Close, from the Reason list, select the reason that will be used when the call to which the quick solution is applied is closed.

All reasons are available for selection regardless of any links between reason and type or screen set in the Link Screen Set to Reason or Link Type to Reason setting windows.

  1. In the Quick Description field, type the description the issue for this quick solution that will populate the Description in the Call Details window.
  2. In the Quick Solution field, type the description of the solution that will populate the Actions and Solutions in the Call Details window.
  3. Select to save the changes. Provide the Change Reasons if prompted to do so.

A Quick Solution can be linked to the Type “default”, and need not be assigned a name. If configured this way, the Quick Solution will be linked to all calls.

Deleting a Quick Solution

Before you start

Configured must be selected under Quick Solutions in the IPK Settings (Partitioned) windows. (If you select Disable Checks in the same window, a call to which a quick solution has been applied can be closed immediately, bypassing validation checks such as User Required, CI Required, Service Required and Type Required. This means that even if mandatory fields are left incomplete, the call can still be closed.)

The issue types must be defined in the Type Tiers window.

  1. Select ≡ > Admin > System Administration.

    The System Administration window appears.

    In the Explorer pane, expand IPK Management.

  2. Select Quick Solutions to open the window. A browse table at the top of the window shows all existing quick solutions. To view any that have been deleted, select the Deleted radio button.
  1. Select an entry in the browse table.
  2. Select Delete. This moves the quick solution from the Active list to the Deleted list and it is no longer available to Analysts closing a call.
  3. Select to save the changes. Provide the Change Reasons if prompted to do so.

Restoring a Deleted Quick Solution

Before you start

Configured must be selected under Quick Solutions in the IPK Settings (Partitioned) windows. (If you select Disable Checks in the same window, a call to which a quick solution has been applied can be closed immediately, bypassing validation checks such as User Required, CI Required, Service Required and Type Required. This means that even if mandatory fields are left incomplete, the call can still be closed.)

The issue types must be defined in the Type Tiers window.

  1. Select ≡ > Admin > System Administration.

    The System Administration window appears.

    In the Explorer pane, expand IPK Management.

  2. Select Quick Solutions to open the window. A browse table at the top of the window shows all existing quick solutions. To view any that have been deleted, select the Deleted radio button.
  1. Select the entry you wish to restore.
  2. Select Undelete. This moves the quick solution to the Active list again and will make it available in the system for analysts to apply to calls.
  3. Select to save the changes. Provide the Change Reasons if prompted to do so.
