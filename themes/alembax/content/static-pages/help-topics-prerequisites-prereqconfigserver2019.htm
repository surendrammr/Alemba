Configuring Windows Server 2019 for Alemba Service Manager™

Before you start

Ensure that your server and client machines meet the prerequisites. For more information, see Prerequisites checklist.

If you are using Server 2016, see Configuring Windows Server 2016 for Alemba Service Manager™.

If you are using Server 2012, see Configuring Windows Server 2012 R2 for Alemba Service Manager™.

Alemba® does not support Oracle databases for ASM 10.1.

To configure Windows Server 2019 for ASM Core:

  1. Using Server Manager, select Dashboard, then select Add roles and features to start the Add Roles and Features Wizard.
  2. On the Before you begin page, ensure that you have completed the listed tasks.
  3. Select Next.
  4. On the Select installation type window, select Role-based or feature-based installation, then select Next.
  5. On the Select destination server window, select the server that you are using, then select Next.
  6. On the Select server roles window, select Web Server (IIS) to display a popup requesting to add required features for Web Server (IIS).
  7. Select Add Features to add IIS Management Console to the server and return to the Select server roles window.
  8. Select Next to continue.

  9. In the Select Features window, ensure all the following options are selected:

    • .NET Framework 4.7 (or higher)
      • .NET Framework 4.7 (or higher)
      • ASP.NET 4.7 (or higher)
      • WCF Services
        • HTTP Activation
        • TCP Activation
        • TCP Port Sharing

    10. You are notified if any additional features are required for HTTP Activation. Click Add Features to add the features and return to the Select Features window.

  10. Select Next to view the Web Server Role (IIS) window, which contains information about the web server.

    11. You may not see this window if you are upgrading.

  11. Select Next to continue.

  12. On the Select role services window, ensure that all the following options are selected:

    • Web Server
      • Common HTTP Features
        • Default Document
        • Directory Browsing
        • HTTP Errors
        • Static Content
        • HTTP Redirection
      • Health and Diagnostics
        • HTTP Logging
        • Logging Tools
        • Request Monitor
      • Performance
        • Static Content Compression
      • Security
        • Request Filtering
        • Basic Authentication
        • Client Certificate Mapping Authentication
        • Digest Authentication
        • IIS Client Certificate Mapping Authentication
        • IP and Domain Restrictions
        • URL Authorization
        • Windows Authentication
      • Application Development
        • .NET Extensibility 3.5
        • .NET Extensibility 4.5 (or higher)
        • Application Initialization
        • ASP
        • ASP.NET 3.5
        • ASP.NET 4.5 (or higher)
        • ISAPI Extensions
        • ISAPI Filters
        • WebSocket Protocol
      • Management Tools
        • IIS Management Console
        • IIS Management Scripts and Tools

    13. Alemba Service Manager™ provides its own Dynamic Content Compression, which performs slightly better than IIS Dynamic Content Compression. You can choose to disable IIS Dynamic Content Compression.
    If you disable IIS Dynamic Content Compression, the Web Server (IIS) Support Role is also disabled. We therefore recommend that you review the Role Services enabled by the Web Server (IIS) Support Role and manually re-enable those required for your environment.

  13. Select Next to continue.

  14. On the Confirm installation selections window, confirm your choices.

  15. Select Install.
  16. When installation completes, select Close to close the wizard.
