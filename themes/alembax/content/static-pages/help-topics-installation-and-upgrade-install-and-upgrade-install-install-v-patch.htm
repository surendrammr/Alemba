About Installing or Patching to ASM 10.1

You may need to install any of the following:

  • Alemba Service Manager x Setup.msi – for installing new systems.
    For more information, see Installing Alemba Service Manager™.

  • Alemba Service Manager x Patch.msi – for upgrading systems.
    For more information, see Upgrading your system.

    Where x is the ASM version number.

Depending on your circumstances, you may need to do the following:

What I want to achieve What I need to do
I want to create a brand new vanilla System Install the latest ASM Setup file, and then use the Server Console system creation wizard.
I want to upgrade my existing v9.x system on the same server

Always take a backup of your system and database before performing an upgrade.

Install the latest ASM Patch file on the same server as your existing system.

The Alemba Service Manager™ Patch Tool will upgrade the system files and database.
I want to upgrade my existing v9.x system, and move it to a new server at the same time

Always take a backup of your system and database before performing an upgrade.

  1. Install the latest ASM Patch file on the old server and upgrade the database.
  2. Decommission the old server – turn off the Services and IIS Admin Service, and set their Startup Type setting to Manual.
  3. Install the latest ASM Setup file on the new server.
  4. On the new server, create a new system using the Server Console system creation wizard.
  5. During system creation, point to the existing database, and when asked if you wish to upgrade the database, select No.

If upgrading a system on v9.2 or below, please follow the instructions in Upgrading from vFire 9.2 or Below.
I want to upgrade between minor point releases.

Always take a backup of your system and database before performing an upgrade.

You can upgrade between minor point releases (e.g. v10.5.2 to v10.5.4) by applying the Patch file.
I have a v8 System I want to upgrade

The minimum version from which you can upgrade from v8 to v9 is v8RP40. If you have a version earlier than this, you must upgrade to v8RP40 before attempting to upgrade to v9. After you have upgraded to v9, you can upgrade to 10.1.

Always take a backup of your system and database before performing an upgrade.

Follow the instructions in Upgrading from vFire 9.2 or Below.
