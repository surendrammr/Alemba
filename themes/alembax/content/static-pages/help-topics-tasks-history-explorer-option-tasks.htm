History Explorer Option - Tasks

The task history shows all actions carried out for the task at previous stages, including both the notes entered in the Action Information field by handling analysts and automatic entries generated by the system.

Click the History explorer option to .

You can view the current task's history, such as, who created it, its description, or who owns the task from the Task Details window.

  1. Select the History explorer option or click the View History button.
  2. The Task <number> - History window displays the task history for the current task.
  • If the task is escalated (that is, the planned start date or the target date has exceeded), and the Escalation Log option is selected in the Workflow Management Settings, an entry will be added to the Task History.

  • The Physical Date appears on the left-hand side which is indicated by a label with the same name. This is the database server date/time stamp at the time of the action on the task.

  • By default, the history displays all actions on a request in ascending chronological order of the physical date. However, the request description is always displayed first, regardless of the order selected. You can use the buttons to sort the events in Descending or Ascending chronological order.

    Task action times are displayed according to the local server time, regardless of the time zone in which the analyst is located.

  • When you open the Task History window first, a summary of all the actions appears. To view the details, click Expand All to expand all the entries displayed on this window. If you navigate out of this window, ASM will remember this setting and when you view the history for this task again, it will display the history entries in an expended format. To go back to the summary view again, click this button again. (It will have changed to a Contract All button.) If you expand only individual items using on a particular item, this action will not be remembered.
