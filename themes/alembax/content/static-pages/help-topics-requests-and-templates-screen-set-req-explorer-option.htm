Screen Set Explorer Option

Where a number of different Request Details windows have been created through ASM Designer, the Screen Set explorer option enables you to change the request screen, by selecting another screen set which contains your custom request details screen.

 

Custom request screens (including Request Details windows, submission forms, and portal review screens) created in ASM Designer must be associated with a screen set. The Screen Set option on the Request Details Explorer allows you to choose a custom request screen set and thus use another Request Details window, that is, the one associated with the screen set you select.

Selecting a custom request screen set, also allows you to associate a custom submission form with a request or workflow template. Submission forms are used by Users when logging requests through the Self Service Portal.

Before you start

The Screen Set Explorer option is only displayed if Multiple Task Screen Sets is selected on the Workflow Management Settings window.

You must have access to more than one task screen set on the Requests Screen Sets tab of your Workflow Management security role.

 

  1. In the Request Details window, select the Screen Set explorer option.
  2. In the Request Screen Set window, select another screen set. All currently defined task and approval screen sets appear.

    3. If you select an incompatible screen set, you will receive an error message and see a list of compatible screen sets.

  3. The Request Details window will refresh to use the new details window in the selected screen set.
  4. Any details that have already been added to the request will be copied into the equivalent fields, where they are displayed on the new window. Even if these fields are not displayed on the selected screen, the information is stored in the database.
