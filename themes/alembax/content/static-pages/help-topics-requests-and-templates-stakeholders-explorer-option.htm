Stakeholders Explorer Option

The Stakeholders Explorer option on the Request Details window enables you to link stakeholders to the current request so that they are informed of developments while the request is being implemented. Stakeholders might include users, analysts, organizations and groups.

Â 

Any stakeholders linked to a CMDB item, for which a call or request has been logged, are not carried over as stakeholders. Stakeholders define a relationship between the call or request and various parties, and are associated with the call or request when they are manually added.

  1. Select the Take Action button on the Details window if necessary to allow you to work on the call/request.
  2. Select the Stakeholders option from the Workflow group to display the Stakeholders window. This window displays a list of any currently defined stakeholders.
  3. To add a stakeholder, select a stakeholder type from the Stakeholder Type drop-down list:

    4. Person to add a user, analyst or external contact as a stakeholder
    Organization
