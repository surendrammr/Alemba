Managing Scheduled Integration Scans

You can schedule integration scans in order to synchronize ASM Core with sources. The schedule will be applied to all sources specified in the schedule details. These are the sources which have been defined and mapped to be integrated with ASM Core.

You can view all integration schedules, that is, the scheduled scans configured to run on sources.

Adding a scheduled scan

Before you start

The ASM Core Connector Polling Service must be running in order for scheduled scans to run in the defined period.

You must have Integration Setup selected in the Admin tab of your General Access Security Role.

  1. Select ≡ > Admin > Integration.

  2. In the Explorer pane, under Integration, select Scheduling.

    A browse table appears, listing the existing scanning schedules.

  3. Select and complete the details.
  4. Select to save the changes.

Updating a scheduled scan

Before you start

The ASM Core Connector Polling Service must be running in order for scheduled scans to run in the defined period.

You must have Integration Setup selected in the Admin tab of your General Access Security Role.

  1. Select ≡ > Admin > Integration.

  2. In the Explorer pane, under Integration, select Scheduling.

    A browse table appears, listing the existing scanning schedules.

  3. Select the schedule you want to update and either double-click or select to display the details.
  4. Update the details as necessary.
  5. Select to save the changes.

    6. Run the connector service to ensure that you are importing the latest resources.

    After a scan has run, select the Scheduling explorer options once more, and select the Activity tab to check the status of the schedule scan for the specific time.

Deleting a scheduled scan

Before you start

The ASM Core Connector Polling Service must be running in order for scheduled scans to run in the defined period.

You must have Integration Setup selected in the Admin tab of your General Access Security Role.

  1. Select ≡ > Admin > Integration.

  2. In the Explorer pane, under Integration, select Scheduling.

    A browse table appears, listing the existing scanning schedules.

  3. Select the schedule you want to delete, then select .
  4. Confirm the deletion. Select to delete the schedule from the database.

Specifying the Integration Schedule Details

Before you start

Ensure you have defined sources as you will need to select the source/s for the schedule.

You must have Integration Setup selected in the Admin tab of your General Access Security Role.

  1. Select ≡ > Admin > Integration.

  2. In the Explorer pane, under Integration, select Scheduling.

    A browse table appears, listing the current scanning schedules.

  3. Select Add.
  4. In the Name field, specify a name for the scan.
  5. Complete the details.

    6. Frequency Select how often you want the system to perform a scan.

    You can schedule scans to occur hourly, daily, weekly, monthly, or as a Once-Off scan. Selecting Once-Off will execute the scan on the date specified at Next Scan.

    For integration with a directory server, this is how often you want to synchronize person data from directory servers that are integrated with ASM Core. The frequency indicates how often the polling service scans the mapped directory server/s for user and group data.

    For integration with a network discovery tool, this is how often you want a scan to be run on the source to discover resources. The frequency indicates how often the polling service scans the mapped network discovery tool for new or updated resources.
    Next Scan Select the date and time at which you want the schedule to start. If you selected a frequency of Once-Off, this is the date on which the once-off scan will take place.
    Update All Records

    This option only applies to directory server integration sources.

    Select to enable a complete scan of each person record imported into ASM Core from a directory server. This means the system will scan every directory server user regardless of whether their information has been updated.

    Clearing this option enables an incremental scan to be performed. The system will locate user details that have been updated or add person records if there are new users in the directory server.
    Scan Type

    Select the scan type you wish to perform.

    Incremental

    Select this option to enable an incremental scan to be performed. The system will locate analyst or user details that have been updated or add records if there are new analysts or users from the directory server.

    This is the default option.
    Full then Incremental Select this option to run a full scan initially but run an incremental scan for all other subsequent scans.
    Full Select this option to enable a complete scan of each analyst or user imported into ASM Core from a directory server. This means the system will scan every person record, regardless of whether their information has been updated.
  6. Select the Sources tab to add the sources for the schedule. Select Add, select a source and click OK. (You can also remove a source and change the order of the sources on which the schedule will run.)

    7. Scans are run on each source in the order defined here, with the top-most source being the first source to be scanned. This means that data imported from a source may be overwritten by data imported from a source lower on the list. If a source that has been added to a schedule is later designated as Inactive, it will not be scanned.

  7. Select the Notifications tab to add the people who need to be notified by email when a scan completes. Click Add. The Select Schedule Notification Recipients window appears. Find and add the analysts, IPK and Workflow groups, or email addresses you wish to receive notifications for this schedule. To remove a recipient, select the added recipient and click Remove.
  8. Select the Activity tab to view the status of any scans which were run for this schedule. You can click the toolbar button to update the Activity, for instance, if you wish to view the status of the current scan.
  9. Select to save the changes.
