Logging a Request in Self Service Portal

You can log a request in Self Service Portal. You will be offered the appropriate template for your request type, and it will be automatically assigned accordingly.

You must have Log Requests selected in the Options tab of your Self Service Portal role.

Analysts linked to users can log on to the Self Service Portal and log requests on behalf of a user.

  1. On the My Options menu, select Requests > Log a Request.
  2. A list of Change Management templates appears. Select the one that matches the nature of your request.
  3. The Log a Request page appears. The fields will differ depending on the template selected. However, there is a number of standard fields that are displayed for most requests.

    4. Request Services Identify the service you want to log a request against. You can use the button to do a search if you wish.
    Type Use the drop-down tiered list to identify the problem type. There may be several tiers of problem type. You can select from any tier.
    Priority Use the drop-down list to identify the priority of the request.
    Description Type a free text description of the request.
    Send Email

    Select the checkbox to send an email to the analyst to whom the request is assigned.

    Any request logged via Self Service Portal will automatically send an email confirmation to the user.
    Send Pager Select the checkbox to send a pager message to the analyst to whom the request is assigned.
    Attach File If you have screen shots or additional information relating to your request contained in a file, you have the option to attach the file. Click Browseâ€¦ and select the file(s) you want to attach.
    Forward To Select the support analyst or group to whom your request should be assigned. If an analyst or group has already been specified in the template, you need not select an analyst or group from this list. However, if a recipient has not been specified in the template, you must select one here. This is a mandatory field, and unless completed, the system will not enable you to proceed.
  4. Click Submit to submit the request. The request is forwarded to the selected support analyst or group you selected in the previous step. Once you have submitted your request, the system will display an acknowledgment with the Request No and a summary of the request that you logged.
  5. To defer the form for later submission, click Submit Later. If an object is attached or a service is selected on the form, the object will be uploaded and the service linked. The details of the fields you have completed are displayed and the link Edit/Submit this Request appears below the fields. Click this link to re-open the submission form. Alternative, this form can be accessed via the Review Past/Current Requests by selecting the Not Submitted option.

    6. If you select Submit Later, the value selected in the Forward To field will not be saved. When you re-open the form for submission, you will need to select a value for this field again. The object/service will be displayed as a hyperlink when the form is re-opened.

Analysts cannot action approvals for requests they have logged themselves, even if these are assigned to them. They can only view these approvals via the request itself.
