Transactions Explorer Option

A transaction is simply a way of designating how a particular quantity of licenses/items is being used in the CMDB. It is an action you can perform on a Software Product or Inventory CMDB item, such as a new order or new purchase, allocation, or reservation of a quantity of licenses/items. Transactions can also be automated through requests.

A transaction transfers licenses/items to and from various “transaction pools”. For instance you might have 25 licenses for a software product, Microsoft Word, in the On Hand pool and 60 in the In Use pool, meaning that you have 25 available to allocate to users, and 60 already allocated. After each transaction, ASM automatically recalculates the levels in each pool (that is the quantity of items).

The Transactions Explorer option enables you to initiate Asset Management transactions from the current task/call. Asset Management transactions include receiving, updating and deleting asset orders, and allocating assets to people and/or CMDB items.

  1. In the Explorer pane, expand Data, then select Transactions.
  2. The Transactions window has two tabs:

  • Orders allows you to manage the orders added to the task for software product or inventory items you have ordered but have not yet received.   
  • Purchases allows you to manage the purchases added to the task for software products or inventory items you have received or purchased. The state of these items can change, that is they can be in use or reserved. Each state (also referred to as a “transaction pool”) is represented by the tabs or views in the Purchases view (that is In Use and Reserved). 
  • Click Orders to view a list of order records for a software product or inventory that were added to the current task. The browse table for the Orders List has columns for all the order transaction details. A blank value in a column means that this transaction detail was intentionally left unspecified or it is not relevant to the particular transaction. Click Filter Columns to show or hide columns.
  • Select Display Incomplete Orders only to only show orders which are not yet complete, that is, orders for which you have only received some of the items in the order. Clear the option to show both complete and incomplete orders. You can add multiple orders of an item. A record is created for each order you add. If there are multiple orders, ASM automatically calculates the total quantity of items in all the orders to display in the Item Summary.
  • Click Purchases to view a list of purchase records for a software product or inventory that were added to the current task. This view has two browse tables. The top browse table is the list of purchases, and has columns for all the purchase transaction details. The bottom browse table has two tabs or views, In Use and Reserved. Each view represents the state of the purchased items after a particular transaction was performed. Note that a blank value in a column means that this transaction detail was intentionally left unspecified or it is not relevant to the particular transaction. Click Filter Columns to show or hide columns in the browse table. If there are orders, the Update, Receive Order, and Delete buttons are enabled.
  • Select Show transactions for selected purchases only to only display the items in the bottom browse table for a selected purchase. You can add multiple purchases of an item. A record is created for each purchase you add. Clear this option to list the items in all purchases. If there are multiple purchases, ASM automatically calculates the total quantity of items in all the purchases, as well as the number of items at a particular state, to display in the Item Summary.
  • If there are purchases, the Update and Delete buttons are available. This enables action buttons below the relevant views, In Use and Reserved. The buttons provide specific actions you can perform on the items in the selected purchase pool.
  • Click In Use to view items in the In Use transaction pool. Items are transferred to the In Use pool after you allocate them from the On Hand and Reserved pools. You can Return to On Hand from the In Use pool.
  • Click Reserved to view items in the Reserved transaction pool. Items are transferred to the Reserved pool after you reserve them from the On Hand pool. You can Allocate and Return to On Hand from the Reserved pool.
