Jurisdictions Explorer Option

A jurisdiction is a defined group of cost centers and organizations. You can allocate jurisdictions to an Analyst to designate the cost centers and organizations they can select when making transactions against Software Product or Inventory records. You can add and remove jurisdictions, as well as view the details of a jurisdiction linked to an Analyst.

Allocating a Jurisdiction to an Analyst

Before you start

The jurisdictions must be defined in the Jurisdiction window.

The person must be identified as an Analyst in the person details.

You must have Analyst Update selected in the People tab of your Configuration Management security role.

  1. Display the Details window, if it is not already on screen. Select if necessary to display the details in edit mode.
  2. Select the Jurisdictions explorer option. This usually appears within the Data group. Scroll up to locate the group, and expand it if necessary.
  3. If there are already jurisdictions linked to the person, the option will appear in orange text with a small dot to its left.
  4. You must be in edit mode to make any changes to the person details, so select from the toolbar if necessary to enable the fields on the window.
  5. Select Add. The Jurisdiction Search window appears.
  6. Specify your search criteria and select Search.
  7. Select the jurisdiction(s) you want to allocate to the Analyst from the browse table.
  8. Select the button. The Person Jurisdictions window appears once more, with the linked jurisdictions shown on the table.
  9. Select from the toolbar or Return To Main at the top of the Explorer pane to return to the Details window.

Deleting a Jurisdiction from the Person Details

Before you start

You must have Analyst Update selected in the People tab of your Configuration Management security role.

  1. Display the Details window, if it is not already on screen. Select if necessary to display the details in edit mode.
  2. Select the Jurisdictions explorer option. This usually appears within the Data group. Scroll up to locate the group, and expand it if necessary.
  3. If there are already jurisdictions linked to the person, the option will appear in orange text with a small dot to its left.
  4. You must be in edit mode to make any changes to the person details, so select from the toolbar if necessary to enable the fields on the window.
  5. Select the jurisdiction you want to delete from the Jurisdictions browse table.
  6. Select Remove.
  7. Select from the toolbar or Return To Main at the top of the Explorer pane to return to the Details window.

Viewing Jurisdictions Linked to Person Details

Before you start

You must have Analyst View selected in the People tab of your Configuration Management security role.

  1. From the Details window, select the Jurisdictions explorer option. This usually appears within the Data group. Scroll up to locate the group, and expand it if necessary.
  2. If there are jurisdictions linked to the person, the option will appear in orange text with a small dot to its left.
  3. Select the jurisdiction you want to view in the browse table.
  4. Select Open. The Jurisdiction Details window appears.
  5. To return to the Person Details window, close the Jurisdiction Details window, or select the Person Details tab at the top of the screen. Then select or at the top of the Explorer pane to return to the details window.
