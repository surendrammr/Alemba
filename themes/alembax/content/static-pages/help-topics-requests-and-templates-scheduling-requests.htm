Scheduling Requests

Analysts can schedule requests to be created at a future date, and view and search for these schedules.

Before you start

Admin Request Schedules must be selected in the Requests tab of your Workflow Management security role.

To schedule a request to be created at a future date:

  1. Select ≡ > New > IPK and Workflow Schedule.

    The Schedule and IPK or Workflow from a Template window appears.

    2. Alternatively, select ≡ > View > IPK and Workflow Schedule, then select from the toolbar.

  2. Select the Request Template option, and complete the details.

    3. Using Template Use the Search to find the workflow template that will be used to create this request from the list.
    Title

    Specify a name for the request schedule

    This field is only used to describe the title of the schedule; it is not transferred to the request itself
    Description

    Type in the relevant information about the request schedule you are creating.

    This field is useful if you wish to provide details as to why this request is being scheduled. The details are not transferred to the request.
    Schedule For Specify the date and time on which the request should be created
    Repeat Use the drop-down list to specify how often this request should be created. You can select the request to be created once (no repeat of this request), daily, weekly, fortnightly, monthly or quarterly
  3. Click OK. The selected request is now scheduled for the time and date specified.

Viewing Scheduled Requests

Analysts can view scheduled requests from the Scheduled Requests window.

Before you start

View Request Schedules must be selected in the Requests tab of your Workflow Management security role.

  1. Select ≡ > New > IPK and Workflow Schedule.

    The Schedule and IPK or Workflow from a Template window appears.

  2. By default, all calls and requests scheduled for the current day are displayed in the browse table on the right pane of the window. To view the details of a scheduled call, double-click the name of the call on the browse table.
  3. Use the search pane to specify your filtering criteria:

    4. Analyst Partition

    Use this list to display scheduled requests that were created within this partition.

    To search for scheduled requests created in all the partitions to which you have access, select Your Partitions.
    Analyst

    Use this field to display the scheduled requests of a particular analyst.

    For the search to return the scheduled requests for that analyst, ensure that the Analyst checkbox is selected.
    Type of Schedule Select the Requests checkbox to view scheduled requests.
    Schedule Status Select Active to display current scheduled requests, and/or Deleted to display scheduled requests which were deleted before the request’s time of scheduling
    Request Status Select Pending to display scheduled requests which have not yet been created, and/or  Created to display scheduled requests that have already been created (that is, whose scheduling time of the request has passed)
    Date From and Date To Use these fields to specify the date range in which to display scheduled requests
    Today Select this option to display requests that have been scheduled for the current day
    Day Select this option to display requests scheduled on the date specified in the Date From and Date To fields
    Week Select this option to display requests scheduled during the current week
    Month Select this option to display requests scheduled during the month ending on the current date

Whenever you change any search parameters in the search pane, the browse table on the right automatically refreshes to display any matching scheduled requests.
