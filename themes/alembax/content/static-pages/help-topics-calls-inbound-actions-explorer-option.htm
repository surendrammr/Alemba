Inbound Actions Explorer Option

If the current call was logged as a result of an Inbound Action received from another system, you can view the messages exchanged between ASM and the other system relating to the Inbound Action.

  1. Display the Call Details window, if it is not already on screen.
  2. In the Explorer pane, expand Data, then select Inbound Actions.
  3. The Inbound Actions window appears. The top part of the window displays the following information about the Inbound Action:
    • Source: The name of the external system which generated the Inbound Action.
    • Type: The name of the Inbound Action.
    • Reference: The reference ID of the Inbound Action generated.
  4. View the action information in the browse table. The table displays the following information about each action transaction:
    • Date/Time: The date and time the transaction was sent.
    • Origin: The name of the system that sent the transaction (either ASM or the external system)
    • Transaction Type: The type of transaction sent (such as CREATE, UPDATE, CANCEL)
    • Details: The content of the transaction, if any.
    • Status: The status of the transaction, that is, Succeeded or Failed.
