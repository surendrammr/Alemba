Enabling User Management Server for Active Directory authentication

You can authenticate your Dashboard users and passwords using Active Directory.

For more information about setting up ASM for Active Directory authentication, see Directory Server Parameters.

Configure Active Directory authentication before you connect ASM to the User Management Server so that the ASM users are imported with the correct authentication method.

  1. Using the User Management Server, in the left-hand menu bar, select to view the Settings.

  2. On the User Directory tab, configure the following Windows Active Directory settings:

    Setting Description
    Username and Password Credentials of the Active Directory user account with rights to read the information you specify in the Distinguished Name field.
    LDAP URL(LDAP://) The LDAP protocol where your Active Directory Domain Services are enabled, in the following format:

    LDAP://DNS, where DNS is the Domain Name System or IP address.
    Distinguished Name

    The Active Directory DN, which you can define using any of the following in a comma-separated list:

    String – Attribute Type

    CN – Common Name

    OU – Organizational Unit

    DC – Domain Component

    For example:

    To search and list all the users in the domain www.example.com, type: DC=www,DC=example,DC=com

    To search and list all the users in a specific group (Support) of an organization (Alemba) in the domain www.example.com, type:

    CN=Support,OU=Alemba,DC=www.example,DC=com
    Enable SSL

    If you have configured an SSL connection between your Windows Server and Active Directory, activate Enable SSL in the User Management Server by moving the toggle to the right.
    Port number

    Default for non-SSL connection: 389.

    Default for SSL connection: 636.
    Import Users While Login If an Active Directory user logs into the Dashboard and does not yet exist in the User Management Server, enabling this option will immediately import the user into the User Management Server. Activate this option by moving the toggle to the right.

  3. In the top-right corner, select the Synchronization drop-down list, then select AD schedule synchronization.

    The Active Directory Synchronization Schedule appears.

  4. Select the Enable Active Directory Synchronization Schedule check box.

  5. Configure the following settings:

    Setting Description
    Type The synchronization schedule frequency, either hourly, daily, weekly, monthly or yearly.
    Recurs A dynamic option that changes according to your Type selection. How often you want the schedule to recur.
    Time The time at which the synchronization takes place.
  6. Select Save to save the synchronization settings.
  7. In the left-hand menu bar, select to view the Home page.

  8. Under Users, select Import Users > Active Directory.

    The Active Directory users are displayed.

  9. If required, refine the returned results by typing a search term into the Search users box.

  10. Select the checkbox alongside each user you want to import into the User Management Server.

    Alternatively, select the checkbox alongside Name to select all users. Clear the checkbox to remove the selection.

  11. In the top-right corner, select Import and Activate.

    Duplicate accounts are highlighted and are not synchronized into Active Directory.
